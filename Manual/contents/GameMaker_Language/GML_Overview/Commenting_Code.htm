Código de comentarios

Cuando se trabaja en colaboración con otras personas, cuando se trata de un proyecto grande, o incluso sólo para su propia referencia y depuración, dejar comentarios en su código de eventos y scripts es muy importante. En GameMaker Studio 2 tienes varios mecanismos para ayudarte a dejar notas y comentarios sobre secciones de código e incluso bloques enteros de código que pueden verse en la lista de eventos del objeto que estás editando en ese momento. De este modo, puedes dejar notas para ti mismo y para tus colegas, o para explicar un trozo de código especialmente complicado, o incluso dejar un recordatorio sobre lo que hace algo, o lo que quieres que haga algo en el futuro.

Para dejar un simple comentario de una línea, lo primero que puede hacer es utilizar // antes del texto. Por ejemplo:

//Inicializar variables
sr = 10;
hp = 100;

También puede dejar comentarios de varias líneas, para dar crédito, para omitir una sección completa de código para la depuración, o incluso para dejar notas sobre cómo quiere cambiar el código en el futuro. Para ello puede utilizar /* ... */ como este:

/*
Configurar las variables del GIF a los valores iniciales (estos pueden ser editados para que no sea necesario usar las funciones setter todo el tiempo si siempre está usando los mismos valores)

¡IMPORTANTE! Por favor, vea los scripts individuales de "set" para una información más extensa sobre cada una de las diferentes variables.
*/

frame_num = 30;
frame_steps = 5;
frame_width = 256;
frame_height = 256;
frame_quality = gifcam_quality_high;

Además de los comentarios, también puede utilizar etiquetas especiales #region para marcar una sección de código para el plegado de código, y cada región puede contener un comentario. La forma en que funciona es añadiendo una etiqueta #region - junto con cualquier texto de comentario - en la parte superior de una sección de código y luego añadiendo la correspondiente etiqueta #endregion en la parte inferior. de una sección de código y luego añadir una etiqueta #endregion correspondiente a la parte inferior de la sección y entonces usted puede "doblar" el código, es decir, ocultarlo para que pueda reducir el número de líneas de código visible en el editor en cualquier en cualquier momento:

El código real es algo así:

#región Esta es una región de ejemplo

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Comente más aquí si es necesario

Por último, vale la pena mencionar que cuando se trabaja con eventos en el Editor de Objetos , se puede utilizar un comentario especial "description" al comienzo del evento para decir cuál es el contenido general del evento, y esto se mostrará en la ventana de eventos del Editor de Objetos. Esto debe ir en la primera línea del editor de código, y tiene el siguiente formato:

/// @description <Esta es una descripción del evento>

Esto se mostrará ahora en la ventana de eventos, por ejemplo

Tenga en cuenta que puede definir un comentario por defecto que se creará para todos los eventos desde las Preferencias del Editor de Objetos.

Finalmente, si está escribiendo funciones de script y quiere tener algún control sobre cómo son tratadas por el IDE (por ejemplo: tener autocompletado o mostrar los parámetros del script) entonces debería escribir sus comentarios usando JSDoc como se explica aquí.
