Estructura básica del código

El código se escribe en bloques y un bloque de código típico consiste en un conjunto de instrucciones, llamadassentencias, que luego son interpretadas por GameMaker Studio 2 y se utiliza para hacer algo dentro de su juego. Ese "algo" puede ser tan simple como sumar 2 y 2 para obtener 4, o tan complejo como hacer que un enemigo huya cuando su salud esté por debajo de un determinado valor. La estructura real del programa puede variar en gran medida, dependiendo de las funciones que utiliza, pero desglosado a lo básico sólo se parece a esto:

<declaración>;
<declaración>;
...

Las declaraciones deben estar separadas con un símbolo ';' para evitar errores con las declaraciones de variables y para mantener su código limpio y ordenado, y pueden consistir en declaraciones de variables, expresiones y llamadas a funciones específicas. También puede "agrupar" las sentencias como un bloque utilizando las llaves {} para que se ejecuten juntas, como en el siguiente ejemplo condicional:

if (<condicional>)
{
<declaración>;
<declaración>;
...
}

NOTA: El Lenguaje GameMaker también acepta el inicio y el final en lugar de los corchetes {}, aunque esta no es la forma más común de hacerlo:

if (<condicional>)
begin
<declaración>;
<declaración>;
...
end
