Funciones detiempo de ejecución

La definición general de una función es algo así:

Una función tiene una entrada y una salida, y la salida está relacionada de alguna manera con la entrada.

En GameMaker Studio 2 esto se traduce en dos cosas diferentes, pero ambas funcionan de la misma manera:

  • Funcionesen tiempo de ejecución - Una tiempo de ejecución función es aquella que se suministra como parte del Lenguaje GameMaker(GML). Están incorporadas al lenguaje y pueden utilizarse para hacer que ocurran cosas en tus juegos y hay un gran número de funciones de tiempo de ejecución de GML disponibles, todas ellas explicadas en la sección de Referencia del Lenguaje del manual.
  • Funciones y métodos de script - Script funciones y Metos Variables son funciones que has creado tú mismo usando los bloques básicos de bloques de la Lenguaje GameMaker(GML), y pueden incluir funciones de ejecución como parte de su código.

En ambos casos, una función tiene la forma de un nombre de función, seguido de los argumentos de entrada entre corchetes () y separados por comas (si la función no tiene argumentos de entrada entonces sólo se utilizan los corchetes). Esta página se concentra en las funcionesde tiempo de ejecución (es decir, las que están incorporadas a GameMaker para formar el Lenguaje GameMaker), pero las reglas generales mostradas abajo siguen para todos los tipos de funciones.

A continuación se presenta un esquema de la estructura de una función:

<función>(<carga0>, <carga1> ,... <cargaN>);

Algunas funciones pueden devolver valores y pueden utilizarse enexpresiones, mientras que otras simplemente ejecutan comandos, como se ilustra en los dos siguientes ejemplos de funciones en tiempo de ejecución en tiempo de ejecución:

// Destruye la instancia que llama
// No requiere argumentos y no devuelve nada
instance_destroy();

// Obtiene la distancia desde la posición actual de la instancia a la posición del ratón
// Toma cuatro argumentos y devuelve un valor real
dist = point_distance(x, y, mouse_x, mouse_y);

NOTA: En cualquier lugar del manual donde aparezca "N/A" como valor de retorno de una función, significa que la función no está destinada a devolver ningún valor como parte de su operación. Si intenta comprobar el valor de retorno de estas funciones, obtendrá el valor obtendrá el valor undefined.

A veces, el valor de retorno de una función puede ser un valor que se quiere utilizar en una asignación, pero hay que tener en cuenta que es imposible utilizar cualquier función directamente como el lado izquierdo de una asignación. de una asignación. Por ejemplo, el siguiente código para establecer la velocidad en una instancia le daría un error:

instance_nearest(x, y, obj).speed = 0;

El valor de retorno de la expresión en ese ejemplo de código es un número entero (el valor de identificación único para la instancia más cercana) y por lo tanto debe ser encerrado entre paréntesis para ser utilizado de esta manera y dirigirse correctamente a la instancia requerida (ver Direccionamiento de variables en otras instancias para más información). El código anterior se escribiría correctamente como

(instance_nearest(x, y, obj)).speed = 0;

/o más correctamente aún

var inst = instance_nearest(x, y, obj);
inst.speed = 0;

La sección de Referencia del Lenguaje del manual enumera todas las funciones de tiempo de ejecución disponibles y da ejemplos de cómo se pueden utilizar, así como otra información importante sobre lo que pueden devolver o los eventos que pueden desencadenar, etc... Para más información sobre las funciones y métodos de script definidos por el usuario, vea la sección sobre Funciones de Script y la sección sobre Variables de Métodos.
