Variables globales

Una descripción básica de una variable global es aquella que, una vez declarada, no pertenece a ninguna instancia en particular y, sin embargo, puede ser accedida por todas. Al igual que las variables locales, las variables globales deben declararse utilizando un identificador, pero a diferencia de una variable local, una variable global permanece en la memoria hasta el final del juego. Por lo tanto, puedes crear una variable global para llevar la cuenta de (por ejemplo) el número de balas que tiene el jugador y luego simplemente actualizar esta variable en diferentes puntos del juego, desde cualquier instancia o función y en cualquier momento. Esencialmente, una variable global no pertenece a ninguna instancia específica y puede ser accedida, modificada y utilizada por todas las instancias, y cualquier cambio realizado en la son "globales", en el sentido de que todas las instancias que utilicen la variable se verán afectadas por el cambio. Veamos un ejemplo:

global.food = 5;

Declaramos la variable "food" escribiendo primero "global" y luego un "." para decirle a GameMaker Studio 2 que esta variable es ahora de alcance global. Tendremos que utilizar esta forma a partir de ahora cada vez que tengamos que acceder o modificar esta variable de alguna manera. Así pues, hemos creado una nueva variable llamada "food" y la hemos declarado como global. Ahora, cualquier Ahora, cualquier instancia o función puede usar y cambiar esta variable de cualquier manera y todas las demás instancias la "verán". Por ejemplo podríamos tener un objeto de comida diferente con el que el jugador colisiona y en el evento de colisión tenemos:

global.food +=1;

También tenemos otro objeto que dibuja este valor así:

draw_text(32, 32, "food = " + string(global.food));

Con las variables globales podemos cambiar valores y ver esos cambios reflejados en todas las instancias de los objetos que hacen referencia a esta variable. Al igual que con las variables locales, hay que tener cuidado de no nombrar las variables globales igual que las variables de instancia ya que esto puede causar problemas y hacer que los bugs se cuelen en tus juegos debido a la superposición de variables, lo que puede ser un problema difícil de depurar a veces. En general, deberías tener un único objeto que declare todas tus variables globales al (por ejemplo, en el Evento de Inicio de la Sala de la primera instancia del objeto colocado en la primera sala del juego) o una sola función de script que las declare todas juntas, ya que esto le da un lugar práctico para volver y hacer referencia a todo a la vez en caso de que necesite comprobar un nombre de variable o editar un valor.

GameMaker Studio 2 también tiene una colección de variables globales "incorporadas", por lo que debes tenerlas en cuenta ya que puedes nombrar una de tus variables de instancia igual o desear tener tu propia variable global con el mismo nombre y preguntarte por qué estás errores. Sin embargo, son fáciles de detectar, ya que se muestran en un color diferente en el editor de código y también aparecen en la barra de autocompletar en la parte inferior.   • async_load

También hay tres variables globales incorporadas obsoletas que debe conocer (estas variables sólo están diseñadas para soportar proyectos heredados de versiones anteriores de GameMaker y deben no se puede utilizar en los nuevos proyectos):

  • puntuación
  • salud
  • vidas

Por último, hay dos variables que se pueden utilizar para las funciones y los métodos de los scripts:

  • argumento
  • conteo_de_argumentos

La siguiente forma también puede ser usada para declarar variables globales, pero es sólo se incluye por compatibilidad con versiones anterioresy no se recomienda utilizar este formulario para los nuevos proyectos, ya que es posible que las futuras versiones de GameMaker no lo admitan.

La segunda forma de crear variables globales es declararlas como tales mediante la declaración globalvar, de la misma forma que lo harías con una variable local mediante la declaración var.

IMPORTANTE! La declaración globalvar está obsoleta y sólo se admite para fines heredados. Siempre debe utilizar el identificador global. para marcar las variables globales.

Esta declaración (obsoleta) se utilizaría de la siguiente manera:

globalvar food;
food = 5;

Una vez declarada de esta manera, la variable "food" se considera ahora global y no requiere ningún prefijo global - lo que también significa que es mucho más difícil identificar las variables globales en tu código y también es mucho más fácil que se superpongan las variables al utilizar el mismo nombre de variable en diferentes objetos o desde extensiones que hayas instalado. Una vez declarada de esta manera, se accede a la variable global de la siguiente manera:

comida += 2;

o:

draw_text(32, 32, "food = " + string(food));

Como puedes ver, sin nada que muestre que la variable es de ámbito global, estás preparando potencialmente muchos problemas sutiles para tu juego, por lo que esta declaración debería evitarse.
