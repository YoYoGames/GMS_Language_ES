Funciones numéricas

Los números reales en GameMaker Studio 2 se consideran números de punto flotante de doble precisión, es decir, que tienen un punto decimal sin un número fijo de dígitos antes o después del punto. antes o después del punto - o enteros - es decir, son números enteros sin valor decimal. El 2, por ejemplo, es un número entero, pero el 2,01 es un real de coma flotante.

NOTA: En el objetivo HTML5, todos los números reales son dobles.

Esta distinción entre enteros y flotantes es muy importante cuando se trata de desarrollo entre plataformas, ya que la precisión de los cálculos realizados en un PC con Windows no es la misma que la de esos mismos cálculos cuando se realizan en un dispositivo móvil. Esto significa que hay que prestar especial atención al realizar comparaciones, por ejemplo:

if (image_index == 1)
{
hacer algo...
}

En el ejemplo anterior, si hemos establecido la velocidad de la imagen a 0,25 - por ejemplo - entonces después de cuatro pasos se puede asumir que el valor de image_index sería 1, pero puede ser un valor como 1.0000002 debido a la forma en que funcionan las matemáticas de punto flotante y por lo tanto la evaluación no será verdadera. Este tipo de errores puede ser bastante difícil de depurar, por lo que siempre es bueno ser consciente de ellos y planificar con antelación y utilizar otros medios de comprobación de valores o utilizar las funciones de suelo o redondeo apropiadas (listadas más abajo) para convertir el número a comprobar en un entero (para más información sobre las matemáticas en coma flotante y por qué esto es un problema, por favor, vea aquí). Por ejemplo, el código anterior podría escribirse como

if (floor(image_index) == 1)
{
hacer algo...
}

También hay que tener en cuenta que cuando se utilizan los objetivos del CompiladorYoYo, todas las expresiones y funciones se evalúan de izquierda a derecha, mientras que en el resto de plataformas objetivo se evalúan de derecha a izquierda, lo que significa que esto - por ejemplo - dará diferentes resultados dependiendo de la plataforma:

val = max(num, ++num, num++);

NOTA: Para más información, consulte la sección sobre la orden de evaluación.

También puede utilizar una función especial disponible en GameMaker Studio 2 para establecer el valor de épsilon para las matemáticas en coma flotante. Cuando un número real se redondea al número de coma flotante más cercano el épsilon (también conocido como "épsilon de la máquina") forma un límite superior en el error relativo, y puede obtener y establecer el valor del épsilon utilizando las siguientes funciones:

  • math_set_epsilon
  • math_get_epsilon

NOTA: Cuando se utilizan las funciones aleatorias, GameMaker Studio 2 mantiene la misma semilla aleatoria cada vez que se inicia el juego.
