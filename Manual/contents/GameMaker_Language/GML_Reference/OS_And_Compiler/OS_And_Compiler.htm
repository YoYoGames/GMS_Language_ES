Sistema operativo y compilador

Al crear juegos multiplataforma, suele ser de vital importancia conocer los detalles del sistema operativo que utiliza el dispositivo que ejecuta el juego. Cosas como el idioma, la versión o el estado de la red pueden servir para adaptar tu juego al sistema que lo ejecuta y hacer que la experiencia del usuario final sea la mejor posible, por lo que GameMaker Studio 2 tiene una serie de funciones especiales que se pueden utilizar para obtener la información necesaria.

Las siguientes funciones y variables existen para que puedas utilizarlas para obtener ciertos detalles sobre el sistema operativo o el navegador que está ejecutando tu juego:

  • os_browser
  • os_device
  • tipo_os
  • os_versión
  • os_is_paused
  • os_is_network_connected
  • os_get_config
  • os_get_language
  • os_get_region
  • os_get_info
  • os_powersave_enable
  • os_lock_orientation
  • os_check_permission
  • os_request_permission

Algunas funciones se precompilan cuando se ejecuta el juego (es decir, se utilizan para definir cómo se compilará el juego final), y otras también se utilizan cuando el juego se ha compilado para obtener detalles específicos sobre el entorno de ejecución. En puedes encontrar la lista completa de estas funciones:

  • GM_build_date
  • Versión GM
  • GM_runtime_version
  • gml_release_mode
  • gml_pragma
  • número_de_parámetros
  • cadena_de_parámetro
  • environment_get_variable

En Windows, la resolución del programador de hilos de Windows se establece por defecto en 1ms cuando se ejecuta el juego. Las siguientes funciones están disponibles para recuperar y cambiar la resolución del programador de hilos de Windows en tiempo de ejecución:

  • scheduler_resolution_get
  • scheduler_resolution_set
