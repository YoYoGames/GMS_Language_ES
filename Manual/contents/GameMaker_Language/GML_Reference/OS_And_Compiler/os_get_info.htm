os_get_info

Esta función devuelve un Mapa DS con información detallada sobre el sistema operativo en el que se está ejecutando el juego. La información exacta devuelta dependerá del sistema operativo y del dispositivo, por lo que se recomienda que pruebes esta función en todas las plataformas a las que desees dirigirte y que describas los valores devueltos utilizando una función como show_debug_message(). Tenga en cuenta que el mapa DS no se automáticamente de la memoria y debe utilizar la función ds_map_destroy() cuando ya no necesites los datos que contiene.

A continuación puede encontrar algunos detalles sobre los datos extra que se pondrán a disposición del mapa DS específicos para las siguientes plataformas, y debe tenerse en cuenta que en todas las plataformas , excepto en HTML5, el mapa tendrá la clave "is64bit" que será true o false dependiendo de si el sistema operativo de la plataforma es de 64 bits o no.

Windows / UWP

En Windows Desktop y Windows UWP, obtendrá algunos datos extra de DX11 con las siguientes claves disponibles en el mapa DS (útil para las extensiones):

  • video_d3d11_device - puntero al dispositivo DX11
  • video_d3d11_context - puntero al contexto DX11
  • video_adapter_vendorid - cadena con el identificador de proveedor de los adaptadores
  • video_adapter_deviceid - cadena con el id del dispositivo adaptador
  • video_adapter_subsysid - cadena con el id de subsistema del adaptador
  • video_adapter_revision - cadena con el número de revisión del adaptador
  • video_adapter_dedicatedvideomemory - cadena con el tamaño de la memoria de vídeo
  • video_adapter_dedicatedsystemmemory - cadena con el tamaño de la memoria del sistema (utilizada por el adaptador DX11)

Android

En la Android plataforma obtendrá algunos datos adicionales de varias API diferentes con las siguientes claves disponibles en el mapa de DS:

  • android_tv - devolverá true si se detecta que el dispositivo es un dispositivo Android TV, o false en caso contrario
  • GL_VERSION - la versión de OpenGL según el controlador
  • GL_VENDOR - el proveedor de OpenGL según el controlador
  • GL_RENDERER - el renderizador de OpenGL según el controlador
  • GL_EXTENSIONS - las extensiones de OpenGL que están disponibles según lo informado por el controlador
  • GL_SHADING_LANGUAGE_VERSION - la versión de GLSL que soporta este controlador OpenGL
  • GL_MAX_TEXTURE_SIZE - el tamaño máximo de la textura soportado por este controlador OpenGL
  • SDK_INT - valor de android.os.Build.VERSION.SDK_INT ver aquí para más información.
  • RELEASE - valor de android.os.Build.VERSION.RELEASE ver aquí para más información.
  • MODEL - valor de android.os.Build.MODEL ver aquí para más información.
  • DEVICE - valor de android.os.Build.DEVICE ver aquí para más información.
  • MANUFACTURER - valor de android.os.Build.MANUFACTURER ver aquí para más información.
  • CPU_ABI - valor de android.os.Build.CPU_ABI ver aquí para más información.
  • CPU_ABI2 - valor de android.os.Build.CPU_ABI2 ver aquí para más información.
  • BOOTLOADER - valor de android.os.Build.BOOTLOADER ver aquí para más información.
  • BOARD - valor de android.os.Build.BOARD ver aquí para más información.
  • VERSION - valor de os.version del método System.getProperty de Android
  • REGION - valores de user.region del método System.getProperty de Android
  • VERSION_NAME - valor del nombre de la versión de este paquete - ver aquí para más información.
  • PHYSICAL_KEYBOARD - si pensamos que hay un teclado físico entonces la cadena "TRUE" de lo contrario "FALSE"

Sintaxis:

os_get_info()

Devuelve:

Real (índice ds_map)

Ejemplo:

if (os_type == os_android)
{
var _info = os_get_info();

if (_info[? "android_tv"])
{
global.android_tv = true;
}
}

El código anterior comprueba si el sistema operativo actual es Android, obtiene la información del sistema operativo y lee el mapa devuelto para comprobar si el juego se está ejecutando en un Android TV. En ese caso, establece una variable global a true.
