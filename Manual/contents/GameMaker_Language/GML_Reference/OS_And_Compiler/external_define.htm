definición_externa

Esta función puede utilizarse para definir una llamada de función externa a una dll (para Windows) o dylib (para Mac) específica. Este archivo puede ser un archivo incluido o parte de una extensión. Se proporciona el nombre (y la ruta) del archivo y, a continuación, el nombre de la función que desea definir. A continuación, debe definir la convención de llamada que se utilizará (véase la lista de constantes más abajo), así como el tipo de resultado que se espera (también una constante, como se indica más abajo). Por último, debe indicar el número de argumentos que la (de 0 a 15) y para cada uno de los argumentos debe especificar también su tipo. Tenga en cuenta que para las funciones con 4 o más argumentos, todos ellos deben ser del tipo ty_real.

NOTA: Esto es sólo para dll o dylib que han sido añadidos como "Archivos Incluidos" al IDE de GameMaker Studio 2. No funcionará con esos archivos añadidos como extensiones, ya que esos requieren que usted defina las funciones en la extensión en el propio paquete de la extensión.

Constante Descripción
dll_cdecl Esta es la llamada por defecto de C, C++
dll_stdcall Esta es la llamada estándar de WinAPI( sólo paraWindows dll )
ty_real Un argumento de número real
cadena_de_tiempo un argumento de cadena terminada en cero

Sintaxis:

external_define(dll, name, calltype, restype, argnumb, argtype[0], argtype[1], ...argtype[10]) ;

Argumento Descripción
dll El nombre del archivo dll (cadena)
nombre El nombre de la función (cadena)
tipo de llamada La convención de llamada utilizada
restype El tipo de resultado que se espera
argnumb El número de argumentos (0 - 10)
argtype[0 ... 10] Los diferentes tipos de argumentos que se utilizan

Devuelve:

Real (Valor único de identificación de la función)

Ejemplo:

mi_funcion = external_define("MiDLL.dll", "MiMin", dll_cdecl, ty_real, 2, ty_real, ty_real);

El código de ejemplo anterior definirá una función externa llamada "MyMin" con dos argumentos.
