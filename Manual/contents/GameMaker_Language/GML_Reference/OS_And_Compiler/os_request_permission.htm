os_check_permission

Con esta función puedes solicitar un permiso específico. El permiso a solicitar se proporciona como una cadena usando el formato " android.permission.<permission> ", por lo que para solicitar el permiso RECORD_AUDIO (por ejemplo) se debe llamar a

os_request_permission("android.permission.RECORD_AUDIO");

La función devolverá la activación de un evento asíncrono del sistema en el que el mapa async_load DS contendrá los siguientes pares clave/valor:

  • " tipo": Es el tipo de evento que se activó y el valor será la cadena "permission_request_result".
  • " <permiso>": Será el permiso solicitado por la función como una cadena, que contendrá una de las siguientes constantes:
    Constante Descripción
    os_permission_granted Indica que el permiso ha sido concedido
    os_permission_denied Indica que el permiso no ha sido concedido

Cabe destacar que los siguientes permisos son soportados de forma nativa por GameMaker Studio 2, pero son considerados "peligrosos" por Google y como tal deben ser solicitados explícitamente (tenga en cuenta también que algunos permisos pueden ser solicitados usando las Opciones de Juegode Android sin necesidad de esta función):

  • android.permission.WRITE_EXTERNAL_STORAGE
  • android.permission.READ_PHONE_STATE
  • android.permission.RECORD_AUDIO

Para más información sobre los permisos de las aplicaciones, consulte la documentación deAndroid.

IMPORTANTE! Esta función es para el Android objetivo solamente.

Sintaxis:

os_request_permission(permission)

Argumento Descripción
permiso El permiso a solicitar (una cadena)

Devuelve:

N/A

Ejemplo:

si (os_type == os_android)
{
if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
{
os_request_permission("android.permission.INTERNET");
}
}

El código anterior comprueba el tipo de sistema operativo y si es Android, realiza una comprobación de los permisos y si el permiso "INTERNET" aún no ha sido concedido, lo solicita.
