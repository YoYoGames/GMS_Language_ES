rectángulo_en_rectángulo

Esta función comprobará dos áreas rectangulares que usted defina para ver si el rectángulo de origen no está en colisión, está completamente dentro de los límites del rectángulo de destino, o si simplemente se tocan. Si no se tocan, la función devolverá 0, si el origen está completamente dentro del destino devolverá 1, y si simplemente se superponen entonces devolverá 2. La imagen siguiente lo ilustra:

Sintaxis:

rectangle_in_rectangle(sx1, sy1, sx2, sy2, dx1, dy1, dx2, dy2);

Argumento Descripción
sx1 La coordenada x del lado izquierdo del rectángulo de origen.
sy1 La coordenada y del lado superior del rectángulo de origen.
sx2 La coordenada x del lado derecho del rectángulo de origen.
sy2 La coordenada y del lado inferior del rectángulo de origen.
dx1 La coordenada x del lado izquierdo del rectángulo de destino a comprobar.
dy1 La coordenada y del lado superior del rectángulo de destino para comprobarlo.
dx2 La coordenada x del lado derecho del rectángulo de destino a comprobar.
dy2 La coordenada y del lado inferior del rectángulo de destino para comprobarlo.

Devuelve:

Real

Ejemplo:

switch(rectangle_in_rectangle(x - 50, y - 50, x + 50, y + 50, mouse_x - 50, mouse_y - 50, mouse_x + 50, mouse_y + 50))
{
case 0: image_blend = c_yellow; break;
case 1: image_blend = c_red; break;
case 2: image_blend = c_orange; break;
}

El código anterior comprueba la colisión de dos áreas rectangulares, y establecerá la instancia image_blend basándose en el valor de retorno.
