Colisiones

Cuando se planifican movimientos o se deciden ciertas acciones, a menudo es importante ver si hay colisiones con otros objetos en ciertos lugares del mundo del juego, y a menudo elegir la colisión adecuada para el trabajo es la tarea más importante de todas. GameMaker Studio 2 cuenta con una serie de funciones integradas que te ayudarán a tratar las colisiones correctamente y de acuerdo con las necesidades de tu proyecto.

IMPORTANTE! Todas las funciones de colisión trabajan utilizando posiciones enteras, independientemente de los valores de entrada. Cualquier valor no entero se redondeará al entero más cercano.

Las siguientes funciones se ocupan de las distintas formas en las que se puede hacer que una instancia compruebe las colisiones con otra instancia:

  • lugar_vacío
  • lugar_libre
  • lugar_de_reunión
  • posición_vacía
  • posición_reunión
  • cambio_de_posición
  • posición_destrucción
  • lugar_de_instancia
  • lista_de_lugares_de_instancia
  • posición_de_instancia
  • lista_de_posición_de_instancia

Comprobación avanzada de colisiones

Para ello se pueden utilizar las siguientes rutinas (así como otras funciones detalladas en las secciones relativas a los desplazamientos e instancias). Estas funciones de colisión funcionan todas de forma ligeramente diferente, pero mantienen tres argumentos comunes que explicaremos aquí:

  • obj. Aquí es donde especificamos el objeto con el que la función tiene que comprobar la colisión. Puede ser un id de instancia, un tipo de objeto o la palabra clave especial GM all. También hay que tener en cuenta que si el tipo de objeto elegido es el designado padre de otros objetos, entonces todos los hijos de ese objeto se incluirán también en esta comprobación.
  • prec. Puede ser verdadero o falso y se refiere a la precisión de la comprobación de colisiones. Si se establece como verdadero, se calcularán las colisiones precisas (pero sólo si el objeto que se está comprobando tiene marcado "preciso" marcado para su detección de colisiones en las propiedades de su máscara), lo cual es una tarea ligeramente más compleja para la CPU y, por tanto, debe utilizarse con precaución. Si es falso, las colisiones se calcularán basándose en la caja de delimitación del objeto que se va a comprobar para.
  • notme. Es posible que necesites comprobar las colisiones con un objeto que tenga el mismo índice que el objeto que ejecuta el código, así que para evitar que el código encuentre una colisión consigo mismo, puedes establecer esto a true. Si está buscando específicamente la instancia del objeto que ejecuta el código, o no es importante, se puede dejar como falso .

Las funciones básicas devolverán el ID de una instancia que colisiona, o la palabra clave especial noone cuando no hay colisión, mientras que las funciones de lista devolverán el número de instancias que están en colisión y rellenarán una lista DS preconfeccionada con sus IDs. Tenga en cuenta que si hay múltiples colisiones con las áreas definidas por estas funciones e instancias del objeto dado, sólo se devuelve el ID de una instancia, y puede ser cualquiera de las instancias en la colisión.

Existen las siguientes funciones que se ocupan de las colisiones avanzadas.

  • círculo_de_colisión
  • lista_de_círculos_de_colisión
  • colisión_elipse
  • lista_de_colisión_de_elipso
  • línea_de_colisión
  • lista de líneas de colisión
  • punto_de_colisión
  • lista_de_puntos_de_colisión
  • rectángulo_de_colisión
  • lista_de_rectángulo_de_colisión

Comprobación de colisiones sin máscara

Todas las funciones anteriores están relacionadas con las instancias de comprobación de colisiones, y como tales dependen de la máscara de colisión que se define para la instancia. Sin embargo, hay muchos momentos en los que se requiere comprobar las "colisiones" con un punto o un área, especialmente cuando su instancia no tiene un sprite asignado, o cuando está trabajando con el ratón, etc. Por lo tanto, GameMaker Studio 2 también proporciona las siguientes funciones para ayudarle en estas situaciones:

  • punto_en_rectángulo
  • punto_en_triángulo
  • punto_en_círculo
  • rectángulo_en_rectángulo
  • rectángulo_en_triángulo
  • rectángulo_en_círculo

Colisiones físicas

Cuando se utiliza la física incorporada, no se garantiza que las funciones de colisión anteriores funcionen para las instancias habilitadas para la física. Esto se debe principalmente al hecho de que estas instancias ya no utilizan la mayoría de las variables regulares incorporadas (en su lugar, las instancias habilitadas para la física tienen su propio conjunto de variables) y tampoco utilizan la máscara de colisión o el cuadro delimitador, ya que utilizan accesorios. Sin embargo, hay una función especial para probar las colisiones con la física activada que comprueba la superposición de accesorios:

  • physics_test_overlap
