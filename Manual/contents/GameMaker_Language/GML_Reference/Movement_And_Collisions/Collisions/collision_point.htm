punto_de_colisión

Collision_point comprueba el punto especificado por los argumentos x1,y1 para una colisión con cualquier instancia del objeto especificado por el argumento "obj". colisiones precisas, el objeto o la instancia que se está comprobando también debe tener activadas las colisiones precisas para su sprite. Si no es así, la comprobación por defecto se basa en los cuadros delimitadores. La siguiente imagen ilustra cómo funciona esto:

Recuerde que para que las colisiones precisas sean consideradas tanto el sprite del objeto y la función de colisión deben tener marcada la precisión como activada. También hay que tener en cuenta que el valor de retorno de la función puede ser el id de cualquiera de las instancias consideradas en colisión, por lo que si tres instancias se solapan en ese punto, cualquiera uno de sus ids podría ser el valor de retorno de la función.

Tenga en cuenta que en lugar de un índice de objeto puede proporcionar una instancia id para buscar una instancia específica, o las palabras clave de instancia all, o other (dependiendo del evento y del ámbito del código actual).

Sintaxis:

punto_colisión(x, y, obj, prec, notme);

Argumento Descripción
x La coordenada x del punto a comprobar.
y La coordenada y del punto a comprobar.
obj El objeto para comprobar las colisiones de instancia.
prec Si la comprobación se basa en colisiones precisas(verdadero, que es más lento) o en su caja delimitadora en general(falso, más rápido).
notme Si la instancia que llama, si es relevante, debe ser excluida(true) o no(false).

Devuelve:

ID de la instancia o nadie

Ejemplo:

if collision_point(x, y, obj_Cursor, false, true)
{
Puntuación += 10S;
}

Aquí estamos comprobando el punto en la posición del objeto que tiene el código del objeto "obj_Cursor". Si hay uno, entonces añadimos 10 a la variable de puntuación.
