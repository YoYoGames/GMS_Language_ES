Funciones asíncronas

En el contexto de GameMaker Studio 2, una función asíncrona es aquella que envía a un servidor web, o pide la entrada del usuario, o incluso transmite datos, mientras deja que GameMaker Studio 2 siga funcionando sin bloquear el juego que se está ejecutando. Cuando Cuando se recibe la llamada de retorno de información, se dispara un Evento Asíncrono especial que, a su vez, permite que cualquier instancia con un Evento Asíncrono asignado a ella ejecute otras funciones (aunque esto no tiene por qué ser así). Así que, básicamente, es una forma de comunicarse con algún recurso externo (servidor, usuario o dispositivo) sin que el juego o el proyecto se detenga mientras se espera una respuesta.

¿Por qué es esto importante? Bueno, porque permite que GameMaker Studio 2 siga funcionando mientras envía o recibe datos, y esto significa que puedes hacer muchas cosas sin bloquear el bucle del juego, por ejemplo:

  • Transmitir los datos del dispositivo al juego sin que el jugador tenga que esperar a que se cargue.
  • tener eventos de devolución de llamada para hacer cosas sólo cuando se recibe la información correcta
  • comunicar e intercambiar datos con un servidor web

Esto hace que todo funcione de una manera mucho más fluida y discreta. Hay que tener en cuenta que los Eventos Asíncronos se dispararán para todas las instancias que los tengan, por lo que puedes usar una función asíncrona en una instancia, y tener el Evento Asíncrono que se ocupa de la respuesta en otra, o incluso varias más. Puedes ir aquí para más información sobre cómo funciona el evento asíncrono - Eventos asíncronos.
