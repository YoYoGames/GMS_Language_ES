Notificaciones Push

En los dispositivos móviles, normalmente sólo puede haber una aplicación activa en primer plano en todo momento, pero muchos juegos y aplicaciones funcionan en un entorno temporal o interconectado en el que pueden producirse eventos de interés para los usuarios cuando la aplicación no está en primer plano. En estos casos, las notificaciones push locales y remotas pueden permitir que estos juegos notifiquen a sus usuarios cuando se produzcan eventos.

Cuando el sistema operativo entrega una notificación push local o remota y la aplicación de destino no se está ejecutando en primer plano, presenta la notificación al usuario en forma de banner. Si hay una alerta de notificación y el usuario pulsa en ella, entonces se lanza el juego y se llama a un Evento Push Asíncrono donde se puede comprobar la cadena de devolución de llamada que se especifica y resolver el push basándose en su valor. Si la aplicación se está ejecutando en primer plano cuando se entrega la notificación, la aplicación seguirá recibiendo un Evento Push Asíncrono, pero el usuario no recibirá ningún mensaje en pantalla a menos que lo cree usted mismo.

Hay algunas diferencias a tener en cuenta entre las plataformas soportadas:

  • En Android, todas las notificaciones recibidas cuando la aplicación no se está ejecutando/en segundo plano, serán almacenadas, y los datos entregados por el evento async la próxima vez que la aplicación se ejecute y tendrás que habilitar las notificaciones push dentro de las Opciones de Juegode Android para que funcionen.
  • En iOS, los datos de una determinada notificación sólo se entregarán cuando se seleccione la notificación (o si la aplicación se está ejecutando en primer plano cuando se reciba la notificación) y tendrás que habilitar las notificaciones push en las Opciones de Juegode iOS para que funcionen.

Notificaciones locales

Una notificación push local es local para el dispositivo en el que está instalado el juego, y no requiere ningún servidor back-end. Basta con establecer la fecha y la hora de la notificación y ésta se mostrará al usuario si el juego no se está ejecutando en ese momento. Este tipo Este tipo de notificación es útil para establecer "recordatorios" para que el usuario juegue de nuevo, o para ofrecer una recompensa diaria por jugar, etc... y cuando el usuario toca la notificación se inicia el juego y pasa una cadena de datos que puede ser que puede ser analizada por el juego para dar una recompensa o lo que sea.

NOTA: Esta funcionalidad está limitada a los iOS y Android módulos de destino. Para Android deberá tener instalada la extensión de notificaciones push de Google.

Las siguientes funciones están disponibles para las notificaciones locales:

  • push_local_notification
  • push_get_first_local_notification
  • push_get_next_local_notification
  • push_cancel_local_notification

Los mensajes de notificación remota son enviados por un servidor a un servicio proporcionado por la tienda de aplicaciones de la plataforma del dispositivo, y éste reenvía esos mensajes a todos los dispositivos en los que está instalada tu aplicación. Esto es compatible con GameMaker Studio 2 en los módulos de destino de módulos de destino de iOS y Android.

No hay funciones en GameMaker Studio 2 para gestionar las notificaciones remotas, ya que todas deben ser generadas por tu servidor y gestionadas por las respectivas tiendas de aplicaciones. Sin embargo, una vez configurado correctamente, los juegos de GameMaker Studio 2 recibirán estas notificaciones, que luego pueden ser tratados en el evento push asíncrono, como lo haría una notificación local.

Tenga en cuenta que no hay garantía de que se entreguen las notificaciones push remotas, y que la carga de datos permitida es bastante pequeña. Esto varía entre las plataformas, pero iOS es particularmente limitado - el servicio de la manzana sólo entrega la notificación más reciente que debe ser seleccionada por el destinatario para que los datos de la carga útil sean entregados a su evento asíncrono, y estas notificaciones tienen un tamaño máximo de carga útil de 256bytes. Normalmente, una notificación push remota sólo indicaría que hay nuevos datos están disponibles en su servidor, por ejemplo.

NOTA: La implementación del lado del servidor depende totalmente del usuario final, y YoYo Games no proporciona ningún tipo de apoyo para ese lado de las cosas, aparte de la información básica de configuración disponible en su Base de Conocimiento.
