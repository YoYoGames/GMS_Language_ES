mostrar_método_de_tiempo_de_presentación

Esta función se puede utilizar para establecer el método de sincronización que se utilizará para renderizar su juego. El método puede ser una de las siguientes constantes:

Constante Descripción
tm_sleep Utilizar el valor del margen de sueño como método principal de temporización
tm_countvsyncs Utilizar la sincronización vsync como método principal de sincronización (por defecto para todas las plataformas soportadas)

El método de sincronización vsync utiliza el soporte de las plataformas de destino para la sincronización vertical con el fin de proporcionar un ancla para los cálculos de tiempo de renderización de los juegos, mientras que el establecimiento del modo de margen de sueño sólo tratará de asegurar que cada cuadro dura la cantidad correcta cantidad de tiempo (digamos 1/30 o 1/60 de segundo) esperando o durmiendo. En general, la sincronización vsync por defecto dará los resultados más suaves, pero ten en cuenta que cuando se utiliza el método vsync, los valores del margen de espera siguen siendo relevantes, aunque tendrán un impacto reducido y recomendamos mantener el margen de espera. aunque su impacto será menor y recomendamos mantenerlo en su valor por defecto cuando se utilice este método.

Por defecto, en todas las plataformas , excepto PS4, Ubuntu y HTML5, GameMaker Studio 2 utilizará el método de temporización vsync, mientras que en las plataformas no compatibles, sólo se puede utilizar la temporización del margen de sueño.

Si desea establecer el margen de reposo puede hacerlo con la función display_set_sleep_margin() y puede encontrar qué método de temporización se está utilizando actualmente con la función display_get_timing_method().

Sintaxis:

display_set_timing_method(method);

Argumento Descripción
método El método de cronometraje a utilizar (véase la lista de constantes, más arriba)

Devuelve:

N/A

Ejemplo:

si display_get_timing_method() != tm_sleep
{
display_set_timing_method(tm_sleep);
si display_get_sleep_margin() != 20
{
display_set_sleep_margin(20);
}
}

El código anterior comprobará el método de temporización y, si no está establecido como tm_sleep , se establecerá como tal y el margen de suspensión se establecerá en 20.
