object_set_persistent

Con esta función puede establecer la persistencia de un objeto específico. Un objeto persistente se "traslada" de una sala a otra y mantiene sus propiedades en todo momento (para más información sobre la persistencia, véase la sección sobre el Editor de objetos ). Establecer la persistencia del objeto significa que todas las instancias de este objeto que se creen después de cambiar la persistencia se crearán persistentes, mientras que las instancias que ya están en la sala pueden no verse afectadas.

Cuando se crea una instancia con la persistencia activada en una sala, se le asignará una capa o una profundidad (dependiendo de la función que se haya utilizado para crear la instancia del objeto). Cuando se cambie de sala, y si la siguiente no tiene una capa con el mismo nombre o profundidad que la asignada, entonces se creará una nueva capa para la instancia que persiste a través de las habitaciones. Si usted asignó una capa con nombre cuando se creó la instancia, entonces la nueva capa también tendrá el mismo nombre que la capa original, mientras que si asignó una profundidad a la instancia, la nueva capa se llamará "_capa_XXX", donde "XXX" es un valor hexadecimal utilizado para dar a la capa un nombre único. Estas capas se eliminarán de la sala al finalizar la sala. Otra cosa a tener en cuenta sobre las capas y las instancias persistentes es que si ha asignado a la instancia una capa con nombre en la creación, y hay otra capa en las siguientes salas con el mismo nombre, entonces la instancia persistente será asignada a la capa con el mismo nombre, independientemente de la profundidad de la capa. Por último, si una instancia persistente se traslada a una sala con una capa con la misma profundidad en la que se creó la instancia, no se asignará a esta capa, sino que en su lugar se creará una nueva capa a la misma profundidad (siguiendo la convención de nombres explicada anteriormente).

Tenga en cuenta que esta no es una función de instancia. Puede establecer la bandera persistente de instancias individuales utilizando la variable persistente y así tener diez instancias marcadas como persistentes aunque el objeto en cuestión tenga la bandera fijada en false en el IDE, e incluso si cambia la bandera persistente del objeto a true usando esta función, todas las instancias que actualmente están en la sala permanecerán como estaban, y sólo las instancias creadas después de llamar a la función comenzarán con la bandera persistente como verdadera.

Sintaxis:

object_set_persistent(index, pers);

Argumento Descripción
índice El índice del objeto a modificar.
pers Si el objeto debe ser marcado como persistente(true) o no(false).

Devuelve:

N/A

Ejemplo:

if (room == rm_final)
{
object_set_persistent(obj_Player, false);
}
instance_create_layer(32, 32, "Instances", obj_Player);

El código anterior comprueba en qué sala se encuentra actualmente la instancia y si está en la especificada "rm_final" entonces establece la persistencia del objeto "obj_Player" a false y luego crea la instancia.
