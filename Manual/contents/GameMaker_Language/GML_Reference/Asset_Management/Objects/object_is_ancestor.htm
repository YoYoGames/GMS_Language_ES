object_is_ancestor

Esta función se puede utilizar para comprobar un objeto y ver si es un ancestro o no de otro objeto. Por lo tanto, ambos argumentos deben ser índices de objetos y no los ids de instancia, y el primero es siempre el objeto que se quiere quiere averiguar si es hijo del segundo argumento, que siempre es el objeto que quiere comprobar como ancestro (padre).

Sintaxis:

object_is_ancestor(obj, par);

Argumento Descripción
obj El objeto que se está comprobando como hijo.
par El objeto que se está comprobando como antecesor (padre).

Devuelve:

Booleano

Ejemplo:

if object_is_ancestor(object_index, obj_Enemy)
{
instance_destroy();
}

El código anterior comprueba si la instancia que ejecuta el código es hija del objeto "obj_Enemigo", y si es así se destruye.
