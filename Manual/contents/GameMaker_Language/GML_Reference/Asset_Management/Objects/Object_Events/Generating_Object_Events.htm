Generación de eventos de objetos

GameMaker Studio 2 está completamente orientado a los eventos, y todas las acciones ocurren como resultado de ciertos eventos que se activan en un orden específico o por una ocurrencia específica dentro del juego. Hay una serie de eventos diferentes, cada uno con una tarea o tiempo específico y algunos de ellos están incluso divididos en "sub" eventos separados. Normalmente estos eventos son ejecutados por GameMaker Studio 2 cuando detecta un disparador en el juego, o cada tick de juego (como es el caso de los eventos de paso y dibujo), pero también puedes aplicar un evento a una instancia en la sala actual desde un trozo de código.

Existen las siguientes funciones que se encargan de generar eventos:

  • evento_heredado
  • evento_performance
  • evento_realizar_objeto
  • usuario_del_evento

También hay una serie de variables de sólo lectura que se pueden utilizar con las funciones mencionadas anteriormente y que son especialmente útiles para la depuración:

  • acción_de_evento
  • número_de_evento
  • objeto_evento
  • tipo_de_evento

Para obtener más información sobre los eventos, consulte la sección Eventos de los objetos.
