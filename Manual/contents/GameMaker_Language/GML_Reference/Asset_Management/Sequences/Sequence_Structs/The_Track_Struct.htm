La estructura de la pista

La estructura de pista puede utilizarse tanto para laspistas deactivosde nivel superior como para laspistas de parámetros de subpistas , y el comportamiento de la pista se define por dos cosas, su nombre y su tipo . Los structs de pista se crean con la función sequence_track_new() y pueden ser recuperados de los activos de la secuencia utilizando la propiedad activeTracks de la Estructura de la Instancia de la Secuencia o la propiedad tracks de la Estructura del Objeto de la Secuencia.

Las propiedades disponibles para la estructura de la pista son:

  • tipo: Las siguientes constantes se pueden utilizar para definir el tipo de pista que se está creando en la secuencia (estas constantes también se utilizan cuando se generan fotogramas clave y datos de fotogramas clave, que se explican en las secciones siguientes):
Constante Descripción Valor
seqtracktype_graphic Esta es una pista de activos gráficos (sprite). 1
seqtracktype_audio Esta es una pista de activos de audio. 2
seqtracktype_instance Esta es una pista de activos de instancia. 14
seqtracktype_sequence Se trata de una pista de activos secuenciales. 7
seqtracktype_clipmask Esta es una pista de activos de grupo de máscara de clip. 8
seqtracktype_clipmask_mask Esta es una pista de activos de máscara de clip utilizada para generar la máscara de clip. 9
seqtracktype_clipmask_subject Esta es una pista de activos de sprite de máscara de clip que se está enmascarando. 10
seqtracktype_group Este seguimiento de los activos de la carpeta de grupo. 11
seqtracktype_colour Esta es una pista de parámetros de datos de color. 4
seqtracktype_real Se trata de una pista de parámetros de valor numérico real. 3
seqtracktype_message Esta es una pista de mensajes de difusión. 15
seqtracktype_moment Esta es una pista de eventos/momentos. 16
seqtracktype_bool No se utiliza actualmente. 5
seqtracktype_string No se utiliza actualmente. 6
seqtracktype_spriteframes No se utiliza actualmente. 13
seqtracktype_empty No se utiliza actualmente. 12

  • nombre: Cuando se crea una pista de activos, el nombre que se da aquí puede ser cualquier cadena que se requiera para identificar la pista, sin embargo para las pistas de parámetros, es necesario especificar cadenas específicas para decirle a GameMaker Studio 2 qué tipo de pista de parámetros que está creando (estas cadenas sólo son aplicables a los tipos de pista de parámetros seqtracktype_real y seqtracktype_colour ):
    • "Posición": cuando se establece en una pista de tipo seqtracktype_real , indica que el contenido de esta pista debe utilizarse como datos de posición.
    • "Escala": cuando se establece en una pista que es de tipo seqtracktype_real , indica que el contenido de esta pista debe utilizarse como datos de escala.
    • "rotation": cuando se establece en una pista que es de tipo seqtracktype_real , indica que el contenido de esta pista debe utilizarse como datos de rotación.
    • "Mezclar_multiplicar": cuando se establece en una pista que es de tipo seqtracktype_colour , indica que el contenido de esta pista debe utilizarse como datos de multiplicación de color. Los datos de los fotogramas clave de esta pista deben ser una matriz de cuatro valores con el formato [A, R, G, B]. Tenga en cuenta que los valores de cada componente se expresan entre 0 y 1, donde 0 corresponde al valor HEX #00 y 1 corresponde al valor HEX #FF (0 - 255 como se muestra en el selector de color para las pistas de mezcla de imágenes en el Editor de Secuencias).
    • "image_speed"- cuando se establece en una pista que es de tipo seqtracktype_real indica que el contenido de esta pista debe ser utilizado como el valor de la velocidad de la imagen.
    • "image_index"- cuando se establece en una pista que es de tipo seqtracktype_real , indica que el contenido de esta pista debe utilizarse como valor de índice de la imagen.
    • "image_angle"- cuando se establece en una pista de tipo seqtracktype_real , indica que el contenido de esta pista debe utilizarse como valor de ángulo de la imagen.
    • "image_blend"- cuando se establece en una pista de tipo seqtracktype_colour , indica que el contenido de esta pista debe utilizarse como valor de mezcla de la imagen. Los datos de los fotogramas clave de esta pista deben ser una matriz de cuatro valores con el formato [A, R, G, B]. Tenga en cuenta que los valores de cada componente se expresan entre 0 y 1, donde 0 corresponde al valor HEX #00 y 1 corresponde al valor HEX #FF (0 - 255 como se muestra en el selector de color para las pistas de colores múltiples en el Editor de Secuencias).
  • pistas: Esta propiedad permite acceder a la lista de pistas que son hijas de esta pista. Cuando se obtiene esta propiedad se devuelve un array de structs de pistas, y cuando se establece esta propiedad se debe especificar un array de structs de pistas.
  • visible: Indica si esta pista es visible (el valor es verdadero) o no (el valor es falso). Puedes obtener o establecer este valor y si una pista no es visible entonces ninguna de sus pistas hijas se dibujará tampoco.
  • fotogramas clave: Esta propiedad permite acceder a la lista de structs de fotogramas clave de la pista. Cuando se obtiene esta propiedad se devuelve un array de structs de fotogramas clave, y cuando se establece esta propiedad se debe especificar un array de structs de fotogramas clave debe especificarse una matriz de estructuras de fotogramas clave.
