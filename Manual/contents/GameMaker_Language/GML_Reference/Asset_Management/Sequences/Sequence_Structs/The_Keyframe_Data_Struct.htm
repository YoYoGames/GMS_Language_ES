La estructura de datos de los fotogramas clave

Cuando se define una estructura defotogramas clave para una pista, hay que definir también los datos que se asociarán a ella. Esto se compone de diferentes canales, y a cada canal se le puede dar un tipo de datos específico dependiendo del tipo de pista para el que se estén definiendo los datos de los fotogramas clave. Estas propiedades son:

  • canal: Es el canal al que deben aplicarse los datos de los fotogramas clave. Es un valor entero positivo que comienza en 0, y vale la pena señalar que al crear pistas de parámetros para fotogramas clave de "posición" o "escala", es necesario utilizar valores de canal muy específicos. Estos son: el canal 0 es la posición X o la escala X, el canal 1 es la posición Y o la escala Y.
  • spriteIndex: El índice del activo sprite a utilizar para la pista. Esta propiedad sólo está disponible para las pistas del tipo seqtracktype_graphic y se puede obtener o establecer.
  • soundIndex: El índice del activo de audio a utilizar para la pista. Esta propiedad sólo está disponible para las pistas del tipo seqtracktype_audio y se puede obtener o establecer.
  • playbackMode: El modo de reproducción del sonido. Esta propiedad sólo está disponible para las pistas del tipo seqtracktype_audio y se puede obtener o establecer. La propiedad debe ser una de las siguientes constantes:
Constante Descripción Valor
seqaudiokey_loop El sonido se reproducirá en bucle. 0
seqaudiokey_oneshot El sonido sólo se reproducirá una vez y luego se detendrá. 1

  • curva: Esta propiedad requiere una estructura de curva de animación (ver aquí para más información) y sólo está disponible para pistas del tipo seqtracktype_real. Si no se utiliza una estructura de curva el valor de esta propiedad será -1.
  • valor: Esta propiedad es simplemente un valor que se asocia con el canal de datos de fotogramas clave, y sólo está disponible para las pistas del tipo seqtracktype_real cuando no se suministra ninguna estructura de curva se suministra.
  • color: Esta propiedad devuelve (o requiere, si se establece) una matriz para el valor del color del fotograma clave con el formato [A, R, G, B]. Sólo está disponible para las pistas del tipo seqtracktype_colour. Tenga en cuenta que los valores de cada componente deben expresarse entre 0 y 1, donde 0 corresponde al valor HEX #00 y 1 corresponde al valor HEX #FF (0 - 255 como se muestra en el selector de color para las pistas de color en el Editor de secuencias).
  • secuencia: Esta propiedad devolverá (o requiere, si se está estableciendo) una estructura de objeto de secuencia y sólo está disponible para las pistas del tipo seqtracktype_sequence.
  • objectIndex: Esta propiedad devolverá (o requiere, si se está estableciendo) un índice de objeto y sólo está disponible para las pistas del tipo seqtracktype_instance.
  • eventos: Esta propiedad permite acceder a los eventos y mensajes de difusión asociados a la estructura de datos de fotogramas clave. Se puede obtener o establecer esta propiedad, y al obtenerla se devuelve un Cuando se obtiene, se devuelve una matriz de cadenas, y cuando se establece, se debe especificar una matriz de cadenas. Para más información sobre los eventos, consulte la sección Eventos y momentos de la secuencia. Esta propiedad propiedad sólo está disponible para las pistas del tipo seqtracktype_message.
  • evento: Esta propiedad devolverá (o puede establecerse) el método asociado a la estructura de datos del fotograma clave. Si no se ha especificado ningún método o se desea eliminar el método, entonces la propiedad debe ser -1. Esta propiedad sólo está disponible para las pistas del tipo seqtracktype_moment.
