Habitaciones

Las salas se crean en el Editor de Salas de GameMaker Studio 2 y es donde todo sucede en su juego. Usted debe tener al menos una sala en cualquier juego para que se ejecute, pero puede pero puedes necesitar muchas más dependiendo del tipo de juego y de cómo estructures tu proyecto. Esta sección tiene todas las funciones generales de GML y las variables relacionadas con las habitaciones y para obtener información sobre ellas, así como para establecer ciertas propiedades dentro de en ellas.

Las siguientes variables y funciones globales pueden utilizarse para obtener información básica sobre cualquier sala:

  • habitación_primera
  • habitación_última
  • habitación_siguiente
  • habitación_anterior
  • habitación
  • velocidad_de_habitación
  • altura_de_la_habitación
  • ancho_de_habitación
  • habitación_persistente

También puede utilizar las siguientes funciones para obtener el nombre de la sala actual (tal y como se define en el Navegador de Activos), así como para moverse entre salas:

  • room_get_name
  • habitación_goto
  • room_goto_next
  • room_goto_previous
  • room_restart

A continuación, tenemos una serie de funciones que se pueden utilizar para cambiar las salas y su contenido o incluso crear nuevas salas y añadirlas. Hay que tener en cuenta que estas funciones no deben ejecutarse nunca desde la sala que se desea cambiar, sino que ejecutarse primero desde una instancia en otra sala. También hay que tener en cuenta que si se cambia cualquier activo de la sala de esta manera, el cambio es permanente y se mantendrá durante toda la duración del juego, incluso si se llama a la función room_restart() (sin embargo, cerrar y reiniciar el juego restablecerá las propiedades de la sala).

  • habitación_existe
  • room_add
  • habitación_duplicada
  • asignación_de_habitaciones
  • room_instance_add
  • room_instance_clear
  • room_set_height
  • room_set_width
  • room_set_persistent
  • room_set_view_enabled
  • room_set_viewport
  • room_get_viewport
  • room_set_camera
  • room_get_camera

También hay una serie de funciones diferentes relacionadas con la creación y manipulación de los diferentes tipos de elementos que se pueden colocar en las capas de la sala. Estas funciones se enumeran en las secciones apropiadas para cada tipo de elemento enumerado a continuación:

  • Funciones generales de la capa
  • Capas del mapa de azulejos
  • Capas de fondo
  • Capas de sprites
  • Capas de secuencia

Una cosa importante a tener en cuenta al crear las capas de la sala es que usted puede tener varios tipos de elementos diferentes en una capa. Así, por ejemplo, puede tener una capa para los enemigos, y en esa capa tener las instancias de los enemigos, así como ciertos activos de sprites que sólo están relacionados con ese enemigo. O podrías tener un elemento de fondo con un elemento de mapa de azulejos para crear el aspecto de una habitación. Esto no está permitido en el editor de habitaciones, pero cuando se crean cosas dinámicamente se puede hacer esto, por lo que funciones como layer_get_all_elements() existen.
