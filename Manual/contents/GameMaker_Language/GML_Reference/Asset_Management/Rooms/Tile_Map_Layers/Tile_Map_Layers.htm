Capas del mapa de azulejos

El editor de habitaciones de GameMaker Studio 2 te permite añadir azulejos en cualquier habitación utilizando capas. Como su nombre indica, una capa de mapa de azulejos es simplemente una colección de azulejos que se consideran todos que se consideran a la misma profundidad dentro de la habitación, y apilando capas puedes hacer que algunas cosas se dibujen por encima o por debajo de otras. También puedes controlar ciertos aspectos de las capas desde el código, añadiendo o quitando cosas, o cambiando ciertas propiedades de la capa o lo que contiene en tiempo de ejecución. Ten en cuenta que en el editor de salas estás limitado a un mapa de azulejos por capa (el Capa de azulejos), pero cuando se trabaja con ellas en puede tener varios mapas de azulejos asignados a una sola capa, y cada mapa tiene su propio y único ID y propiedades, así:

  • El conjunto de az ulejos es el recurso que se utiliza para crear capas y mapas de azulejos
  • La capa de azulejos es la que se crea en el editor de habitaciones para añadir conjuntos de azulejos
  • Se llama mapa de azulejos a la colección de azulejos que se añaden a una capa, ya sea en el editor de salas o a través de código, como un único elemento en esa capa

Aparte de las funciones mencionadas anteriormente para establecer mapas de azulejos, también hay una serie de funciones que se pueden utilizar para manipular los datos de los azulejos individuales. Un "blob" de datos de un mosaico está compuesto por el índice del mosaico más algunos bits adicionales para mostrar si Si el azulejo ha sido girado, volteado o reflejado. También tienes la posibilidad de añadir tu propia máscara de bits a los datos del mosaico, aunque esto es sólo para usuarios avanzados, ya que estarás enmascarando bits que están reservados para el índice del mosaico - esencialmente hay 19 bits reservados para los índices de las baldosas, pero si sólo utilizas 8 de ellos, entonces te quedan once bits que puedes utilizar para crear una máscara personalizada. Esta máscara se puede utilizar y comprobar en el código para crear colisiones de baldosas personalizadas o lo que sea. Sin embargo, normalmente no necesitarás crear tus propios blobs de datos de baldosas, y en su lugar utilizarás la función tilemap_get() para obtener los datos de las baldosas, luego los manipularías usando las funciones de abajo, y luego lo volverías a fijar con la función tilemap_set().
