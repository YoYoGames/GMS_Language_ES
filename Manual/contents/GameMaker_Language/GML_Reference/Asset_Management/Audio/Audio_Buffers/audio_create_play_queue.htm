audio_create_play_queue

Esta función prepara una cola de búfer para el audio. Se pasa el formato de los datos (actualmente sólo se soportan buffer_u8 o buffer_s16 ), la frecuencia de muestreo (que puede estar entre 1000hz y 48000hz), y los canales que el audio requiere de una de las constantes enumeradas a continuación:

Constante Descripción
audio_mono Audio mono (un solo canal).
audio_stereo Audio estéreo (doble canal).
audio_3D Audio 3D (5.1).


Tenga en cuenta que debe liberar las colas de audio creadas con esta función cuando ya no sean necesarias para evitar fugas de memoria utilizando la función audio_free_play_queue().

Esta función activará un Evento Asíncrono de Reproducción de Audio cuando la cola haya llegado al final, y en este evento se creará un mapa especial DS en la variable async_load con los siguientes pares clave/valor:

  • " queue_id" - la cola que ha terminado de reproducir datos
  • " buffer_id" - el buffer del que ya no se reproduce
  • " queue_shutdown" - se pone a 0 durante la reproducción normal y a 1 cuando se recibe el evento porque audio_free_play_queue() ha sido llamado. Cuando esto se establece en 1, no quiere poner en cola más datos.

Sintaxis:

audio_create_play_queue(queueFormat, queueRate, queueChannels);

Argumento Descripción
queueFormat El formato de los datos del buffer a utilizar(buffer_u8 o buffer_s16).
queueRate La frecuencia de muestreo de los datos en la cola del buffer.
queueChannels Los canales a utilizar a partir de una de las constantes indicadas a continuación.

Devuelve:

Real (ID del búfer)

Ejemplo:

audio_queue = audio_create_play_queue(buffer_s16, 11052, audio_mono);

El código anterior crea una nueva cola de audio y asigna su ID a una variable para futuras referencias.
