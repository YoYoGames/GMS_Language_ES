audio_listener_set_velocity

Esta función puede utilizarse para dar al oyente dado efectos Doppler y simular el movimiento del audio basado en el vector que se resuelve a partir de las posiciones relativas x, y y z dadas. El índice del oyente por defecto es 0, pero se puede utilizar la función audio_get_listener_info() para obtener los diferentes índices disponibles para la plataforma de destino.

Si el oyente dado no se va a mover nunca, o el movimiento no es un movimiento constante, normalmente no necesitarás establecer estos valores, pero, por ejemplo, si estás haciendo un juego de desplazamiento donde el jugador tiene un movimiento constante de abajo a arriba y los enemigos un movimiento constante de arriba a abajo, deberías establecer las velocidades del oyente y del emisor (para los emisores usarías audio_emitter_velocity()) a los vectores adecuados para simular el efecto Doppler correcto a medida que se mueven más allá de la instancia del jugador.

La imagen siguiente muestra cómo se podría configurar para el juego de ejemplo dado anteriormente:

Sintaxis:

audio_listener_set_velocity(index, x, y, z);

Argumento Descripción
índice El oyente para el que se obtienen los datos (por defecto 0).
x La velocidad x del oyente (por defecto 0).
y La velocidad y del oyente (por defecto 0).
z La velocidad z del oyente (por defecto 0).

Devuelve:

N/A

Ejemplo:

var num = audio_get_listener_count();
for(var i = 0; i < num; ++i;)
{
var info = audio_get_listener_info(i);
var data = audio_listener_get_data(info[? "index"]);
if data[? "vx"] != 0
{
audio_listener_set_velocity(info[? "index"], 0, 0, 0);
}
ds_map_destroy(info);
ds_map_destroy(data);
}

El código anterior comprueba el número de oyentes disponibles, luego hace un bucle a través de ellos y si su velocidad x no es 0, establece sus valores de velocidad a 0, 0, 0.
