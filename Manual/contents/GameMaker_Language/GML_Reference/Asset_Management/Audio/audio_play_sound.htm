audio_play_sound

Con esta función puedes reproducir cualquier recurso de sonido en tu juego. Usted proporciona el índice del sonido y le asigna una prioridad, que luego se utiliza para determinar cómo se tratan los sonidos cuando el número de sonidos que se reproducen supera el límite establecido por la función audio_channel_num(). Los sonidos de menor prioridad se detendrán en favor de los sonidos de mayor prioridad, y el valor de la prioridad puede ser cualquier número real (el valor real es arbitrario, y puede ser de 0 a 1 o de 0 a 100, ya que GameMaker Studio 2 prioridad de la misma manera). Tenga en cuenta que cuando se trata de la prioridad, cuanto mayor sea el número, mayor será la prioridad, de tal manera que un sonido con prioridad 100 será favorecido sobre un sonido con prioridad 1. El argumento final es para hacer el bucle de sonido y establecerlo como verdadero hará que el sonido se repita hasta que se detenga y establecerlo como falso reproducirá el sonido una vez una sola vez.

Esta función también devolverá un número de índice único para el sonido que se está reproduciendo, el cual puede ser almacenado en una variable para que luego puedas pausarlo o detenerlo con las funciones apropiadas. Esto significa que si tiene varias instancias del mismo sonido que se está reproduciendo en un momento dado, se puede seleccionar sólo una instancia de ese sonido para tratarla con las funciones de audio.

Sintaxis:

audio_play_sound(index, priority, loop);

Argumento Descripción
índice El índice del sonido a reproducir.
prioridad Establece la prioridad del canal para el sonido.
bucle Establece que el sonido se reproduzca en bucle o no.

Devuelve:

Índice (ID de sonido)

Ejemplo:

si salud <= 0
{
vidas -= 1;
audio_play_sound(snd_PlayerDead, 10, false);
}

El código anterior comprueba la variable global "salud" y si es menor o igual a 0, eliminará 1 de la variable global "vidas" y reproducirá un sonido.
