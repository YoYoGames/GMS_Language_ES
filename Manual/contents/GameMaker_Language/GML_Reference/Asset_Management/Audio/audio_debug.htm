audio_debug

Esta función puede utilizarse para mostrar información de depuración sobre el sistema de audio, con true para activarla y false para desactivarla. Cuando se está depurando el audio de esta manera, la superposición mostrará la siguiente información:

  • Handle - El handle de la fuente OpenAL (único para cada sonido reproducido)
  • Manejador de datos - El manejador del búfer de OpenAL (muestra el mismo valor para múltiples sonidos con el mismo manejador de origen)
  • Buffers procesados: cuántos buffers se han reproducido
  • Bufferes en cola: cuántos bufferes se han puesto en cola (y aún están esperando para reproducirse)
  • Ganancia - El valor de la ganancia para el audio (0 - 65536)
  • Play Offset - la posición de la muestra (posición actual de reproducción) en el buffer actual
  • Barras de búferes en cola: un pequeño diagrama de caja de los búferes procesados/en cola con el desplazamiento de la reproducción (posición actual) dibujado como una línea azul

Para los grupos de audio sincronizados, también tiene la función audio_sync_group_debug().

NOTA: Esta función no funciona en la plataforma de destino HTML5.

Sintaxis:

audio_debug(enable);

Argumento Descripción
activar Habilitar (establecer como verdadero) o deshabilitar(falso) la depuración de audio.

Devuelve:

N/A

Ejemplo:

if (debug_mode)
{
audio_debug(true);
}
si no
{
audio_debug(false);
}

El código anterior activará o desactivará la superposición de la depuración de audio dependiendo de si el juego se está ejecutando en modo de depuración o no.
