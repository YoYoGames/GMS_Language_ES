audio_play_sound_on

Con esta función puedes reproducir cualquier recurso sonoro a través de un emisor, con cualquier cambio en la ganancia, posición, tono o velocidad del emisor que afecte a cómo el usuario oye el sonido final que se reproduce. Se proporciona el índice del emisor a utilizar, el índice del sonido del sonido que se va a reproducir y, a continuación, se especifica si el sonido debe reproducirse en bucle o no. Por último, puede asignar una prioridad, que se utiliza para determinar cómo se tratan los sonidos cuando el número de sonidos que se reproducen supera el límite establecido por la función audio_channel_num(). Los sonidos de menor prioridad se detendrán en favor de los sonidos de mayor prioridad, y el valor de la prioridad puede ser cualquier número real (el valor real es arbitrario, y puede ser de 0 a 1 o de 0 a 100, ya que GameMaker Studio 2 prioridad de la misma manera). Tenga en cuenta que cuando se trata de la prioridad, cuanto mayor sea el número, mayor será la prioridad, de tal manera que un sonido con prioridad 100 será favorecido sobre un sonido con prioridad 1.

Esta función también devolverá un número de índice único para el sonido que se está reproduciendo, el cual puede ser almacenado en una variable para que luego puedas pausarlo o detenerlo con las funciones apropiadas. Esto significa que si tiene varias instancias del mismo sonido que se está reproduciendo en un momento dado, se puede seleccionar sólo una instancia de ese sonido para tratarla con las funciones de audio.

Sintaxis:

audio_play_sound_on(emisor, sonido, bucle, prioridad);

Argumento Descripción
emisor El índice del emisor a utilizar.
sonido El índice del sonido a utilizar.
bucle Marca el sonido como en bucle o no.
prioridad Establece la prioridad del canal para el sonido.

Devuelve:

Índice

Ejemplo:

si global.SFX
{
audio_play_sound_on(s_emit, snd_Explode, false, 1);
}

El código anterior comprueba la variable global "SFX" y si devuelve true entonces el sonido indexado en la variable "snd_Explode" se reproducirá a través del emisor indexado en la variable "s_emit" con una prioridad baja y sin bucle.
