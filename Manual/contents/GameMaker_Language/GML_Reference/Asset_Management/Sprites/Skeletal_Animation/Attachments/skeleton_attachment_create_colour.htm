skeleton_attachment_create_colour

Cuando creas tu sprite de animación esquelética, puedes asignar ranuras de adj untos y adjuntos para ir en ellas. Estas son simplemente imágenes (sprites) que están separadas de la animación, pero cuando se adjuntan se mueven junto con el hueso al que están hueso al que se adjuntan. Normalmente, usted asignaría estos accesorios en su programa de animación (Spine), pero con esta función puede crear su propio en tiempo de ejecución utilizando un activo de sprite de su juego. la función requiere que usted da el archivo adjunto un nombre (como una cadena) y luego establecer los valores sprite_index y image_index a utilizar, así como el origen x e y (que puede (que puede ser diferente al definido por el sprite en las propiedades del sprite), y también puede establecer cualquier transformación que desee aplicar a la imagen cuando se adjunta, así como el color que se mezcla con la imagen y su valor alfa (transparencia). de la imagen. Si el archivo adjunto se ha creado con éxito, la función devolverá 1 y si no lo ha hecho (has suministrado un índice de sprite no válido, o el sprite base no es un sprite de Spine) entonces devolverá -1.

Cabe destacar que el sprite de instancia puede tener un color de mezcla y un ajuste alfa (mezcla_de_imagen y imagen_alfa), al igual que la ranura de fijación (véase la función skeleton_slot_colour_set()), por lo que el color y el alfa finales que tendrá el sprite adjunto asignado a la ranura serán un compuesto de todos estos valores.

Sintaxis:

skeleton_attachment_create_colour(nombre, sprite, ind, xorigin, yorigin, xscale, yscale, rot, color, alpha);

Argumento Descripción
nombre El nombre (en forma de cadena) del archivo adjunto a crear.
sprite El sprite_index para obtener la imagen adjunta.
ind El image_index para obtener la imagen adjunta.
xorigin El origen x de la imagen utilizada.
yorigen El origen y de la imagen utilizada.
xscale La escala horizontal de la imagen, como multiplicador: 1 = escala normal, 0,5 es la mitad, etc.
yscale La escala vertical de la imagen, como multiplicador: 1 = escala normal, 0,5 es la mitad, etc.
pudrirse La rotación de la imagen: 0 = normal, 90 = girada 90° en sentido contrario a las agujas del reloj, etc.
Color El valor del color a utilizar (una constante, un entero o un valor hexadecimal).
Alpha El valor alfa a utilizar (de 0 a 1).

Devuelve:

1 si tiene éxito, -1 si no

Ejemplo:

skeleton_attachment_create_colour("espada", spr_Weapons, 0, 0, 80, 1, 1, 90, c_red, 1);
skeleton_attachment_create_colour("cuchillo", spr_Weapons, 1, 0, 45, 1, 1, 90, c_yellow, 1);
skeleton_attachment_create_colour("ballesta", spr_Weapons, 0, 10, 30, 1, 1, 0, c_white, 0.5);
skeleton_attachment_set("slot_leftHand", choose("sword", "knife", "crossbow"));

El código anterior comprobaría el nombre de adjunto asignado actualmente para la ranura llamada "slot_leftHand" y si se devuelve una cadena vacía, se adjunta un nuevo sprite.
