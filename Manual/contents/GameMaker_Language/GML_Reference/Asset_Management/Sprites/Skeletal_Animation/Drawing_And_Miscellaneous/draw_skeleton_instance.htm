dibujar_esqueleto_instancia

Esta función es sólo para usar con instancias que tienen sprites asignados creados con el programa de animación esquelética Spine. Aunque se pueden dibujar estos sprites utilizando las funciones normales de dibujo (como draw_self()), hay ocasiones en las que puede querer dibujar un solo fotograma o mostrar un cambio en la textura de la piel sin cambiar realmente las propiedades de los sprites de una instancia. En estos casos puedes dibujar el sprite que ha sido asignado a la instancia con esta función, en la que se da el ID de la instancia y el nombre de la animación de la que se va a obtener el fotograma (una cadena, tal y como se ha establecido en el programa utilizado para crear el sprite). A continuación se da el nombre del skin a utilizar (de nuevo, tal y como se estableció cuando se hizo el sprite), y un valor de valor de fotograma. El valor del marco es para decirle a GameMaker Studio 2 qué parte de la animación mostrar y es un valor entre 0 y el número_de_imagen. El resto de los argumentos son los mismos que se utilizan para el dibujo normal de sprites y cambiarán la escala, el ángulo, el color de fusión y el alfa del fotograma final de la animación que se está dibujando.

ADVERTENCIA Debido a la forma en que las animaciones de esqueletos se interpolan entre varios fotogramas "clave", hay una sobrecarga adicional asociada a esta llamada y, por lo tanto, se recomienda evitar dibujar sprites basados en esqueletos utilizando este método a menos que sea absolutamente necesario.

Sintaxis:

draw_skeleton_instance(instance, animname, skinname, frame, x, y, xscale, yscale, rot, colour, alpha);

Argumento Descripción
instancia El índice de la instancia para dibujar el sprite.
animname El nombre de la animación de la que se va a obtener el fotograma (una cadena).
nombre de la piel El nombre del skin a utilizar (una cadena).
marco El cuadro de animación a dibujar (de 0 a número_de_imagen - 1).
x La coordenada x de donde dibujar el sprite.
y La coordenada y de dónde dibujar el sprite.
xscale La escala horizontal del sprite, como multiplicador: 1 = escala normal, 0,5 es la mitad, etc.
yscale La escala vertical del sprite, como multiplicador: 1 = escala normal, 0,5 es la mitad, etc.
pudrirse La rotación del sprite. 0=normal, 90=girado 90 grados en sentido contrario a las agujas del reloj, etc.
color El color con el que se mezclará el sprite.
alfa El alfa del sprite (de 0 a 1 donde 0 es transparente y 1 opaco).

Devuelve:

N/A

Ejemplo:

draw_skeleton_instance(obj_Player, "jump", "skin1", 0, x, y, 1, 1, 0, c_white, 1);

El código anterior dibujará el primer fotograma del sprite de animación esquelético asignado a la instancia dada, utilizando el conjunto de animación "salto" y el skin definido como "skin1".
