dibujar_esqueleto

Esta función es sólo para usar con sprites que han sido creados usando un programa de animación esquelética como Spine. Aunque puedes dibujar estos sprites usando las funciones normales de dibujo (como draw_self()), hay ocasiones en las que puede querer dibujar un solo fotograma o mostrar un cambio de textura de la piel sin cambiar realmente las propiedades del sprite. En estos casos puedes dibujar el sprite con esta función, donde das el índice del sprite y luego el nombre de la animación para obtener el fotograma (una cadena, como se establece en el programa utilizado para hacer el sprite). A continuación, se da el nombre de la piel a utilizar (de nuevo, como se establece cuando el sprite se hizo), y un valor de marco. El valor del marco es para decirle a GameMaker Studio 2 qué parte de la animación mostrar y es un valor entre 0 y el número_de_imagen. El resto de los argumentos son los mismos que los utilizados para el dibujo normal de sprites y cambiarán la escala, el ángulo, el color de fusión y el alfa del fotograma final de la animación que se está dibujando.

ADVERTENCIA Debido a la forma en que las animaciones de esqueletos se interpolan entre varios fotogramas "clave", hay una sobrecarga adicional asociada a esta llamada y, por lo tanto, se recomienda evitar dibujar sprites basados en esqueletos utilizando este método a menos que sea absolutamente necesario.

Sintaxis:

draw_skeleton(sprite, animname, skinname, frame, x, y, xscale, yscale, rot, color, alpha);

Argumento Descripción
sprite El índice del sprite a dibujar.
animname El nombre de la animación de la que se va a obtener el fotograma (una cadena).
nombre de la piel El nombre del skin a utilizar (una cadena).
marco El cuadro de animación a dibujar (de 0 a número_de_imagen - 1).
x La coordenada x de donde dibujar el sprite.
y La coordenada y de dónde dibujar el sprite.
xscale La escala horizontal del sprite, como multiplicador: 1 = escala normal, 0,5 es la mitad, etc.
yscale La escala vertical del sprite, como un multiplicador: 1 = escala normal, 0,5 es la mitad, etc.
pudrirse La rotación del sprite. 0=normal, 90=girado 90 grados en sentido contrario a las agujas del reloj, etc.
color El color con el que se mezclará el sprite.
alfa El alfa del sprite (de 0 a 1 donde 0 es transparente y 1 opaco).

Devuelve:

N/A

Ejemplo:

draw_skeleton(spr_Jump, "jump", "skin1", 0, x, y, image_xscale, image_yscale, image_angle, c_white, 0.5);

El código anterior dibujará el primer fotograma del sprite de animación esquelético dado, utilizando el conjunto de animación "salto", la piel definida como "skin1" y en la misma escala y posición que definen las propiedades de la instancia, pero con un alfa de 0,5.
