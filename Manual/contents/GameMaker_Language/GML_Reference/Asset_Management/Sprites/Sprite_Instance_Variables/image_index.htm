índice_de_imagen

Un sprite se compone de una o más sub-imágenes que pueden hacer que el sprite aparezca animado al cambiar de una a otra, o pueden ser cambiadas en código para dar diferentes "estados", muy parecido a lo que tiene un botón en las ventanas. Si el sprite está animado, puedes obtener el fotograma actual de la animación comprobando la variable image_index, o si quieres cambiar el estado de un sprite estático, puedes seleccionar una nueva sub-imagen ajustando esta variable a la sub-imagen deseada deseada del sprite. Ten en cuenta que para que los cambios en esta variable sean visibles, la instancia debe no tener ningún evento de dibujo (y así GameMaker Studio 2 dibujará por defecto el sprite) o ser dibujado usando una de las funciones de dibujo como dibujar_self() o draw_sprite_ext() (suministrando el image_index en el argumento apropiado).

Tenga en cuenta que al establecer esta variable directamente en un fotograma no se activarán los mensajes de difusión que puedan estar presentes en ese fotograma.

También hay que tener en cuenta que mientras se usan los sprites de animación esquelética, todavía se pueden obtener y establecer los valores de image_index - ver la función skeleton_animation_get_duration() para ejemplos de cómo hacerlo.

Sintaxis:

image_index;

Devuelve:

Real (valor de punto flotante de precisión única)

Ejemplo:

if (image_speed > 0)
{
si (image_index >= image_number - 1) instance_destroy();
}

El código anterior comprueba si el sprite está animando, y si lo está entonces comprueba si el image_index actual está en el último fotograma y en ese caso destruye la instancia.
