sprite_add_from_surface

Esta función funciona exactamente igual que sprite_create_from_surface() sólo que en lugar de crear un nuevo sprite a partir de la zona de la superficie indexada que se seleccione, se añade el área definida de la superficie como una nueva subimagen a un sprite previamente creado (esto significa que no se puede añadir directamente a un sprite del árbol de recursos, sino sólo a uno previamente creado a partir de una superficie, o a uno que haya sido duplicado a partir de un sprite de recursos utilizando sprite_duplicate()).

Hay que tener en cuenta que la memoria utilizada al añadir un sprite de esta manera será mayor de lo que se puede esperar. Esto se debe a que GameMaker Studio 2 almacenará el sprite como una página de textura y también se almacenará en la memoria de la GPU, por lo que la memoria total será mayor de lo que cabría esperar para un archivo de imagen del mismo sprite.

Sintaxis:

sprite_add_from_surface(index, surface, x, y, w, h, removeback, smooth);

Argumento Descripción
índice El índice del sprite al que se va a añadir la nueva imagen.
superficie El índice de la superficie de la que se obtiene la imagen.
x La posición x desde la que copiar.
y La posición y desde la que se va a copiar.
w La anchura del área a copiar (desde la posición x).
h La altura del área a copiar (desde la posición y).
quitar la espalda Indica si se deben hacer transparentes todos los píxeles con el color de fondo (píxel inferior izquierdo).
suave Indica si se deben suavizar los bordes.

Devuelve:

N/A

Ejemplo:

spr_custom = sprite_create_from_surface(surf, 0, 0, 32, 32, true, true, 16, 16);
var i;
for (i = 1; i < 8; i +=1)
{
sprite_add_from_surface(spr_Custom, surf, i, 0, 32, 32, true, true, 16, 16);
}

El código anterior crea un sprite a partir de la superficie indexada en la variable "surf", asignando su índice a la variable "spr_Custom", y luego utiliza un bucle for para moverse por la superficie y capturar varias secciones que se añaden al sprite como subimágenes.
