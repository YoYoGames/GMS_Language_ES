sprite_collision_mask

Con esta función puedes establecer las propiedades de la máscara de colisión que debe tener un sprite. Si selecciona el modo automático (0) o imagen completa (1) como modo de cuadro delimitador, entonces los valores individuales del cuadro delimitador pueden establecerse en 0. Sin embargo, para una máscara definida por el usuario (2) tendrá que establecer estos valores. Sin embargo, para una máscara definida por el usuario (2) tendrá que establecer estos valores. Los diferentes valores de los cuadros delimitadores son siempre relativos a la esquina superior izquierda del sprite (independientemente del origen x e y) que se considera la posición (0, 0).

La configuración del tipo de máscara establece la forma general de la propia máscara, pero tenga en cuenta que cualquier cosa que no sea una máscara rectangular requerirá más potencia de procesamiento a la hora de resolver las colisiones, con la consiguiente disminución del rendimiento. En general, sólo debería utilizar tipos de máscara distintos a la rectangular cuando sea absolutamente necesario.

NOTA: Esta función no permite el tipo de máscara de colisión de rectángulo girado.

El tipo de máscara que se puede establecer será una de las siguientes constantes:

Constante Descripción
bboxkind_rectangular Una forma de máscara de colisión rectangular (no giratoria)
bboxkind_ellipse Una forma de máscara de colisión elíptica
bboxkind_diamante Una forma de máscara de colisión en forma de diamante
bboxkind_precise Una máscara de colisión precisa, en la que la máscara se ajustará a los píxeles no transparentes del sprite, basándose en el valor de tolerancia dado (véase más abajo))


Por último, la tolerancia se utiliza para definir la precisión de la máscara (cuando se utiliza con una máscara de "imagen completa", esto no tendrá ningún efecto), con una tolerancia de 0 significa que la máscara seguirá cada uno de los píxeles que tienen una transparencia superior a 0, mientras que otros valores desplazarán el perímetro de la máscara de colisión en función de la transparencia de los píxeles.

NOTA: Esta función sólo es útil para sprites de mapa de bits y no funcionará con sprites SWF o JSON (Spine).

NOTA: Esta función sólo funcionará en sprites añadidos o duplicados y no directamente en recursos pre-fabricados. Puedes duplicar sprites utilizando la función sprite_duplicate().

Sintaxis:

sprite_collision_mask(ind, sepmasks, bboxmode, bbleft, bbtop, bbright, bbbottom, kind, tolerance);

Argumento Descripción
ind El índice del sprite para establecer la caja delimitadora.
sepmasks Si se crean máscaras de colisión para cada sub-imagen del sprite(true), o una máscara para todas(false).
bboxmode Qué tipo de cuadro delimitador utilizar. 0 = automático, 1 = imagen completa, 2 = definido por el usuario.
bbleft La posición máxima a la izquierda del cuadro delimitador.
bbtop La posición superior máxima del cuadro delimitador.
bbright La posición máxima a la derecha del cuadro delimitador.
bbbottom La posición inferior máxima del cuadro delimitador.
amable El tipo de máscara, una constante (véase la tabla en la descripción).
tolerancia Indica la tolerancia en el valor de la transparencia (0= sin tolerancia, 255= tolerancia total).

Devuelve

N/A

Ejemplo:

spr = sprite_add("player_5.png", 16, true, true, 0, 0);
sprite_collision_mask(spr, true, 1, 0, 0, 0, 0, 0, 0);

El código anterior carga un sprite desde una fuente externa y almacena el nuevo índice en la variable "spr". A continuación, el código configura el nuevo sprite para que tenga una máscara de colisión precisa para cada una de sus subimágenes.
