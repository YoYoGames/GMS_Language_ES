persistente

Esta variable puede leerse para saber si la instancia está marcada como persistente o no, o puede utilizarse para establecer la persistencia como verdadera (persistente) o falsa (no persistente) para la instancia. Una instancia persistente es aquella que Una instancia persistente es aquella que se "traslada" de una sala a otra, lo que significa (por ejemplo) que sólo tiene que crearse una vez al comienzo del juego y estará presente en todas las salas posteriores. Hay que tener cuidado con la persistencia, ya que es fácil perder la pista de las instancias persistentes lo que puede llevar a problemas más adelante en el desarrollo del juego.

Cuando se crea una instancia con la persistencia activada en una sala, esta variable se establecerá como verdadera y se le asignará una capa o una profundidad (dependiendo de la función que se haya utilizado para crear la instancia del objeto). Cuando se cambie la sala y si la siguiente habitación no tiene una capa con el mismo nombre o profundidad que la asignada, entonces se creará una nueva capa para la instancia que persiste a través de las habitaciones. Si se asignó una capa con nombre cuando se creó la instancia, entonces la nueva capa también tendrá el mismo nombre que la capa original, mientras que si asignó una profundidad a la instancia entonces la nueva capa se llamará " _layer_XXX", donde "XXX" es un valor hexadecimal utilizado para dar a la capa un nombre único. nombre único. Estas capas se eliminarán de la sala al finalizar la misma. Otra cosa a tener en cuenta sobre las capas y las instancias persistentes es que si ha asignado a la instancia una capa con nombre en la creación, y hay otra capa en las siguientes salas con el mismo nombre, la instancia persistente se asignará a la capa con el mismo nombre, independientemente de la profundidad de la capa. Por último, si una instancia persistente se traslada a una sala con una capa con la misma profundidad en la que se creó la instancia, no se asignará a esta capa, sino que se asignará a la capa con el mismo nombre. no se asignará a esta capa, sino que se creará una nueva capa a la misma profundidad (siguiendo la convención de nombres explicada anteriormente).

Un objeto persistente seguirá teniendo sus eventos de Inicio de Juego, Fin de Juego, Inicio de Sala y Fin de Sala, sin embargo si reinicias el juego (con, por ejemplo, la función game_restart() ) todos los objetos persistentes serán serán eliminados y sólo existirán cuando sean creados de nuevo por el juego. Ten en cuenta también que si desactivas un objeto persistente, ya no pasará de una sala a otra a no ser que se reactive antes de que se dispare el evento Room End.
