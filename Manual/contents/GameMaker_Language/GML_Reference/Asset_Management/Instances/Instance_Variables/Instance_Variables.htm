Variables de instancia

Cuando creas un nuevo objeto, éste vendrá con ciertas variables ya inicializadas con valores por defecto. A estas variables las llamamos variables de instancia, ya que se crearán para cada instancia que coloques del objeto en tus salas de juego, y una vez creada una instancia, los valores de estas variables son únicos para esa instancia y sólo para esa instancia. Algunas de estas variables las usarás mucho, como x e y, mientras que otras se requieren menos dependiendo de lo que quieras pero, en general, son muy útiles y, siempre que sea posible, deberías utilizarlas en lugar de crear tus propias variables personalizadas.

La mayoría de las variables de instancia pueden establecerse así como leerse, lo que permite cambiar las propiedades y el comportamiento de una instancia simplemente ajustando el valor de una determinada variable - se puede evitar que una instancia se dibuje, por ejemplo, simplemente estableciendo la variable incorporada visible a false. A continuación puedes encontrar las diferentes variables que se inicializan para todas las instancias de todos los objetos de tu juego.

IMPORTANTE! Si una instancia está en una secuencia, algunas de estas variables - como image_xscale / image_yscale / image_angle / x / y - se sobrescribirán cuando la secuencia actualiza cada paso después de empezar a reproducirse.

Estas variables se ocupan de las propiedades generales de la instancia:

  • id
  • visible
  • sólido
  • persistente
  • profundidad
  • capa
  • alarma

Estas variables se ocupan de la posición y el movimiento de la instancia:

  • dirección
  • fricción
  • gravedad
  • dirección_de_gravedad
  • hspeed
  • vspeed
  • velocidad
  • xstart
  • ystart
  • x
  • y
  • xprevio
  • yprevio

Propiedades de los objetos

La siguiente variable contiene el ID de índice único del objeto del que se creó la instancia:

  • objeto_índice

Estas variables están relacionadas con el sprite asignado a la instancia y se pueden utilizar para cambiar lo que se dibuja y cómo:

  • sprite_index
  • ancho_de_sprites
  • altura_de_sprites
  • sprite_xoffset
  • sprite_yoffset
  • imagen_alfa
  • ángulo_de_imagen
  • mezcla_de_imágenes
  • índice_de_imagen
  • número_de_imagen
  • velocidad_de_imagen
  • imagen_escala_x
  • imagen_escala

Estas variables se ocupan de la máscara de colisión:

  • índice_de_máscara
  • bbox_bottom
  • bbox_left
  • bbox_right
  • bbox_top

Estas variables están relacionadas con las rutas y con la forma en que la instancia interactúa con una, si se le asigna:

  • índice_de_camino
  • posición_de_camino
  • posición_de_camino_anterior
  • velocidad_de_camino
  • escala_de_trayectoria
  • orientación_de_la_vía
  • ruta_de_actuación

Estas variables sirven para configurar una instancia para utilizar una línea de tiempo:

  • índice_de_línea_de_tiempo
  • cronograma_de_ejecución
  • velocidad de la línea de tiempo
  • posición_temporal
  • bucle_de_tiempo

Secuencias

Las siguientes variables almacenan información relativa a la Secuencia que puede estar controlando la instancia en un momento dado:

  • secuencia_in
  • secuencia_instancia

Hay un gran número de variables incorporadas para su uso con las funciones de física de GameMaker Studio 2, por lo que para mantener las cosas más claras, se pueden encontrar en la sección del manual que cubre todo lo relacionado con la simulación de la física:

  • Física
