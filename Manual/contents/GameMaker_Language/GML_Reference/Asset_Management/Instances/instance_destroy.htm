instance_destroy

Se llama a esta función siempre que se desee "destruir" una instancia, normalmente activando un Evento de Destrucción y también un Evento de Limpieza . Esto la eliminará de la sala hasta que ésta se reinicie (a menos que la sala sea persistente). Llamar a la función sin argumentos simplemente destruirá la instancia que está actualmente en el ámbito y ejecutando el código, pero puedes proporcionar un argumento opcional "id" opcional y apuntar a una instancia específica usando el valor de la instancia id, o puede apuntar a todas las instancias de un objeto en particular usando un argumento object_index. Por ejemplo:

instance_destroy(other); // destruir la instancia "other" en un evento de colisión
instance_destroy(obj_Bullet); // destruir TODAS las instancias del objeto "obj_Bullet"

El segundo indicador opcional permite "desactivar" el evento de destrucción para la instancia que se está destruyendo. Por defecto, el Evento de Destrucción siempre se llevará a cabo, pero si establece esta bandera en falso, entonces puede hacer que la instancia se destruya a sí misma y omitir la realización de ese evento.

IMPORTANTE! Esto omitirá el evento de destrucción, pero el evento de limpieza seguirá siendo llamado.

Cabe destacar que cuando se destruye una instancia, su evento de destrucción se llama inmediatamente después del código o acción que llama a la función de destrucción. Además, aunque se ejecute el evento de destrucción, la instancia no se elimina inmediatamente del juego, y continuará realizando el código contenido en el evento actual. Sólo cuando el evento actual termine será eliminado del juego.

Así que, si tienes, por ejemplo, este código:

f hp <= 0 instance_destroy();
score += 10;

La variable "score" se incrementará aunque la función instance_destroy haya sido llamada, y la instancia será finalmente eliminada de tu juego al final del evento. Ten cuidado con esto, ya que si has (por ejemplo) creado un recurso dinámico para la instancia, como una estructura de datos, y luego has destruido ese recurso en el evento de destrucción, si hay alguna referencia a él después de que la función o acción de destrucción se haya realizado, entonces obtendrás un error de "recurso desconocido", ya que el evento de destrucción desconocido", ya que el evento de destrucción lo ha eliminado del juego.

Sintaxis:

instance_destroy([id, execute_event_flag]);

Argumento Descripción
id El ID de la instancia o object_index a destruir (opcional, por defecto es la instancia que llama)
ejecutar_evento_bandera Establecer en true o false para realizar el evento de destrucción o no (opcional, por defecto es true)

Devuelve:

N/A

Ejemplo:

if bbox_right < 0 || bbox_left > room_width || bbox_bottom < 0 | bbox_top > room_height
{
instance_destroy(id, false);
}

El código anterior destruirá la instancia que ejecuta el código sin llamar al evento Destroy si la instancia está fuera de los límites de la sala.
