Teclas y teclados virtuales

Al desarrollar juegos para dispositivos de pantalla táctil, a menudo es importante obtener la entrada del teclado del usuario. Cuando se utiliza GameMaker Studio 2, esto se puede hacer de dos maneras:

  • Crea tus propias teclas virtuales en pantalla para obtener entradas
  • Abrir el teclado virtual del sistema operativo y obtener información de él

Lo que utilices dependerá mucho del juego y de lo que necesites, pero en general utilizarás las teclas virtuales cuando quieras tener botones en pantalla para el control del jugador o para los menús, mientras que utilizarás el teclado virtual cuando necesites la entrada del jugador, como obtener un nombre o establecer un valor.

Para que tus juegos funcionen correctamente en dispositivos móviles con pantalla táctil, es posible que tengas que asignar zonas de la pantalla para que respondan a determinadas cosas cuando las toque el usuario. Esto se hace asignando una "Tecla virtual" a una tecla estándar del teclado mapeada internamente y luego utilizando los eventos de teclado que generan para controlar su aplicación. Vale la pena señalar que estas "teclas virtuales" se asignan a la posición de la pantalla en lugar de la posición de la habitación por lo que los valores x/y son absolutos, basados en el tamaño de la pantalla. Esto significa que no necesitas preocuparte por el uso de vistas o las coordenadas relativas de la habitación, y puedes simplemente definir las teclas en el evento Create de un objeto (sólo necesitas definir una llave virtual una vez por habitación), y luego dibujar sus sprites clave en el evento Draw GUI, dimensionando la capa GUI para que sea la misma que la pantalla.

NOTA: Estas funciones no funcionarán en las plataformas de destino Windows, Ubuntu(Linux), ni MacOS, utilizando una pantalla táctil, aunque recibirán los clics del ratón como "toques".

Las funciones de las teclas virtuales disponibles son:

  • virtual_key_add
  • virtual_key_show
  • virtual_key_hide
  • virtual_key_delete

Si necesitas introducir texto o números en tu juego en un dispositivo con pantalla táctil, puedes utilizar las funciones de teclado virtual que se ofrecen para utilizar el teclado del sistema operativo del dispositivo. Estas funciones llamarán u ocultarán el teclado del SO y aceptarán la entrada desde él, Sin embargo, no activarán los eventos regulares del teclado, sino que actualizarán la cadena_de_teclado variable con la entrada del usuario (esto se debe al hecho de que el teclado virtual puede puede introducir palabras enteras y no sólo caracteres individuales).

Abrir o cerrar el teclado virtual generará Eventos Asíncronos del Sistema, en los que el async_load DS se llenará con los siguientes pares clave/valor:

  • " event_type" - el tipo de evento del sistema recibido, que será "virtualkeyboard status" para los tecladosvirtuales.
  • "screen_height" - la altura del teclado virtual (en píxeles). Será 0 si el teclado es invisible.
  • "keyboard_status" - el estado actual del teclado, devuelto como una de las siguientes cadenas:
    • "hiding"
    • "oculto"
    • "mostrando"
    • "visible"

Las funciones del teclado virtual disponibles son:

NOTA: Estas funciones sólo son válidas para el Android (incluyendo AndroidTV), iOS (incluyendo tvOS) y UWP .

  • teclado_virtual_show
  • keyboard_virtual_hide
  • estado_virtual_del_teclado
  • altura_virtual_del_teclado
