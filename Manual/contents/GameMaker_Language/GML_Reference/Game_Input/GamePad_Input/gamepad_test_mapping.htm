gamepad_test_mapping

Esta función se puede utilizar para establecer el mapeo del gamepad en aquellos objetivos que lo permitan. Se suministra el índice de la "ranura" para el gamepad a establecer, y luego la cadena del mapa, que debe haber sido creada usando el formato SDL con los siguientes campos:

<guid>,<descripción>,plataforma:<nombre-de-la-plataforma>,<enlaces>

Los datos que deben incluirse en cada uno de estos campos son:

  • guid - una cadena de dígitos y letras que identifica de forma exclusiva un tipo de dispositivo (puede recuperarse mediante la función gamepad_get_guid() función)
  • description - una descripción Ascii del dispositivo (se puede recuperar con la función gamepad_get_description() función)
  • platform-name - será "platform:android", "platform:windows", "linux", o "mac" (tenga en cuenta que esto se añadirá automáticamente a la cadena cuando llame a esta función, y por lo tanto no es necesario suministrarlo cuando construya sus propias cadenas de asignación)
  • bindings - un conjunto de entradas separadas por un ',' para vincular la entrada real a constantes GML específicas

La lista de entradas separadas por comas sigue un formato específico para cada entrada:

<nombre-gp abstracto>:<enlace-valor>

Aquí <value-binding> (es decir, la entrada directa del gamepad) puede ser cualquiera de las siguientes:

  • a<N> - Representa un eje del gamepad, donde el <N> es uno de los ejes de 0 a gamepad_axis_count() -1
  • b<N> - Representa un botón del gamepad, donde el <N> es uno de los botones de 0 a gamepad_button_count() -1
  • h<N>.<máscara> - Esto representa un sombrero del gamepad, donde la <N> representa un sombrero de 0 a gamepad_hat_count() -1, y la <máscara> será 1, 2, 4 u 8.

Cada valor de entrada debe estar vinculado a un nombre SDL, que se expresa como <>nombre-gp abstracto> , que a su vez está vinculado a una constante GML. La siguiente tabla muestra la equivalencia entre el nombre SDL y la constante de GameMaker Studio 2:

"050000005e040000fd020000ffff3f00,Xbox Wireless Controller,a:b0,b:b1,start:b4,dpdown:h0.4,dpleft:h0.8,dpright:h0.2,dpup:h0.1,guide:b6,leftshoulder:b9,leftstick:b7,lefttrigger:a4,leftx:a0,lefty:a1,righthoulder:b10,righttick:b8,righttrigger:a5,rightx:a2,righty:a3,x:b2,y:b3,platform:android"

Sintaxis:

gamepad_test_mapping(index, mapping_string);

Argumento Descripción
índice Qué "ranura" del índice del gamepad hay que poner.
color La cadena del mapa que se va a utilizar (véase la descripción para más información).

Devuelve:

N/A

Ejemplo:

var mapping = gamepad_get_guid(global.padIndex) + "," + gamepad_get_description(global.padIndex);
var len = array_length(global.PadInstances);
for( i = 0; i < len; i += 2)
{
var left = global.PadInstances[i];
var right = global.PadInstances[i+1];
mapping += "," + left.sdlLabel + ":" + right.binding;
}
gamepad_test_mapping(global.padIndex, mapping);

El código anterior recorrerá un número de instancias y utilizará los valores de las diferentes variables que contienen para crear una cadena de mapeo, que luego se establece para su uso en el gamepad en el índice de ranura dado.
