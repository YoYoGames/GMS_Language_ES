red_crear_servidor

Esta función se utiliza para crear un nuevo servidor de red para su juego, utilizando uno de los protocolos de conexión permitidos (ver las constantes listadas abajo). Usted suministra el tipo de servidor, luego le da un puerto para usar, y finalmente el número de conexiones máximas que deben permitirse en cualquier momento al servidor (tenga en cuenta que este número depende de usted, pero demasiados clientes conectados saturarán la red o la CPU del dispositivo no podrá manejar el procesamiento de ese número de jugadores, así que úselo con cuidado). La función devuelve un id único que debe ser usado almacenado en una variable y utilizado para identificar el servidor en todas las funciones de red posteriores, o un valor menor que 0 si la conexión falla.

Constante Descripción
network_socket_tcp Crear un socket usando TCP.
network_socket_udp Crear un socket usando UDP.
network_socket_ws Crear un socket web(sólo para conectarse a proyectos HTML5), utilizando TCP.
network_socket_bluetooth Crear una toma Bluetooth(¡actualmente no disponible!).

Sintaxis:

network_create_server(type, port, max_client);

Argumento Descripción
tipo El tipo de servidor que se va a crear (ver las constantes que se indican a continuación).
puerto El puerto que utilizará el servidor.
max_cliente El número máximo de clientes que pueden conectarse a la vez.

Devuelve:

Real

Ejemplo:

var port = 6510;
server = network_create_server(network_socket_tcp, port, 32);
while (server < 0 && port < 65535)
{
puerto++
server = network_create_server(network_socket_tcp, port, 32);
}

El código anterior intentará crear un servidor usando TCP a través del puerto 6510. Si ese puerto no está disponible, entonces hará un bucle a través de los puertos para encontrar uno que sí lo esté.
