network_send_udp_raw

Con esta función puedes enviar datos a través de la red usando UDP a un servidor. La función toma el identificador de socket para conectarse, la URL a la que conectarse y el puerto a utilizar. A continuación, debe suministrar el identificador del búfer que contiene los datos a (para más información sobre los búferes, véase Referencia - Búferes) y, por último, el tamaño (en bytes) de los datos. UDP es "sin conexión" en el sentido de que en realidad no se hace una conexión, sólo se envía un paquete directamente a una IP, y el servidor recibe los datos entrantes de una dirección IP y tiene que tratarlos "tal cual". Los datos enviados no son formateados por GameMaker Studio 2 de ninguna manera y los dispositivos receptores obtendrán los datos como un flujo que significa que tendrá que manejarlo usted mismo. La función devolverá el número de bytes de datos enviados, o un número menor que 0 si el envío ha fallado.

NOTA: Esta función no funcionará cuando se utilice en un proyecto que se ejecute en el objetivo HTML5, y tampoco los proyectos HTML5 podrán recibir UDP.

Sintaxis:

network_send_udp_raw(socket, url, port, buffer, size);

Argumento Descripción
toma de corriente El id del socket a utilizar.
url La url o IP a la que conectarse (una cadena).
puerto El puerto al que hay que conectarse.
buffer El id del buffer del que se van a obtener los datos.
tamaño El tamaño (en bytes) de los datos.

Devuelve:

Real

Ejemplo:

network_send_udp_raw(sock, "www.macsweeneygames.com", 6510, buff, buffer_tell(buff));

El código anterior enviará un paquete UDP sin procesar al servidor definido por la URL en el puerto 6510. Los datos se toman del buffer indexado en la variable "buff".
