network_send_broadcast

Con esta función puede difundir los datos de un búfer localmente a un rango de direcciones IP (para más información sobre los búferes, véase Referencia - Búferes). El rango está limitado al del dispositivo que ejecuta el servidor, por lo que si el dispositivo tiene una IP de 92.168.11.130, los datos se difundirán en el rango 92.168.11.*. La función devolverá el número de bytes de datos enviados, o un número menor que 0 si el envío ha fallado.

NOTA: Esta función sólo funcionará cuando se utilice con UDP - su servidor necesita ser TCP y su cliente necesita tener un socket de cliente UDP creado con network_create_socket_ext() para poder recibir cualquier emisión enviada desde el servidor.

NOTA: Esta función no funcionará cuando se utilice en un proyecto que se ejecute en el objetivo HTML5.

Sintaxis:

network_send_broadcast(socket, port, buffer, size);

Argumento Descripción
toma de corriente El id del socket a utilizar.
puerto El puerto que utilizará el servidor.
buffer El id del buffer del que se van a obtener los datos.
tamaño El tamaño (en bytes) de los datos.

Devuelve:

Real

Ejemplo:

buffer_seek(broadcast_buffer, buffer_seek_start, 0);
buffer_write(broadcast_buffer, buffer_string, global.ServerName);
network_send_broadcast(server, 6511, broadcast_buffer, buffer_tell(broadcast_buffer));

El código anterior escribe la cadena de nombre del servidor actual (almacenada en "global.ServerName"), y luego la escribe en un buffer binario con el id "broadcast_buffer". Estos datos se difunden localmente a un rango de IPs (la IP del dispositivo está IP del dispositivo está implícita como el rango base de difusión) al puerto 6511.
