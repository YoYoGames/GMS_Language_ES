gc_get_stats

Con esta función puedes recuperar información sobre el estado actual del recolector de basura. La función devolverá una estructura que tendrá las siguientes variables miembro (nótese que "objetos" se refiere a cualquier cosa que pueda ser recolectada por el recolector de basura y no a las instancias generales de los objetos como se definen en el Navegador de Activos):

  • objects_touched - es el número de objetos activos que el recolector de basura encontró en el marco anterior. Esto variará dependiendo de la generación que se haya recogido.
  • objects_collected - el número de objetos que el recolector de basura determinó que no estaban activos en el marco anterior, y que por lo tanto podrían ser eliminados.
  • traversal_time - es el tiempo en microsegundos (en el hilo principal) que el recolector de basura tardó en averiguar qué objetos estaban activos.
  • tiempo_de_recolección - es el tiempo en microsegundos (en un hilo separado) que el recolector de basura tardó en limpiar los objetos considerados inactivos.
  • gc_frame - es un contador que se incrementa cada vez que se produce un pase de recolección de basura. Si la recolección de basura está deshabilitada no se incrementará.
  • generation_collected - es el índice de la última generación recogida. El 0 es la generación más joven y el 3 es la más antigua actualmente.
  • num_generations - es el número total de generaciones de recolección de basura.
  • num_objects_in_generation - es una matriz (de tamaño num_generations) que contiene el número de objetos en cada generación.

NOTA: En la plataforma de destino HTML5 la recolección de basura es manejada por el motor de JavaScript y por lo tanto las variables miembro en la estructura anterior devolverán 0 si esta función se utiliza en esa plataforma.

Al utilizar esta función, tenga en cuenta que la información mostrada para los objetos sólo se actualiza cuando se termina de procesar una generación completa, lo que puede tardar varios fotogramas dependiendo de la configuración del tiempo de los fotogramas (consulte aquí para obtener más información sobre el tiempo de los fotogramas).

Sintaxis:

gc_get_stats();

Devuelve:

Estructurar

Ejemplo:

if (global.debug == true)
{
var _s = gc_get_stats();
var _t = _s.traversal_time;
var _c = _s.collection_time;
show_debug_message("Tiempo de recorrido = " + string(_t))
show_debug_message("Tiempo de recogida = " + string(_c))
}

El código anterior comprueba una variable global y si es verdadera obtiene información del colector de basura y la envía a la consola como mensajes de depuración.
