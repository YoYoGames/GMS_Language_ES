Búferes

Un búfer es una región de memoria física que se utiliza para mantener temporalmente los datos mientras se mueven de un lugar a otro o se manipulan de alguna manera, por ejemplo, se puede utilizar un búfer para mantener los datos de los paquetes a la espera de la transmisión a través de una red o para almacenar un paquete que se ha recibido a través de una red. GameMaker Studio 2 tiene una serie de funciones específicas relacionadas con la creación, modificación y carga/guardado de búferes, pero antes de utilizarlos ten en cuenta los siguientes puntos:

  • Cuando se crea un búfer, se borra automáticamente y se llena a 0.
  • El argumento "size" de las funciones del buffer siempre se refiere al tamaño en bytes del buffer.
  • El argumento "Alineación" se refiere a cómo se almacenan los datos dentro de un buffer. Si tu alineación está configurada, por ejemplo, a 4, y escribes un solo dato de 1 byte de tamaño y haces un buffer_tell, obtendrás un offset de 1. Sin embargo, si escribes otra pieza de datos, también de 1 byte de tamaño, y luego haces un buffer_tell, obtendrás un desplazamiento de 5 ya que la alineación ha "rellenado" los datos a ese tamaño. La alineación sólo afecta al lugar donde se escriben las cosas, así que si haces un buffer_tell después de escribir algo, devolverá la posición de escritura actual que sigue inmediatamente a los datos que has escrito. Tenga en cuenta, sin embargo, que si luego escribe otra pieza de datos, internamente el buffer moverá la posición de escritura hasta el siguiente múltiplo del tamaño de alineación antes de escribir el dato.
  • Cuando se trata de "offset", este es el valor en bytes para desplazar los datos dentro del buffer dado. Por lo tanto, si el búfer está alineado en 2 bytes y quieres saltar los primeros 4 lugares alineados, tendrías un offset de 2 * 4 bytes.
  • Algunas de las funciones del buffer también crean un nuevo buffer (como buffer_load por ejemplo). Recuerda que estos búferes también tendrán que ser eliminados de la memoria cuando no estén en uso utilizando la función buffer_delete.
  • La memoria utilizada para la creación de buffers es la memoria del sistema, por lo que incluso cuando el juego no tiene el foco, cualquier dato almacenado en un buffer debe ser seguro, sin embargo, si la aplicación se cierra o se reinicia entonces se perderá.

  • Uso de los buffers

IMPORTANTE! Cuando se crea un buffer, el valor del índice utilizado para identificarlo es un valor entero que comienza en 0. Estos índices son reutilizados por GameMaker, por lo que un valor de índice de buffer destruido puede ser utilizado por uno recién creado posteriormente, por lo que se recomienda establecer cualquier variable que contenga un índice de buffer a -1 después de haber destruido el buffer.
