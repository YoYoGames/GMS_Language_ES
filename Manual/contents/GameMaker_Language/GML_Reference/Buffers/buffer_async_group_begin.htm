buffer_async_group_begin

Esta función se llama cuando se quiere comenzar a guardar varios buffers en varios archivos. El "nombre del grupo" es una cadena y se utilizará como el nombre del directorio donde se guardarán los archivos, y debe utilizarse como parte de la ruta del archivo cuando se carguen los archivos en el IDE más tarde (usando cualquiera de las funciones buffer_load() ). Esta función sólo se puede utilizar con la función buffer_save_async() función y también debe terminar la definición de guardado llamando a la función buffer_async_group_end() de lo contrario los archivos no se guardarán.

Ten en cuenta que para las plataformas de consola (como PS4, por ejemplo), el "nombre del grupo" se utilizará como la descripción de la ranura de guardado, y el uso de esta función puede ayudarte a evitar que se muestre la interfaz de usuario para cada archivo que se está guardando. También tenga en cuenta que cuando UWP, puedes mover tus partidas guardadas entre las ventanas nativas de XBox y de la tienda de Microsoft creando un grupo que sea el mismo que un grupo de ERA Xbox, por ejemplo:

si os_type == os_uwp
{
buffer_async_group_begin("root/nombre_del_grupo");
}
else
{
buffer_async_group_begin("yourgroupname");
}

Sintaxis:

buffer_async_group_begin(groupname);

Argumento Descripción
nombre del grupo El nombre del grupo (como cadena).

Devuelve:

N/A

Ejemplo:

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
buffer_async_group_end();

El código anterior inicia un grupo de búferes y lo configura para guardar 4 archivos de forma asíncrona. A continuación, se finaliza la definición del grupo (momento en el que comenzará el guardado).
