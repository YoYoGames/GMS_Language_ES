buffer_load_async

Con esta función puedes cargar en un buffer un archivo que hayas creado previamente con la función buffer_save() (o cualquiera de las otras funciones para guardar buffers) en un buffer. El "offset" define la posición inicial dentro del buffer para la carga (en bytes), y el "tamaño" es el tamaño del área del buffer que se cargará a partir de ese offset (también en bytes). Puede proporcionar un valor de -1 para el argumento de tamaño y se cargará todo el buffer. Tenga en cuenta que la función cargará desde una carpeta "por defecto", que no necesita ser incluida como parte de la ruta de archivo que usted proporcione. Esta carpeta se creará si no existe o cuando guarde un archivo usando buffer_save_async().

La función devuelve un valor de ID único que puede ser utilizado en el evento asíncrono Save / Load para comprobar el async_load valor de ID, como se muestra en el ejemplo extendido a continuación. El mapa async_load en el evento tendrá los siguientes dos pares clave/valor:

  • "id": el ID de la función asíncrona devuelto por la función de guardar.
  • "status": devolverá true si los datos se han guardado/cargado correctamente, y false en caso contrario.

NOTA: En HTML5, este es el método preferido para cargar un archivo si se está cargando desde un servidor y no desde el almacenamiento local, ya que la carga sincrónica ha quedado obsoleta en la mayoría de los navegadores y con el tiempo lo será.

Sintaxis:

buffer_load_async(buffer, filename, offset, size);

Argumento Descripción
buffer El índice del buffer a cargar.
nombre de archivo El nombre del archivo a cargar.
offset El desplazamiento dentro del búfer para cargar (en bytes).
tamaño El tamaño del área del buffer a cargar (en bytes).

Real

Ejemplo ampliado:

La función buffer_load_async( ) puede ser llamada desde cualquier evento, y como es asíncrona la llamada de retorno puede ser casi instantánea o puede tardar varios segundos. Llamar a la función es sencillo y sería algo así:

loadid = buffer_load_async(buff, "Player_Save.sav", 0, 16384);

El código anterior carga el contenido del archivo "Player_Save.sav" en el buffer dado, almacenando el ID de la llamada a la función en la variable "loadid". Cuando la carga esté completa, el evento asíncrono Save/Load será y puedes analizar el mapa async_load para encontrar el ID correcto de la función, así:

if ds_map_find_value(async_load, "id") == loadid
{
if ds_map_find_value(async_load, "status") == false
{
show_debug_message("¡Carga fallida!");
}
}

El código anterior comprobará primero el ID del mapa DS que se ha creado, y luego comprobará el estado de la llamada de retorno, publicando un mensaje de depuración si ha habido algún problema.
