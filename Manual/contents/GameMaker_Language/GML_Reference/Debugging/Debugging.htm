Depuración

Al crear un juego, a menudo es muy importante conocer ciertos detalles de la mecánica del juego en momentos concretos, o poder introducir ciertos valores clave para algunas funciones para probar diferentes métodos e ideas para tu juego. Para ello GameMaker Studio 2 le proporciona una serie de funciones de depuración que mostrarán mensajes y pedir la entrada, mientras que en el modo de depuración (o en un objetivo de Windows ), pero que será ignorado cuando se compila para otros módulos de destino debido al hecho de que estas funciones ponen el juego en un bucle cerrado hasta que la entrada se resuelve.

NOTA: Aunque pueden funcionar en determinadas plataformas, no deben utilizarse para nada más que para depurar.

  • modo_depuración
  • evento_depuración
  • debug_get_callstack
  • exception_unhandled_handler
  • get_integer
  • get_string
  • mostrar_error
  • mostrar_mensaje
  • mostrar_pregunta
  • show_debug_message
  • show_debug_overlay
  • código_es_compilado
  • fps
  • fps_real

Aparte de estas funciones especializadas, para ayudarte a crear tu juego, GameMaker Studio 2 también tiene una larga lista de mensajes de error que se pueden mostrar tanto al escribir el código del juego, como al compilarlo o cuando el juego se está ejecutando. Estos mensajes señalan el error específico en cuestión y pueden ser utilizados para localizar un problema dentro de un objeto específico o función de script, ayudando en gran medida con el proceso de depuración de su juego. Sin embargo, no siempre es obvio lo que significa el mensaje de error. Sin embargo, no siempre es obvio lo que el mensaje de error significa, por lo que las siguientes secciones enumeran todos los posibles mensajes y explican lo que significan para hacer la vida más fácil para usted:

  • Errores de sintaxis
  • Errores del compilador
  • Errores del corredor

También puedes encontrar más información sobre cómo depurar tu juego en la sección El depurador.
