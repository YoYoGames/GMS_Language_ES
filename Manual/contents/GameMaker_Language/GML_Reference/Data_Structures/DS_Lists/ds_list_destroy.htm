ds_list_destroy

Esta función eliminará de la memoria la estructura de datos de la lista dada, liberando los recursos que estaba utilizando y eliminando todos los valores que contenía. Esta función debe usarse siempre cuando se haya terminado de usar ds_list para evitar fugas de memoria que pueden ralentizar y bloquear el juego. Ten en cuenta que destruir una lista desreferenciará cualquier estructura de datos almacenada en ella dando lugar a una fuga de memoria, por lo que tendrías que recorrer la lista y destruir todos los elementos de la estructura de datos manualmente antes de destruirla para evitarlo. La única vez que esto no es necesario es cuando has marcado cualquier elemento de la lista como otra lista DS o como un mapa DS, en cuyo caso estos elementos serán destruidos y su y su memoria se limpiará automáticamente.

IMPORTANTE! Cuando se crea una estructura de datos, el valor del índice para identificarla es un valor entero que comienza en 0. Esto significa que las estructuras de datos de diferentes tipos pueden tener el mismo valor de índice, por lo que en caso de duda se debe utilizar la función ds_exists() antes de acceder a ellas. También hay que tener en cuenta que los índices se reutilizan, por lo que el valor del índice de una estructura de datos destruida puede ser utilizado por otra recién creada posteriormente, por lo que recomendamos siempre establecer la variable que contenía el índice DS a -1 después de la destrucción.

Sintaxis:

ds_list_destroy(id);

Argumento Descripción
id El id de la estructura de datos a eliminar.

Devuelve:

N/A

Ejemplo:

si (vidas == 0)
{
ds_list_destroy(AI_list);
AI_list = -1;
room_goto(rm_Menu);
}

El código anterior comprobará el valor de la variable global incorporada "vidas" y si es 0, destruye la lista DS indexada en la variable "AI_list" y luego cambia de habitación.
