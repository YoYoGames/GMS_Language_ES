Listas DS

Una lista DS es una estructura de datos que almacena información de forma secuencial a medida que se va añadiendo (de forma muy parecida a un array). Las listas DS son estructuras de datos muy flexibles que permiten añadir valores al final o insertarlos al final o insertarlos en algún lugar en el medio de la lista, así como la capacidad de barajar el orden (para aleatorizar los valores) o de ordenarlos en forma ascendente o descendente.

Cuando se accede a las estructuras de datos de la lista DS, siempre se debe tratar de utilizar valores enteros para la posición de la lista, y todos los índices no enteros serán floteados por GameMaker Studio 2 si no lo hace. Si esto no es lo que necesitas, entonces tendrás que hacer el redondeo usted mismo previamente antes de pasar el índice que desea comprobar.

NOTA: Como todos los recursos dinámicos, las estructuras de datos ocupan memoria, por lo que siempre deben ser destruidas cuando ya no se necesiten para evitar fugas de memoria que ralentizarán y eventualmente bloquearán tu juego.

NOTA: si necesita comprobar si una estructura de datos existe, puede utilizar la función ds_exists() función.

Aparte de las funciones específicas que se enumeran a continuación, todos pueden utilizar una expresión (llamada accesor) para añadir o modificar el contenido de su lista DS. Este accesor se parece a un array 1D con la siguiente sintaxis:

list_index[| index]

Puede encontrar más información, además de ejemplos, en la página de descripción general de GML sobre los accesorios.

Existen las siguientes funciones que se ocupan de las listas DS.

  • ds_list_create
  • ds_list_destroy
  • ds_list_clear
  • ds_list_empty
  • ds_list_size
  • ds_list_add
  • ds_list_set
  • ds_list_delete
  • ds_list_find_index
  • ds_list_find_value
  • ds_list_insert
  • ds_list_replace
  • ds_list_shuffle
  • ds_list_sort
  • ds_list_copy
  • ds_list_read
  • ds_list_write

Las listas Ds pueden ser usadas para contener mapas DS y otras listas DS también, y por ello tenemos algunas funciones especiales que se usan para "marcar" posiciones dentro de la lista como contenedoras de estas estructuras de datos. En esto no es necesario, pero tiene la ventaja de recoger automáticamente la basura (destruir) las estructuras de datos marcadas cuando se destruye la lista padre, y también es necesario cuando se utiliza junto con la función json_encode() para que la lista y sus datos se codifiquen correctamente. Las funciones para esto son:

  • ds_list_mark_as_list
  • ds_list_mark_as_map
  • ds_list_is_list
  • ds_list_is_map
