ds_map_secure_save

Esta función guardará el contenido del mapa DS dado en un archivo que está vinculado al dispositivo en el que fue creado (lo que significa que no puede ser leído si se transfiere a cualquier otro dispositivo). El archivo en sí puede tener casi cualquier extensión (por ejemplo, *.dat, *. json, *.bin, etc...) y será ofuscado y guardado en el almacenamiento local de la plataforma de destino. A continuación, puede volver a cargar el ds_map utilizando la función ds_map_secure_load(). Tenga en cuenta que si el mapa DS que se está guardando contiene un array, este array se convertirá en una lista DS cuando se guarde.

IMPORTANTE! Una de las características de un archivo guardado de forma segura es que está bloqueado para el dispositivo en el que fue creado, por lo que no se puede cargar un archivo guardado en un dispositivo en un proyecto que se ejecuta en otro dispositivo.

Sintaxis:

ds_map_secure_save(map, filename);

Argumento Descripción
mapa El id de la estructura de datos a utilizar
nombre de archivo El nombre del archivo en el que se guardará el mapa

Devuelve:

Booleano

Ejemplo:

ds_map_secure_save(purchase_map, "p_data.dat");

El código anterior guardará el mapa DS indexado en la variable "p_data" en el archivo dado para su posterior recuperación.
