ds_map_destroy

Los mapas DS ocupan espacio en la memoria, que se les asigna cuando se crean. Esto significa que también hay que liberar esta memoria cuando el mapa DS no se necesita para evitar errores, fugas de memoria y pérdida de rendimiento al ejecutar el juego. Esta función hace precisamente eso. Ten en cuenta que destruir un mapa desreferenciará cualquier estructura de datos almacenada en el mapa dando lugar a una fuga de memoria, por lo que tendrías que revisar el mapa y destruir todos los elementos de la estructura de datos manualmente antes de destruirlo para evitar esto. El único caso en el que esto no es necesario es cuando se ha marcado cualquier elemento del mapa como una lista DS o como otro mapa DS, en cuyo caso estos elementos se destruirán y su memoria se limpiará automáticamente.

IMPORTANTE! Cuando se crea una estructura de datos, el valor del índice para identificarla es un valor entero que comienza en 0. Esto significa que las estructuras de datos de diferentes tipos pueden tener el mismo valor de índice, por lo que en caso de duda se debe utilizar el ds_exists() antes de acceder a ellas. También hay que tener en cuenta que los índices se reutilizan, por lo que el valor del índice de una estructura de datos destruida puede ser utilizado por otra recién creada posteriormente, por lo que recomendamos siempre establecer la variable que contenía el índice DS a -1 después de la destrucción.

Sintaxis:

ds_map_destroy(id);

Argumento Descripción
id El id del mapa a destruir.

Devuelve:

N/A

Ejemplo:

ds_map_destroy(inventory);
inventory = -1;

El código anterior destruirá el mapa DS con el id indexado en la variable "inventario".
