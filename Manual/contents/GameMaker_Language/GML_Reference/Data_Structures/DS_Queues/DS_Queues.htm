Colas DS

Una cola DS es algo similar a una pila DS, pero funciona sobre la base de primero en entrar primero en salir (FIFO). El valor que se pone en la cola primero es también el primero en ser eliminado de ella. Puede pensar en ello como una cola en una tienda para pagar, donde la persona que está primero en la cola paga primero y luego sale de la tienda. Las colas se utilizan normalmente para almacenar acciones o comportamientos (en forma de scripts, por ejemplo) que todavía deben ser realizados por una instancia pero hay muchos otros usos.

NOTA: Como todos los recursos dinámicos, las estructuras de datos ocupan memoria, por lo que siempre deben ser destruidas cuando ya no se necesiten para evitar fugas de memoria que ralentizarán y eventualmente bloquearán tu juego.

NOTA: si necesita comprobar si una estructura de datos existe, puede utilizar la función ds_exists() función

Existen las siguientes funciones que se ocupan de las colas DS:

  • ds_queue_create
  • ds_queue_destroy
  • ds_queue_clear
  • ds_queue_empty
  • ds_queue_size
  • ds_queue_dequeue
  • ds_queue_enqueue
  • ds_queue_head
  • ds_queue_tail
  • ds_queue_copy
  • ds_queue_read
  • ds_queue_write
