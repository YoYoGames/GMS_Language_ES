Rejillas DS

Una cuadrícula DS es básicamente un tipo de matriz con 2 dimensiones. Se define la rejilla estableciendo su anchura y altura enteras, lo que a su vez establece el número total de "celdas" dentro de la rejilla (el ancho y el alto multiplicados) y estas celdas se utilizan para contener diferentes valores de datos (números reales o cadenas).

Este tipo de estructura permite establecer y recuperar el valor de las celdas de la cuadrícula dando el índice x e y de la misma (que comienza con 0 tanto para el eje x como para el eje y y llega hasta el ancho - 1 y el alto -1). Pero también se puede establecer el valor en regiones específicas de la cuadrícula, añadir valores individuales y recuperar la suma, el máximo, el mínimo y el valor medio sobre una región determinada, lo que hace que esta estructura de datos sea la más versátil de todas las disponibles.

Cuando se accede a las estructuras de datos de la cuadrícula DS, siempre se debe tratar de utilizar valores enteros para la posición de la celda, y todos los índices no enteros serán floteados por GameMaker Studio 2 si no lo hace. Si esto no es lo que necesitas, entonces tendrás que hacer el redondeo usted mismo previamente antes de pasar el índice que desea comprobar.

NOTA: Como todos los recursos dinámicos, las estructuras de datos ocupan memoria, por lo que siempre deben ser destruidas cuando ya no se necesiten para evitar fugas de memoria que ralentizarán y eventualmente bloquearán tu juego.

NOTA: si necesita comprobar si una estructura de datos existe, puede utilizar la función ds_exists() función

Aparte de las funciones específicas que se enumeran a continuación, todos pueden utilizar una expresión (llamada accesorio) para añadir o modificar el contenido de su cuadrícula DS. Este accesorio es similar a un array 2D con la siguiente sintaxis:

grid_index[# xpos, ypos]

Puede encontrar más información, además de ejemplos, en la página de descripción general de GML sobre los accesorios.
