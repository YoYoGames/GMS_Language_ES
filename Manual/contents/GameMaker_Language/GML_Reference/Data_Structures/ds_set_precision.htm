ds_set_precision

Al comparar valores, por ejemplo al buscar en un mapa u ordenar una lista, GameMaker Studio 2 debe decidir cuándo dos valores son iguales. En el caso de las cadenas y los valores enteros esto está claro, pero en el caso de los números reales, debido a los errores de redondeo en coma flotante, los números aparentemente iguales pueden convertirse fácilmente en desiguales. de punto flotante, los números aparentemente iguales pueden convertirse fácilmente en desiguales. Por ejemplo, es posible que (5 / 3) * 3 no sea igual a 5! Para evitar esto, se utiliza un valor de precisión se utiliza en todas las funciones de números reales, y cuando la diferencia entre dos números es menor que esta precisión se consideran iguales. Por defecto se utiliza una precisión de 0.0000001 para todas las funciones de estructura de datos a menos que se cambie por esta función.

NOTA: ¡Esta precisión se utiliza en todas las estructuras de datos pero no en otras comparaciones en GML!

Sintaxis:

ds_set_precision(prec);

Argumento Descripción
prec El valor de precisión (por defecto 0,0000001)

Devuelve:

Devoluciones:N/A

Ejemplo:

ds_set_precision(0.0001);

El código anterior cambiará la configuración de precisión por defecto para todas las funciones de estructura de datos.
