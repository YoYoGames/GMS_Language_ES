xboxlive_sprite_add_from_gamerpicture

Con esta función puedes obtener la imagen del jugador para un puntero de ID de usuario dado. La función funciona de forma asíncrona, y lanzará un evento asíncrono Image Loaded para informarle de que la función ha finalizado (como con las llamadas HTML5/URL a sprite_add()). El evento asíncrono Image Loaded tendrá un mapa DS almacenado en la variable incorporada async_load. El mapa contiene la siguiente información:

  • " nombre del archivo": La ruta completa del archivo solicitado.
  • " id": El ID del recurso que has cargado. Será el mismo que la variable a la que has asignado el recurso.
  • "estado": Devuelve un valor inferior a 0 para un error.

NOTA: Cuando cargues dinámicamente un sprite en tu juego en tiempo de ejecución, debes recordar eliminarlo de nuevo usando sprite_delete() cuando ya no lo necesites, de lo contrario existe el riesgo de una fuga de memoria que ralentizará y eventualmente bloqueará tu juego.

Sintaxis:

xboxlive_sprite_add_from_gamerpicture(user_id, imagesize, xorig, yorig);

Argumento Descripción
user_id El ID del usuario (un puntero) para obtener la imagen del jugador para
tamaño de las imágenes Tamaño en píxeles del sprite a devolver
xorig Indica la posición x del origen en el sprite
yorig Indica la posición y del origen en el sprite

Devuelve:

Real

Ejemplo:

var _a = 0;
var _num = xboxlive_get_user_count();
for (var i = 0; i < _num; ++i;)
{
var _uid = xboxlive_get_user(i);
if _uid != pointer_null
{
global.UserName[_a] = xboxlive_gamedisplayname_for_user(_uid);
global.UserScore[_a] = xboxlive_gamerscore_for_user(_uid);
global.UserRep[_a] = xboxlive_reputation_for_user(_uid);
global.UserAvatar[_a] = xboxlive_sprite_add_from_gamerpicture(_uid, 256, 0, 0);
++a;
}
}

El código anterior recorre los usuarios registrados y almacena sus datos de jugador en varias matrices globales.
