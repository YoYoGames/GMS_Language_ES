xboxlive_matchmaking_join_session

Esta función te permite unirte a una sesión específica a través de un identificador de sesión. Esto es similar a unirse a una invitación de sesión, pero es útil cuando tienes tu propio servidor realizando el matchmaking en lugar de confiar en el matchmaking de xbox live.

La función requiere que proporcione el ID de usuario -que puede recuperar para un pad de juego concreto llamando a la función xboxlive_user_for_pad() así como la plantilla de Asociación de Dispositivos Seguros que deberías haber creado de antemano en el panel de control de XDP.

Para el manejo de la sesión, tendrá que haber recuperado primero el "id de correlación" de la sesión a la que desea unirse. Cuando se crea la sesión original, el evento social asíncrono será "session_created" e incluirá una entrada de "correlationid" en el mapa devuelto (véase la sección sobre la correlación). incluirá una entrada "correlationid" en el mapa devuelto (ver xboxlive_matchmaking_create()). Será necesario pasar este ID al servidor para que luego sea pasado cuando recibir una solicitud de matchmaking. Una vez que el juego que se une ha recibido el "correlationid" de vuelta del servidor, pueden llamar a esta función pasando el ID de correlación como el argumento de manejo de la sesión.

La función devolverá -1 si ha habido un error (en cuyo caso debería haber una salida de consola detallando el problema), o un valor de ID de solicitud en caso contrario. El valor del ID de la solicitud puede utilizarse para identificar el evento social asíncrono correcto para esta función. Este evento se activará cuando se haya creado la sesión y contendrá un Mapa DS en la variable async_load con los siguientes pares clave/valor:

  • " requestid" - contiene el ID de la solicitud devuelto por la función de llamada
  • "status" - contiene la cadena "session_player_joined" para informar del tipo de evento que se ha activado.
  • "sessionid" - contiene el ID de la sesión, o -1 en caso de fallo.
  • "sessionOwner< index> "- contiene el ID del anfitrión de la sesión si se ha encontrado una sesión (< index> es un valorde 0 a "num_results" - 1)
  • " id" - contiene la constante MATCHMAKING_SESSION
  • " error" - será 0 si todo está bien, o -1 si algo falló

Cuando un nuevo jugador se une a la sesión, el mapa DS tendrá estos pares clave/valor:

  • " id" - contiene la constante MATCHMAKING_SESSION
  • "sessionid" - contiene el ID de la sesión
  • "estado" - contiene la cadena "session_player_joined"
  • " requestid" - El valor de ID que fue devuelto por la función que creó la solicitud

Sintaxis:

xboxlive_matchmaking_join_session(user_id, session_handle, sdatemplate);

Argumento Descripción
user_id El puntero de ID de usuario que se va a utilizar
mango_de_sesión El "identificador de correlación" que se utilizará para el identificador de la sesión (véase la descripción, más arriba)
sdatemplate El nombre de la plantilla de asociación de dispositivos seguros que se va a utilizar

Devuelve:

ID de la solicitud (entero)

Ejemplo:

Request_ID = xboxlive_matchmaking_join_session(global.UserID, global.SessionID, "PeerServerTraffic");

Lo anterior enviará una solicitud para unirse a la sesión multijugador dada, almacenando el valor del ID de la solicitud en una variable para que pueda ser comprobado en el evento asíncrono.
