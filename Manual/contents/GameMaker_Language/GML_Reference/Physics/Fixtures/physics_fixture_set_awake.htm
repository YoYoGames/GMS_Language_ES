physics_fixture_set_awake

Cuando se inicia una sala con física, o se vinculan accesorios a instancias, los accesorios siempre se consideran "despiertos"... es decir, están procesando eventos e interactuando con las instancias circundantes. Sin embargo, esto puede llevar a veces Sin embargo, esto puede dar lugar a problemas, sobre todo si tienes un número de instancias con accesorios que están uno al lado del otro cuando se inicia una habitación (piensa en una torre hecha de varias instancias con accesorios), ya que al estar "despiertos" se moverán y posiblemente cambien de posición debido al inicio repentino de la gravedad y las colisiones que les afectan. En estos casos se puede utilizar esta función para marcar el accesorio como "dormido", en cuyo caso no procesará nada cuando se cree por primera vez en la sala.

Sintaxis:

physics_fixture_set_awake(fixture, state)

Argumento Descripción
accesorio el índice de la luminaria
bandera si una luminaria está despierta(true) o no(false)

Devuelve:

N/A

Ejemplo:

physics_fixture_set_awake(fix_Cloud, true);

El código anterior marca el accesorio como "despierto" cuando se vincula a una instancia.
