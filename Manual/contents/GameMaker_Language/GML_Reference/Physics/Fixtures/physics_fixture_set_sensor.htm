physics_fixture_set_sensor

Algunas veces querrás que tu juego detecte y reconozca cuando dos instancias con fixtures colisionan, pero sin tener ninguna reacción física a la colisión. Esto se puede hacer convirtiendo el fixture en un sensor, lo que básicamente significa que generar un evento de colisión, pero sin respuesta física, de modo que puedas utilizar estas instancias como "disparadores" para que se produzcan otros eventos en la sala de juego. Cualquier accesorio puede ser marcado como un sensor, y no hace ninguna diferencia si la instancia que está vinculada a es estática o en movimiento.

NOTA: Un accesorio de sensor disparará el evento de colisión sólo cuando la colisión ocurra por primera vez, lo que significa que no se obtiene un flujo de eventos de colisión a medida que los dos cuerpos continúan superponiéndose (que es lo que tradicionalmente ocurriría). Si se Si dejan de superponerse y se superponen posteriormente se disparará otro evento de colisión.

Sintaxis:

physics_fixture_set_sensor(fixture, state)

Argumento Descripción
accesorio el índice de la luminaria
estado si una luminaria es un sensor (verdadero) o no (falso)

Devuelve:

N/A

Ejemplo:

physics_fixture_set_sensor(fix_Cloud, 1);

El código anterior convertirá el estado del sensor del accesorio indexado en "fix_Cloud" en verdadero.
