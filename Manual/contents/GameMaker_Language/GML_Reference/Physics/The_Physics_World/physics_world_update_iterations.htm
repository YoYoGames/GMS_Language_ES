physics_world_update_iterations

Todo en GameMaker Studio 2 se basa en pasos y, normalmente, cada evento sólo ocurrirá una vez por paso. Sin embargo, para conseguir la precisión necesaria con las funciones de física se hacen varios cálculos en cada paso que se llaman "iteraciones" cuyo número es controlado por esta función. El número por defecto para el sistema de física es 10, pero esto puede ser cambiado a un número mayor o menor dependiendo de cuántas veces quieras que la física calcule cada paso, pero es pero se recomienda que no sea inferior a 5 ni superior a 30. También debe tener en cuenta que esta función depende de la función physics_world_update_speed() función.

Sintaxis:

physics_world_update_iterations(iteraciones)

Argumento Descripción
y1 las iteraciones (por paso) que realizará el sistema de física

Devuelve:

N/A

Ejemplo:

physics_world_update_iterations(20);

El código anterior establece el mundo de la física en la sala actual para realizar cálculos 20 veces por paso.
