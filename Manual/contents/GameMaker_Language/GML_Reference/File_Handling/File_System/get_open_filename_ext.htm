get_open_filename_ext

Esta función abre un diálogo y pide al jugador un nombre de archivo para abrirlo con el filtro dado. El filtro tiene la forma " nombre1|máscara1|nombre2|máscara2|... ", donde la máscara puede contener las diferentes opciones con un punto y coma entre ellas y también se puede utilizar un "*" para significar cualquier cadena. Por ejemplo: " bitmaps|*.bmp;*.wmf", pediría al usuario que seleccionara el nombre de un archivo de mapa de bits del sistema, pero sólo aquellos que tengan las dos extensiones especificadas (bmp y png). ¡Esta función no abre el archivo en sí! Sólo devuelve una cadena con el nombre completo y la ruta del archivo. Si el usuario pulsa "Cancelar" se devuelve una cadena vacía "". También puede añadir un directorio para iniciar la búsqueda, y un título que se mostrará en la parte superior de la ventana en lugar del predeterminado.

Una cosa importante a tener en cuenta cuando se utiliza esta función es que le otorga ciertos permisos para ese archivo, durante la duración de su juego. Así, una vez que obtengas la ruta del archivo del usuario, podrás acceder a él una y otra vez sin tener que preguntar. Sin embargo, como esta función es para leer un archivo, los permisos concedidos son sólo para leer y guardar no serán permitidos por el SO. Si necesita leer y escribir en el archivo, puede obtener el permiso utilizando la función get_save_filename().

NOTA: Esta función sólo es válida en los objetivos de Windows y macOS.

Sintaxis:

get_open_filename_ext(filter, fname, directory, caption);

Argumento Descripción
filtrar El filtro del tipo de archivo y/o el nombre.
fname El nombre de archivo que se sugiere utilizar.
directorio El directorio desde el que se inicia la búsqueda.
título El título de la ventana.

Devuelve:

Cadena

Ejemplo:

var file;
file = get_open_filename_ext("text file|*.txt", "", working_directory, "Open a saved level");
if file != ""
