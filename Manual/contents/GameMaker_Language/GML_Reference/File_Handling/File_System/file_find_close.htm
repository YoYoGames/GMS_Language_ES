file_find_close

Debe llamarse después de manejar los archivos abiertos con file_find_first() y file_find_next() para liberar memoria. Las funciones de búsqueda de archivos abren asas en el directorio de archivos y estos ocupan una cantidad mínima de memoria, que (con el tiempo) se acumula. Por lo tanto, siempre debe llamar a esta función después de haber encontrado los archivos que necesita para "cerrar" estos manejadores.

Sintaxis:

file_find_close();

Devuelve

N/A

Ejemplo:

file_find_close();

Esto cerrará y liberará la memoria asociada a los manejadores de la función de búsqueda de archivos.
