file_text_close

Una vez que hayas terminado de trabajar con un archivo determinado (ya sea leyendo de él o escribiendo en él), debes volver a cerrar el archivo, pues de lo contrario corres el riesgo de perder la información que contiene. Esto también evita las fugas de memoria y asegura que nunca se sobrepase el límite de archivos al tener más de 32 archivos abiertos. el límite de archivos por tener más de 32 archivos abiertos.

Sintaxis:

file_text_close(fileid);

Argumento Descripción
fileid El id del archivo a cerrar.

Devuelve:

N/A

Ejemplo:

var file = file_text_open_write(working_directory + "Game_Data.txt");
while (!file_text_eof(file))
{
file_text_readln(archivo);
}
file_text_write_string(file, level_data);
file_text_close(file);

El código anterior abre un archivo para escribirlo y luego recorre las líneas de texto ya escritas en el archivo hasta llegar al final. En este punto escribe una cadena y luego cierra el archivo de nuevo.
