Archivos de texto

Los archivos de texto son una forma útil de almacenar grandes cantidades de datos externos a tu juego. A diferencia de los archivos ini, puedes tener hasta 32 archivos de texto abiertos a la vez, y también puedes leer y escribir grandes cantidades de datos en ellos. Sin embargo, depende de ti decidir cómo deben estructurarse esos datos, ya que no hay una estructura de "sección" o "clave" establecida como en los archivos *.ini.

Tenga en cuenta que para los juegos con localizaciones es muy importante que las letras acentuadas puedan ser leídas desde archivos externos. Esto significa que debe crear primero el archivo txt en formato UTF8 y luego añadirlo en GameMaker Studio 2 como un archivo incluido para que se exporte al ejecutar el juego y se utilice en lugar del archivo txt con formato ANSI por defecto que se crea con las funciones de archivo de GameMaker Studio 2 cuando no se suministra previamente ningún archivo. De este modo, se puede leer y escribir en él correctamente, manteniendo todos los acentos y las letras no romanas.

Existen las siguientes funciones que se ocupan de los archivos:

  • file_text_open_read
  • file_text_open_write
  • file_text_open_append
  • file_text_open_from_string
  • file_text_read_real
  • file_text_read_string
  • file_text_readln
  • file_text_write_real
  • file_text_write_string
  • file_text_writeln
  • file_text_eoln
  • file_text_eof
  • file_text_close
