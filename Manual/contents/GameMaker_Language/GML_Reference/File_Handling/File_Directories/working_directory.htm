directorio_de_trabajo

Working_directory puede en realidad devolver dos valores diferentes dependiendo de para qué lo estés usando. Si está escribiendo un archivo en el disco, working_directory apunta al área de almacenamiento local que está reservada para su juego en el dispositivo de destino (en Windows es "%LOCALAPPDATA%/gamedir/" donde "gamedir" es el directorio con el nombre de tu juego). Sin embargo, cuando se lee desde el disco, working_directory puede ser tanto el almacenamiento local como el área donde se almacenan los archivos incluidos.

NOTA: La variable working_directory devolverá la ruta incluyendo la barra invertida final.

Por ejemplo, si tienes un archivo *.txt por defecto incluido con tu juego y lo lees en la memoria usando working_directory, entonces ese archivo será tomado del área donde se almacenan los archivos incluidos. Si luego elige escribir esa información en un archivo de texto utilizando working_directory, esto escribirá el archivo en el almacenamiento local de las plataformas de destino, desde donde se leerá la próxima vez a menos que indique a GameMaker Studio 2 que elimine el archivo, en cuyo caso working_directory volverá a apuntar al área donde se almacenan los archivos incluidos.

Vale la pena señalar que , en general, no es necesario utilizar esto y simplemente llamar al nombre del archivo (más cualquier información adicional de la ruta) es suficiente, por ejemplo esto:

file_text_open_read(working_directory + "my_file.txt");

sería mejor hacerlo así:

file_text_open_read("mi_fichero.txt");

ATENCIÓN! Esta función puede no funcionar como se espera debido a que GameMaker Studio 2 está en un entorno aislado. Por favor, consulte la sección sobre el sistema de archivos para obtener más información.

Sintaxis:

directorio_de_trabajo

Devuelve:

Cadena

Ejemplo:

ini_open(directorio_de_trabajo + "temp_ini.ini");

Esto abrirá un archivo ini desde el directorio de trabajo del juego (creándolo si no existe ya). Este puede ser el almacenamiento local, o el área donde están los archivos incluidos dependiendo de si el archivo que se busca existe en cualquiera de los dos lugares.
