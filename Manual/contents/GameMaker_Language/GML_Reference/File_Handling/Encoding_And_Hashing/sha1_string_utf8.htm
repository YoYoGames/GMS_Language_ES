sha1_string_utf8

En criptografía, SHA-1 es una función hash criptográfica diseñada por la Agencia de Seguridad Nacional de los Estados Unidos y que se emplea en varias aplicaciones y protocolos ampliamente utilizados como el popular Git donde se utiliza para comprobar los cambios en los archivos. Esta función toma una cadena UTF-8 de entrada (que tiene un número variable de bytes por carácter) y devuelve un resumen de mensaje de 160 bits en formato ASCII, por ejemplo:

sha1 cadena de entrada = "El rápido zorro marrón salta sobre el perro perezoso"
sha1 hash de salida = "2fd4e1c6 7a2d28fc ed849ee1 bb76e739 1b93eb12"

De este modo, se puede generar una clave segura que se puede almacenar y utilizar para comprobar la integridad de la información que se envía a (o se recibe de) un servidor externo (por ejemplo).

NOTA: Hay dos formatos para la codificación sha1, UTF-8 y unicode. Ambos se proporcionan para facilitar la comunicación con diferentes configuraciones de servidor, pero el más común de usar es unicode.

NOTA: SHA-1 no es completamente seguro y puede romperse. Consulte esta página para obtener más información.

Sintaxis:

sha1_string_utf8(cadena)

Argumento Descripción
cadena La cadena a hash.

Devuelve:

Cadena

Ejemplo:

var hash, str;
str = json_encode(hiscore_map);
hash = sha1_string_utf8(str);
ini_open("local.ini");
ini_write_string("info", "0", hash);
ini_close();
get[0] = http_post_string("http://www.MacSweeney Games.com/CatchTheHaggis?game_hiscores=" + string(global.game_id), str)

El código anterior codificará un mapa DS en una cadena JSON. A continuación, se genera un hash sha1 y se almacena en un archivo ini para que pueda utilizarse posteriormente para comprobar la integridad del JSON si se recibe la misma información del servidor. El JSON se envía entonces.
