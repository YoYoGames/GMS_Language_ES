json_decode

IMPORTANTE! Esta función -aunque sigue siendo válida- ha sido superada por la función json_parse(), y le recomendamos que sólo utilice esta función para el soporte heredado.

JSON (JavaScript Object Notation) es un formato de intercambio de datos ligero y fácil de leer y escribir, tanto para las personas como para las máquinas. Se basa en dos estructuras básicas:

  • Una colección de pares nombre/valor, llamada Mapa DS en GameMaker Studio 2 pero que también se conoce como "diccionario" u "objeto".
  • Una lista ordenada de valores, llamada Lista DS en GameMaker Studio 2, pero también puede llamarse "matriz" o "secuencia".

Con esta función, se puede decodificar un trozo de JSON y convertirlo en un mapa DS, listo para ser utilizado en GameMaker Studio 2. Si el JSON a decodificar requiere una jerarquía de listas y mapas dentro del mapa DS central, estos son decodificados también y también creados para usted, utilizando las siguientes reglas (tenga en cuenta que estas reglas se aplican a la estructura de nivel superior solamente):

  • JSON es un único valor - devuelve un mapa DS con una única entrada "default" que es el valor
  • JSON es un array de objetos o valores - devuelve un mapa DS con una única entrada "default" que es una lista DS de los objetos o valores
  • JSON es un objeto - devuelve un mapa DS que tiene las entradas del objeto

NOTA: Cuando se decodifican arrays JSON, hay un mapa con la clave "default" SÓLO cuando un array es la estructura de nivel superior, y SÓLO para ese array de nivel superior. Las listas internas se decodifican directamente en el mapa DS sin estar encerradas en un mapa DS.

NOTA: Si escribiste arrays de GameMaker en el nivel superior, o como el contenido de un mapa DS o una lista DS, estos serán decodificados como listas DS, no arrays.

Normalmente se sabe a qué claves decodifica el JSON, pero si no es así, se puede utilizar la función ds_map_size(), ds_map_find_first() y ds_map_find_next() para analizar el mapa y obtener la información necesaria.

NOTA: GameMaker Studio 2 crea los mapas y listas de DS necesarios a partir del JSON, y para la limpieza sólo tienes que borrar el mapa o lista de nivel superior y GameMaker Studio 2 borrará automáticamente de la memoria todos los mapas y listas inferiores.

IMPORTANTE: No puede tener números de 64 bits en su JSON, ya que no funcionarán correctamente debido a que no son manejados por el formato JSON.

Sintaxis:

json_decode(cadena)

Argumento Descripción
cadena La cadena de formato JSON que se pasa a la función para su decodificación

Devuelve:

DS map id o -1 si falla

Ejemplo:

var resultMap = json_decode(requestResult);
var list = ds_map_find_value(resultMap, "default");
var size = ds_list_size(list);
for (var n = 0; n < ds_list_size(list); n++;)
{
var map = ds_list_find_value(list, n);
var curr = ds_map_find_first(map);
while (is_string(curr))
{
global.Name[n] = ds_map_find_value(map, "name");
curr = ds_map_find_next(map, curr);
}
}
ds_map_destroy(resultMap);

El código anterior decodificará una cadena JSON y la analizará para generar un array global.
