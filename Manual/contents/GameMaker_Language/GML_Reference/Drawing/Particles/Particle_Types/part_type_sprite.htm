part_type_sprite

Esta función se puede utilizar para establecer un tipo de partícula para utilizar un sprite personalizado de los activos del juego. Puede seleccionar el sprite y luego decirle a GameMaker Studio 2 que use el sprite y lo anime o no, y si elige animado, entonces también puede elegir que las subimágenes cambien a un intervalo tal que la animación comience y termine al principio y al final del tiempo de vida de cada partícula creada (de modo que si tiene un tiempo de vida establecido en un mínimo de 15 pasos y un máximo de 30 pasos, no importa qué valor tenga el tiempo de vida, la animación será valor que tenga el tiempo de vida, la animación se ajustará, corriendo más rápido para los tiempos de vida más cortos y más lento para los más largos). También se puede elegir una sub-imagen aleatoria para que si el sprite no se anima, se elija una imagen diferente al azar para crear el sprite. al azar para crear el sprite, o si la partícula está animada comenzará la animación en un punto aleatorio.

Sintaxis:

part_type_sprite(ind, sprite, animate, stretch, random);

Argumento Descripción
ind El índice del tipo de partícula a cambiar.
sprite El índice del sprite para establecer el tipo de partícula.
animar Si se sigue la animación del sprite normalmente (true) o no (false).
estiramiento Si se estira la animación del sprite para que coincida con la duración de las partículas (true) o no (false).
al azar Si se elige una subimagen aleatoria (true) o no (false).

Devuelve:

N/A

Ejemplo:

global.p1 = part_type_create();
part_type_sprite(global.p1 , spr_Coins, 1, 0, 0);
part_type_size(global.p1, 1, 3, 0, 0);
part_type_scale(global.p1, 1, 1);
part_type_colour1(global.p1, c_white);
part_type_alpha2(global.p1, 1, 0);
part_type_speed(global.p1, 2, 4, 0, 0);
part_type_direction(global.p1, 0, 180, 0, 0);
part_type_gravity(global.p1, 0.20, 270);
part_type_orientation(global.p1, 0, 0, 0, 1);
part_type_blend(global.p1, 1);
part_type_life(global.p1, 15, 60);

El código anterior establece el tipo de partícula indexado en la variable global "p1" a la forma del sprite indexado en "spr_Coins", y luego establece varias otras propiedades de las partículas.
