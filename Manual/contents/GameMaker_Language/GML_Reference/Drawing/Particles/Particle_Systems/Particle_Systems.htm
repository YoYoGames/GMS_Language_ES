Sistemas de partículas

Antes de que puedas crear partículas necesitas crear un sistema de partículas. Se trata de un "contenedor" en el que colocas tus partículas y emisores (si los necesitas) listos para su uso, y puedes poner tantas o tan pocas partículas en cualquier sistema como que creas necesario, y puedes tener tantos sistemas como creas necesario también. Sin embargo, siempre es mejor mantener este número tan pequeño como sea posible debido al hecho de que cada sistema, emisor y partícula ocupa memoria y tener demasiados de ellos puede ralentizar el juego o causar problemas. Por ejemplo, si necesita que algunos efectos aparezcan por encima de las instancias, y otros efectos aparezcan por debajo de las instancias, crearía dos sistemas y establecería sus profundidades para obtener el efecto deseado, con todas las partículas que se añaden a cada sistema se dibujan en la profundidad que se especifica.

Dado que un sistema de partículas es un recurso creado dinámicamente, debes crearlo y almacenar el índice devuelto en una variable para referenciar el sistema en todas las llamadas de función posteriores, y es muy importante que también destruyas el sistema cuando ya no se necesite o tendrás una fuga de memoria que ralentizará y eventualmente bloqueará tu juego. También vale la pena señalar que los sistemas de partículas vivirán para siempre después de ser creados, incluso si el índice ya no se almacena. Así que incluso si usted cambia sala o reinicie el juego, los sistemas y las partículas permanecerán, y serán visibles, en todas las salas posteriores, así que será mejor que se asegure de destruirlos una vez que ya no los necesite.

A continuación se enumeran las funciones básicas para configurar un sistema de partículas:

  • part_system_exists
  • part_system_create
  • part_system_create_layer
  • part_system_get_layer
  • capa_sistema_parte
  • profundidad_sistema_parte
  • posición_sistema_parte
  • part_system_clear
  • part_system_destroy
  • part_particles_clear
  • conteo_de_partículas_de_parte

Una vez que las partículas se añaden a un sistema de partículas y luego estallan o se introducen en la sala, normalmente se actualizan automáticamente a cada paso y se dibujan en función de los parámetros que se han utilizado para definirlas. Sin embargo, a veces puede ser necesario controlar cuándo y cómo se actualiza el sistema, así como la forma en que el sistema debe ser dibujado, y para que GameMaker Studio 2 proporciona las siguientes funciones:

  • part_system_automatic_update
  • part_system_automatic_draw
  • part_system_update
  • sistema_parcial_drawit
  • part_system_draw_order

Puedes usar emisores de partículas para hacer estallar o arrojar partículas desde un área, pero en muchos casos no son necesarios y en realidad es mejor crear las partículas directamente usando las siguientes funciones:

  • part_particles_create
  • part_particles_create_colour
