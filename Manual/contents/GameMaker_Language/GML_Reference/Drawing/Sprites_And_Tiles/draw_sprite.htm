draw_sprite

Esta función dibuja el sprite y la sub-imagen dados en una posición dentro de la sala de juego. Para el sprite puedes utilizar la variable de instancia sprite_index para para obtener el sprite actual que está asignado a la instancia que ejecuta el código, o puedes usar cualquier otro activo de sprite. Lo mismo ocurre con la sub-imagen, ya que también se puede establecer en la variable de instancia índice_de_imagen que establecerá la sub-imagen a la seleccionada para el sprite de la instancia actual (ten en cuenta que puedes dibujar un sprite diferente y seguir usando el valor de la sub-imagen para la instancia actual), o puedes usar cualquier otro valor para esto para dibujar una sub-imagen específica del sprite elegido. Si el valor es mayor que el número de sub-imágenes, entonces GameMaker Studio 2 automáticamente hará un bucle con el número para seleccionar la imagen correspondiente (por ejemplo, si el sprite que se está dibujando tiene 5 sub-imágenes numeradas de 0 a 4 y establecemos el valor del índice en 7, entonces la función dibujará la subimagen 3, numerada como 0). Por último, la posición x e y es la posición dentro de la habitación en la que se dibujará el sprite, y está centrada en el desplazamiento x e y del sprite.

NOTA: Esta función puede no funcionar como se espera cuando se utilizan sprites de animación de esqueleto, y puede encontrar que la función sólo dibuja el primer cuadro de la pose por defecto. Deberías usar las funciones draw_skeleton_* en su lugar.

Sintaxis:

draw_sprite(sprite, subimg, x, y);

Argumento Descripción
sprite El índice del sprite a dibujar.
subimg La sub-imagen (fotograma) del sprite a dibujar (image_index o -1 se correlacionan con el fotograma actual de la animación en el objeto).
x La coordenada x de dónde dibujar el sprite.
y La coordenada y de dónde dibujar el sprite.

Devuelve:

N/A

Ejemplo:

draw_sprite(sprite_index, image_index, x, y);
draw_sprite(spr_Halo, 0, x, y-32);

Esto dibujará el sprite asignado a la instancia (sprite_index) con la subimagen actual en la posición x e y de la instancia dentro de la habitación. Luego dibujará el primer cuadro del sprite indexado por "spr_Halo" en la misma posición x e y pero 32 píxeles "por encima".
