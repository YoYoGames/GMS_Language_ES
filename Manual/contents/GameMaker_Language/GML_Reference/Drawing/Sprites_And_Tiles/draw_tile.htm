dibujar_elemento

Con esta función puedes dibujar cualquier baldosa de un conjunto de baldosas - compite con rotaciones, volteos y espejos - estableciendo el marco (si es animado) y la posición dentro de la habitación. El valor del ID del conjunto de baldosas es el índice del conjunto de baldosas establecido en el IDE y el dato de la baldosa es el "blob" de datos que contiene toda la información sobre el índice del mosaico y las transformaciones que se han aplicado. Se puede utilizar la función tilemap_get() para obtener los datos del mosaico, que luego pueden ser manipulados utilizando las funciones del mosaico, o puede construirlos usted mismo utilizando las siguientes constantes del mosaico junto con el valor del índice del mosaico a crear:

Constante Descripción
tile_rotate Se utiliza para establecer/obtener el bit de rotación de un blob de datos de baldosas.
espejo_azulejo Se utiliza para establecer/obtener el bit de espejo de un blob de datos de baldosas.
tile_flip Se utiliza para establecer/obtener el bit de volteo de un blob de datos de baldosas.
tile_index_mask Una constante especial que sirve para "y" con el blob de datos del mosaico para extraer el índice del mosaico.


Estas constantes se combinan con el valor del índice de la baldosa para obtener los datos finales de la baldosa:

var tiledata = 8 | tile_rotate | tile_flip;

El código anterior simplemente crea una variable local para almacenar los datos del azulejo con el índice 8 en el conjunto de azulejos, y se gira y voltea. Esto puede ser utilizado para dibujar el azulejo en ese índice del conjunto de azulejos que usted especifica. Esto no está vinculado a ninguna capa del mapa de azulejos y el azulejo se dibujará en la profundidad de la capa de la instancia que hace el dibujo, en la posición x/y que usted especifica en la habitación. Tenga en cuenta que también puede establecer el marco para dibujar también. Si el azulejo indexado es parte de un conjunto de animación entonces puedes establecer el número de fotograma aquí (desde 0 hasta el número de fotogramas - 1) y la baldosa indexada será simplemente el indicador de qué conjunto de baldosas animadas estás utilizando mientras que será el fotograma dado el que se dibuje. Si el azulejo no es parte de un conjunto animado, entonces el fotograma debe ser fijado en 0.

Sintaxis:

draw_tile(tile set, tiledata, frame, x, y)

Argumento Descripción
juego de baldosas El ID del conjunto de azulejos a utilizar
tiledata Los datos de los azulejos que se van a utilizar
marco El número de fotograma del azulejo animado que se va a utilizar (por defecto 0 para azulejos no animados)
x La posición x dentro de la habitación para dibujar
y La posición y dentro de la habitación para dibujar

Devuelve:

Real

Ejemplo:

var lay_id = layer_get_id("Tiles_sky");
var map_id = layer_tilemap_get_id(lay_id);
var data = tilemap_get(map_id, 0, 0);
tile_set_flip(data, true);
draw_tile(set_Wall, data, 0, x, y);

El código anterior obtendrá los datos de un azulejo del mapa de azulejos dado, luego establecerá la bandera "flip" y utilizará estos datos para dibujar el azulejo volteado en la posición de la instancia.
