draw_set_circle_precision

Para optimizar el rendimiento al dibujar círculos, GameMaker Studio 2 básicamente dibuja una forma poligonal con suficientes lados para que parezca circular. Sin embargo, dependiendo de lo grandes o pequeños que sean los círculos, es posible que cambiar este valor puede ayudar a aumentar el rendimiento de su juego, o hacer que los círculos se vean mejor. El valor de precisión que puede introducir en esta función es el número de lados que tiene el polígono del círculo, y este número puede ser cualquier cosa entre 4 y 64, pero debe ser un número divisible por 4, con un valor por defecto de 24.

NOTA: Si quieres dibujar una forma usando un shader, debes tener en cuenta que la mayoría de los shaders esperan las siguientes entradas: vértice, textura, color. Sin embargo, cuando se utiliza esta función, sólo se pasan datos de vértice y color, por lo que el puede no dibujar nada (o dibujar algo pero no correctamente). Si necesitas dibujar formas de esta manera, entonces el shader debe ser personalizado con esto en mente.

Sintaxis:

draw_set_circle_precision(precisión);

Argumento Descripción
precisión La precisión de todos los círculos dibujados después de llamar a esta función.

Devuelve:

N/A

Ejemplo:

draw_set_circle_precision(12);

El código anterior dibujará todos los círculos utilizando un polígono de 12 lados.
