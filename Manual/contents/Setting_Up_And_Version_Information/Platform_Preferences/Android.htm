Preferencias deAndroid

Las preferencias de Android preferencias se dividen en varias secciones que se enumeran a continuación:

En la parte superior de las preferencias se puede establecer el tamaño máximo del JDK Memory Heap (por defecto es de 2GB). A continuación tienes que configurar las diferentes rutas para que GameMaker Studio 2 sepa dónde buscar las diferentes herramientas necesarias para construir el paquete ejecutable final y probar tu juego. Esto se divide en tres secciones para el SDK de Android, el NDKde Android y el JDK de Java. Estas rutas deberían haberse rellenado automáticamente, pero si tienes algún problema puedes hacer clic en el botón de la derecha para abrir el explorador de archivos y seleccionar manualmente las ubicaciones de cada elemento. Puede encontrar detalles sobre las versiones necesarias para cada uno de los elementos del SDK aquí. Las rutas deben ser validadas a medida que las aplica y un mensaje mostrará "Encontrado" o le indicará cualquier error (en cuyo caso deberá revisar toda la información para asegurarse de que es correcta).

Debajo tiene las diferentes opciones de embalaje que son:

  • Hacer siempre una instalación completa del APK: Marcar esta opción hará que GameMaker Studio 2 elimine cualquier ejecución anterior de tu juego de tu dispositivo, así como todos los archivos asociados (por ejemplo, los archivos *.ini ) de tu dispositivo de prueba antes de instalar una nueva versión del juego.
  • Instalar en el paquete: Cuando esta opción está marcada, la compilación de un paquete final ejecutable *.apk también lo instalará automáticamente en el dispositivo de destino actualmente seleccionado.

Finalmente llegamos al KeyStore. Este es un archivo que se utilizará para "firmar" todas tus aplicaciones Android , así que rellena todos los datos correctamente y ten en cuenta que este archivo será necesario para crear y actualizar todas tus aplicaciones Android en el futuro, lo que significa que después de configurar se aconseja tomar una captura de pantalla de la configuración utilizada y hacer una copia de seguridad del archivo final. Si pierdes este archivo no podrás actualizar ningún juegos existentes que hayan sido subidos a la tienda.

Para crear un nuevo archivo de almacén de claves se necesitan los siguientes datos:

  • Nombre del archivo: Es el nombre del archivo del almacén de claves.

NOTA: Este no es su nombre o el nombre de la empresa, sino el nombre del archivo que se va a generar. Si ya tienes un archivo de keystore de proyectos anteriores, entonces puedes hacer clic en el botón del explorador de archivos aquí y dirigir GameMaker Studio 2 al archivo de keystore previamente creado (o hacer clic en el botón Importar ). En este caso, debe seguir rellenando el resto de la información requerida (exactamente igual que cuando generó el archivo por primera vez), pero no debe hacer clic en el botón Generar almacén de claves. Eso generará un nuevo archivo de almacén de claves, sobrescribiendo el anterior.

  • Nombre común: Este sería normalmente su nombre.
  • Contraseña: Su contraseña de seguridad para el archivo de almacén de claves, que debe tener al menos seis caracteres.
  • Alias: Este es otro nombre que se puede utilizar para el archivo de almacén de claves y debe ser diferente al nombre introducido anteriormente.
  • Contraseña de alias: Su contraseña de seguridad de alias para el archivo de almacén de claves, que debe tener al menos seis caracteres.
  • Unidad organizativa: El departamento de la empresa del que forma parte.
  • Organización: El nombre de su empresa.
  • Ubicación: El nombre del pueblo o ciudad donde se encuentra.
  • País: El código estándar de dos letras del país en el que se encuentra.

Una vez rellenada esta información y si estás satisfecho con ella, pulsa el botón marcado como Generar almacén de claves para GameMaker Studio 2 para crear el archivo de almacén de claves necesario. Esto puede tomar un momento, pero una vez que haya terminado, normalmente no no será necesario volver a modificarlo. Tenga en cuenta que puede hacer clic en el botón Importar para importar un archivo de almacén de claves creado previamente (véase la sección Nombre de archivo , más arriba) y también puede recuperar el hash SHA1 del almacén de claves haciendo clic en el botón Generar hash de claves . Al pulsar este botón se generarán dos claves hash de seguridad especiales vinculadas directamente a su archivo de almacén de claves que algunos sitios web requieren para su correcto funcionamiento (por ejemplo, Facebook, o Google Play para el hash Sha1). Si lo necesita, simplemente pulse el botón y copiar y pegar las claves hash generadas.
