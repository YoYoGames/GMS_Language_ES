Preferencias del editor de texto

Las preferencias de los editores de texto se utilizan para definir el aspecto de los diferentes editores de código/script. La página principal tiene las siguientes opciones:

  • Retraso en la comprobación de la sintaxis (ms): El tiempo de retardo (en milisegundos) antes de que se ejecute el comprobador de sintaxis. Este valor por defecto es de 2000ms.
  • Retraso en la finalización del código (ms): El tiempo de retardo (en milisegundos)
  • Abrir los scripts en el editor de pantalla completa: Si se marca esta opción, los recursos de los scripts se abrirán automáticamente en un nuevo espacio de trabajo. Si se desmarca, se abrirán en una ventana dentro del espacio de trabajo actual. Esta opción está desmarcada por por defecto.
  • Abrir los shaders en el editor de pantalla completa: Si se marca esta opción, los recursos de sombreado se abrirán automáticamente en un nuevo espacio de trabajo. Si se desmarca, se abrirán en una ventana dentro del espacio de trabajo actual. Esta opción está desmarcada por por defecto.
  • Abrir los eventos de objeto en el editor de pantalla completa: Si se marca esta opción, los editores de código de eventos de objeto se abrirán automáticamente en un nuevo espacio de trabajo. Si se desmarca, se abrirán en una ventana encadenada al editor de eventos de objeto. Esta opción está desmarcada por defecto.
  • Abrir scripts de eventos como pestañas en la misma ventana: Cuando se añade código o se arrastra y suelta a un evento, la ventana de código se encadena a la ventana del evento, y cuando esta opción está marcada cualquier otro evento que se abra para su edición se añadirá a la misma ventana como pestañas en la parte superior. Si se desmarca esta opción, cada evento se abrirá como su propia ventana encadenada. Esta opción está marcada por defecto.
  • Abrir scripts vinculados como pestañas en la misma ventana: Cuando tenga varios scripts vinculados y los abra, por defecto se agruparán en la misma ventana como pestañas. Si se desmarca esta opción, se abrirán cada uno en su propia ventana/espacio de trabajo. Esta opción está marcada por defecto.
  • Desplazar el editor de texto a la vista al enfocar: Cuando abra un editor de texto desde un evento de objeto, al seleccionar esta opción se desplazará automáticamente el espacio de trabajo para que el editor esté dentro de los límites. Esta opción está desactivada por defecto.
  • Anchura por defecto de la ventana de código: Establece la anchura por defecto (en píxeles) que tendrá la ventana de código cuando se abra. El valor por defecto es 900.
  • Altura de la ventana de código por defecto: Establece la altura por defecto (en píxeles) que tendrá la ventana de código al abrirse. El valor por defecto es 600.
  • Ancho mínimo de la ventana de código: Establece el ancho mínimo (en píxeles) que permite la ventana de código. El valor por defecto es 300.
  • Altura mínima de la ventana de código: Establece la altura mínima (en píxeles) que permite la ventana de código. El valor por defecto es 200.

Además de las opciones anteriores, también hay una subcategoría para las preferencias del editor de código:

Editor de códigos

La página web Las preferencias de color se utilizan para cambiar la forma en que el editor de código y de scripts muestra los diferentes textos. Cada entrada tiene un número de sub-entradas para permitirle definir exactamente cómo debe verse cada pieza de texto por separado, como se muestra en el ejemplo imagen de arriba. En general, puede establecer el color, la fuente, el tamaño y el estilo para cada una de las diferentes opciones. Más abajo también puede establecer un estilo más general para cosas como la selección de líneas de paréntesis de código, etc...

En las preferencias de color tiene las diferentes Opciones de estilo. Estas preferencias se utilizan para establecer el comportamiento del editor de código/script, con las siguientes opciones disponibles:

  • Mantener tabuladores: Si está marcada, mantendrá las pulsaciones de las teclas de tabulación como caracteres de tabulación - ' \t ' - en el editor, mientras que si se desmarca significa que los tabuladores se convierten en espacios. Por defecto está desactivado.
  • Tamaño del tabulador: El número de espacios de caracteres que al pulsar "Tab" debe sangrar el código. Por defecto es 4.
  • Mostrar espacio en blanco: Mostrar " . " para mostrar los espacios en blanco (véase la imagen siguiente). Por defecto está desactivado.
  • Mostrar números de línea: Mostrar o no los números de línea abajo a la izquierda. Por defecto está activado.
  • Mostrar líneas de pestañas: Mostrar " .... " para mostrar el espacio de tabulación. Por defecto está desactivado.
  • Activar el código de colores: Activa o desactiva la codificación de colores para el editor. Si está desactivado, el código no será estilizado, mientras que si está activado, se estilizará utilizando el formato dado anteriormente en la sección Color. Por defecto está activado.
  • Límite de líneas del documento: Establece el número máximo de líneas para cualquier documento del editor. El valor por defecto es 0 (0 o menos es esencialmente infinito) y cualquier otro valor limitará las líneas permitidas.
  • Coincidir con la sangría en las nuevas líneas: Activa o desactiva la sangría automática para tu código. Por defecto está activada, lo que mantendrá cualquier tabulación establecida en una línea cuando se añada una nueva línea.
  • Auto sangría después de las llaves: Activar o desactivar la sangría automática para su código al añadir llaves {} . El valor predeterminado es "on", que tabulará la siguiente línea de código cuando se añada una nueva llave { y una nueva línea.
  • Habilitar la finalización del código: Activa o desactiva la ventana de finalización de código. Cuando se escribe código y esto está habilitado, obtendrá una ventana emergente que muestra las posibles funciones que puede utilizar basándose en el texto de entrada actual. Por defecto está activado.
  • Añadir automáticamente paréntesis de función: Activar o desactivar la adición automática de paréntesis () cuando se utiliza el autocompletado. Cuando se escribe código y esto está activado, el IDE añadirá automáticamente los dos paréntesis () que requiere una función, colocando el cursor dentro de ellos si la función toma argumentos, o colocando el cursor después de ellos si no los toma. Esta característica sólo funciona cuando la función ha sido seleccionada desde el autocompletado. Por defecto está activada.
  • Límite de la pila de deshacer/rehacer: Aquí puede establecer el límite de la pila de deshacer/rehacer. Cualquier cosa por debajo de 0 es esencialmente infinita (limitada por la memoria disponible) mientras que los valores positivos limitarán la pila a ese número, eliminando cualquier acción en la cola de la pila si se alcanza el límite. El valor por defecto es -1.
  • Resaltar coincidencias de selección: Esto resaltará ligeramente cualquier sección de código que esté duplicada en otra parte. Así, si selecciona una función, por ejemplo, todas las instancias de esta función en la ventana de código actual serán resaltadas también. Por defecto está activado.
  • Codificación por colores/completar código para funciones obsoletas: Esto resaltará cualquier sección de código que sea obsoleta. Por defecto está activado.
