Código de edición

Cuando empiece a utilizar el editor de scripts, sólo tendrá que escribir el código que desee para el activo de script o para el evento, siguiendo la sintaxis GML correcta . Mientras haces esto, obtendrá ayuda de la ventana de autocompletar:

Esta ventana aparecerá mientras está escribiendo su código (normalmente después de una ligera pausa, pero esto puede configurarse en las Preferencias) y le dará una selección de las respuestas "más adecuadas" en orden alfabético.

Al utilizar el editor de código, puedes hacer clic en y arrastrar el ratón para seleccionar una fila de código o mantener pulsado y luego hacer clic en y arrastrar para seleccionar columnas de código. Todo lo que se seleccione de este modo puede copiarse, cortarse y pegarse utilizando los atajos de teclado estándar de / + " C", / + "X" y / + "V".

Vale la pena señalar que puedes comentar tu código de varias maneras. Comentar su código es muy importante para mantener la productividad, ya que mantiene las cosas claras y separa las secciones de código de tal manera que es obvio lo que cada lo que hace cada parte y por qué está ahí. También significa que, si trabajas en equipo, puedes dejar notas para otros miembros y que ellos sepan exactamente lo que has hecho o para qué sirve una sección específica del código. Incluso si eres un desarrollador en solitario, dejar comentarios en tu código es importante, especialmente si tienes que dejar el proyecto y volver a él más adelante.

Comentar tu código es sencillo, ya que sólo tienes que utilizar "//" antes de cualquier línea de texto o puedes comentar párrafos enteros utilizando "/* text here */". Incluso hay dos atajos de teclado para comentar y descomentar las líneas de código seleccionadas: / + "K" para comentar, y / + + "K" para des-comentar. Los scripts también pueden tener ciertos comentarios de estilo JSDoc para definir las propiedades de las funciones que contienen, de modo que cuando se utilicen las funciones en su código aparezcan en autocompletar junto con sus argumentos y otros detalles. Puedes encontrar más información sobre esto en la sección Comentarios de Script JSDoc para más información.

Por defecto, todo el código que se escriba tendrá iconos de plegado de código junto a cada conjunto de llaves coincidentes {}. Estos tienen la forma de unos pequeños iconos de más + o menos -, y al hacer clic en ellos se "doblará" el código dentro de las llaves correspondientes para que ocupe menos espacio visual.

También puedes establecer manualmente regiones de plegado de código utilizando las etiquetas especiales #region para marcar una sección de código. El funcionamiento es el siguiente: añades una etiqueta #region (junto con cualquier texto de comentario) en la parte superior de una sección de código y luego añades la correspondiente etiqueta en la parte inferior de la sección. una sección de código y luego añades la correspondiente etiqueta #endregion en la parte inferior de la sección y entonces puedes "plegar" el código, es decir, ocultarlo para poder reducir el número de líneas de código visibles en el editor en cualquier momento:

El código real es algo así:

#región Esta es una región de ejemplo

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Comente más aquí si es necesario

Como habrás notado, algunas partes del texto del script están coloreadas. El editor de scripts "conoce" los objetos existentes, las variables y funciones incorporadas, etc... y los codifica con colores. La codificación por colores ayuda mucho a evitar errores ya que puedes ver inmediatamente si has escrito mal algún nombre o has usado una palabra clave como variable, o tienes un error de sintaxis. Sin embargo, si no te gusta la codificación por colores, en las Preferencias puedes activarla y desactivarla (también puedes usar F10 para activarla o desactivarla en el editor), así como personalizar los colores que se utilizan.

También puede hacer clic con el botón derecho del ratón en en cualquier lugar del editor para abrir el siguiente menú:

Aparte de las funciones normales que esperarías para cortar, copiar y pegar líneas de código, también tienes la opción de comentar o des-comentar cualquier texto seleccionado, añadir o eliminar sangrías, así como dividir la ventana de código en 1, 2 o 3 paneles separados (ten en cuenta que las opciones relacionadas con #regions no estarán disponibles si no has definido ninguna en el código que se está editando).

La última opción para dividir en paneles dividirá la ventana actual en diferentes partes en las que podrá arrastrar un script o evento y hacer que se muestre junto a los demás para compararlos y editarlos.

Esto es excepcionalmente útil para comparar dos scripts o para ver scripts conectados al mismo tiempo, sin necesidad de otra ventana o tener que cambiar constantemente de pestaña.

Las otras opciones en este menú son para activar un punto de interrupción para su uso con el depurador, y para convertir el código GML en DnD™o viceversa. Tenga en cuenta que si convierte el código a DnD™, entonces no creará acciones individuales para cada línea de código, sino que utilizará una única acción de código para mantener todo lo que estaba previamente en el evento o script.
