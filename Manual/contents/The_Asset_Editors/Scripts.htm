El editor de guiones

Esta sección trata del editor de scripts (también llamado editor de texto) que es donde se escribe todo el código que su juego requerirá para ejecutarse. Este editor se utiliza para crear activos de script personalizados, así como para codificar eventos de objetos, para añadir código de creación de salas, y para muchas otras cosas, y como tal, esta sección simplemente detalla cómo funciona el editor de scripts, y no cómo crear activos de scripts ni cómo usar Drag and Drop™para hacer scripts. Para eso, por favor, vea las siguientes secciones:

  • GuionesGML
  • Guiones deDnD™

El editor de scripts es el lugar donde se escribe el código de GameMaker Language, que es el lenguaje de programación incorporado que utiliza GameMaker Studio 2 (consulte la sección de ReferenciaGML para más detalles). Una vez que te familiarices con GameMaker Studio 2 y quieras utilizarlo en toda su extensión, es aconsejable que empieces a aprender a utilizar este lenguaje, ya que amplía enormemente tus posibilidades a la hora de crear juegos.

Un activo de script puede ser nombrado haciendo clic con el botón derecho del ratón en en el Navegador de Activos y seleccionando Renombrar, pero tenga en cuenta que el nombre del script debe ajustarse a las reglas de scripting para las funciones, por lo que sólo pueden ser alfanuméricos, deben comenzar con una letra, y el único símbolo que pueden contener es el símbolo de la barra inferior "_". El editor de scripts en otros lugares no puede ser renombrado y tendrá un nombre específico para lo que se está editando, como un evento de objeto por ejemplo.

Cuando abra la ventana del editor de scripts, tendrá las siguientes opciones y diseño:

Pestaña Script / Evento

El editor de scripts se abre en una ventana con pestañas en la parte superior para permitirte tener varios scripts o eventos en una sola ventana (aunque este comportamiento puede cambiarse desde las Preferencias para dar una nueva ventana a cada script o evento). Puede hacer clic en en una pestaña y luego arrastrarla para cambiar el orden de las pestañas, o -si lo prefiere- puede sacarla de la ventana actual y colocarla en el espacio de trabajo para crear una nueva ventana para ese script (o añadirla a una ventana diferente), y también puedes maximizar el editor de scripts para crear un nuevo espacio de trabajo. Si arrastra una pestaña de script fuera de la ventana del IDE, entonces se generará un nuevo IDE para mantener este recurso de script, y puede ser utilizado como lo haría con la ventana principal.

Tenga en cuenta que si está editando código desde un evento de objeto en el editor de scripts y ha maximizado el editor de scripts o lo tiene en una ventana o espacio de trabajo separado, entonces el menú del editor del botón derecho tendrá algunas opciones extra:

  • Restaurar al espacio de trabajo: Esto sacará al editor de scripts de un estado maximizado/espacio de trabajo y lo volverá a encadenar al objeto en el espacio de trabajo.
  • Ir al objeto: Esto le llevará al espacio de trabajo en el que se encuentra el objeto con el código y se centrará en el objeto.
  • Añadir /Abrir Evento: Esto le permite añadir un nuevo evento al objeto al que pertenece el script actual, y abrirá una nueva pestaña de código en el Editor de Script para el evento añadido. Si el evento seleccionado ya tiene código, éste se abrirá en una nueva pestaña. se abrirá en una nueva pestaña.

Para obtener información sobre el resto de las opciones del menú con el botón derecho del ratón, consulte la sección sobre Edición, más adelante.

Canalón

El canalón también marcará cualquier línea del script a la que se le haya añadido un punto de interrupción . Un punto de interrupción es simplemente un lugar en el script donde quieres que el depurador detenga la ejecución de tu juego cuando se alcanza. Puedes activar un punto de interrupción desde cualquier línea de cualquier script o evento de objeto pulsando "f9" o utilizando el menú del botón derecho . Cuando se activa esta opción, la función de búsqueda sólo resaltará aquellas cadenas que coincidan con la cadena de entrada completa. Por ejemplo, con la función desactivada, la búsqueda de "random" mostrará todas las palabras que contengan esta cadena -como irandom(), o randomise() - mientras que si se activa sólo se mostrará la función random().
Cuando esta opción está activada, le está diciendo a GameMaker Studio 2 que compruebe no sólo el contenido de la cadena de búsqueda, sino también el caso. Por ejemplo, si tienes un sprite llamado "spr_Dog" y haces una búsqueda de " perro" con esta opción desactivada, entonces la cadena del sprite será resaltada, sin embargo si la opción está activada entonces no lo hará ya que " Perro" ya no se considera lo mismo que " perro".

Editor de código principal

El editor de código principal es donde escribirá todo su código para crear el activo de la secuencia de comandos o llenar un evento de objeto. La edición real de su código se cubre en la sección de Edición más adelante en esta página.

Información

La barra de información en la parte inferior del IDE muestra el número de línea en el que te encuentras y la posición en la línea. También es donde puede ver el ayudante de código, que es una línea de texto que muestra la función que está editando actualmente junto con junto con los argumentos que requiere. A medida que rellenes la función en el editor, los argumentos se resaltarán para mostrarte cuál estás editando en ese momento.

Tenga en cuenta que cualquier argumento opcional en sus funciones de script se mostrará con corchetes [] alrededor de ellos. Si ha utilizado los comentarios de script de JSDoc dentro de un activo de script, la información que ha suministrado también se mostrará aquí.
