Sistema

Este evento sólo puede ser activado por una llamada de retorno de un evento a nivel de sistema (como la detección de un gamepad o el inicio de sesión automático en XBox Live) y devolverá un DS Map almacenado en la variable async_load, que contiene diferentes pares de claves/valores dependiendo del evento de nivel de sistema que desencadenó la devolución de llamada. Estos eventos son:

Tapetes de juego

Cuando este evento se dispara para un gamepad que se conecta o desconecta, devolverá uno de los siguientes pares clave/valor en el mapa async_load:

  • "event_type" - el tipo de evento del sistema recibido, que será una de las siguientes cadenas:
    • "gamepad discovered" - ocurre cuando el sistema informa que se ha conectado un nuevo gamepad
    • "gamepad lost" - ocurre cuando el sistema ha perdido la conexión con un gamepad
  • "pad_index" - el índice de la almohadilla que se ha añadido o eliminado

Este evento le permite ahora mover toda su lógica de comprobación del gamepad desde el Evento de Paso o un evento de Alarma al Evento de Sistema y sólo ejecutarlo cuando sea realmente necesario.

IMPORTANTE! Este evento NO se activará a menos que tengas al menos una gamepad_* función o acciónDnD™ en tu código de juego. El corredor sólo inicializará el subsistema de gamepads cuando las funciones se utilicen en el proyecto, por lo que si no están presentes, añadir/quitar gamepads no activará el evento del sistema.

Teclados virtuales

Cuando este evento se activa para un teclado virtual que se abre o se cierra, devolverá los siguientes pares clave/valor en el mapa async_load:

  • "event_type" - el tipo de evento del sistema recibido, que será "virtual keyboard status" para los teclados virtuales.
  • "screen_height" - la altura del teclado virtual (en píxeles). Será 0 si el teclado es invisible.
  • "keyboard_status" - el estado actual del teclado, devuelto como una de las siguientes cadenas:
    • "oculto"
    • "oculto"
    • "mostrando"
    • "visible"

Consulte aquí para obtener más información sobre el teclado virtual.

XBox Live

El evento asíncrono del sistema puede activarse cuando se dirige a la XBox One utilizando la exportación UWP y marcando la opción Habilitar XBox Live en las Opciones de juego UWP. Cuando se inicia GameMaker Studio 2 que tenga activado Xbox Live, el proyecto intentará automáticamente iniciar sesión en Xbox Live de forma silenciosa. Los resultados de este intento de inicio de sesión se devolverán como uno de los siguientes pares clave/valor en el mapa async_load:

  • "event_type" - el tipo de evento del sistema recibido, que será una de las siguientes cadenas:
  • "user signed in" - el inicio de sesión silencioso del usuario se ha completado con éxito
  • "user sign in failed" - el inicio de sesión silencioso del usuario ha fallado (cuando esto sucede, puede utilizar la función xboxlive_show_account_picker() para para que el usuario elija una cuenta con la que iniciar sesión)
  • "user signed out" - el usuario ha cerrado la sesión

Otras funciones para XBox Live también pueden activar este evento, pero las diferentes devoluciones de llamada para esas funciones se detallan en las páginas de las funciones correspondientes. Para más información sobre las funciones específicas disponibles para XBox Live, consulta aquí.

Estado del sistema de audio

Este tipo de evento se utiliza específicamente cuando se trabaja con el objetivo HTML5, ya que se activará cada vez que el estado del contexto de Web Audio cambie. Esto significa que si tienes, por ejemplo, una pieza de música de fondo en bucle, entonces puedes pausarla o detenerla y reiniciarla, basándose en la activación de este evento. Esto se puede comprobar buscando el siguiente par clave/valor en el mapa async_load:

  • "event_type" - el tipo de evento del sistema recibido, que será la cadena "audio_system_status" si el sistema de audio se ha inicializado o el contexto ha cambiado.

Si se activa este tipo de evento, habrá una clave adicional en el mapa async_load:

  • "status" - El estado del sistema de audio, que será una de las dos cadenas siguientes
  • "available" - El sistema de audio está inicializado y disponible para reproducir sonidos
  • "unavailable" - El sistema de audio no se ha inicializado, o el contexto no se está ejecutando actualmente, por lo que no puede reproducir sonidos (todas las funciones de reproducción de sonido devolverán -1)

Para obtener más información sobre las funciones específicas disponibles para Audio, consulte aquí.

NOTA: Aunque este evento está diseñado para su uso con el objetivo de HTML5, también se activará en todas las demás plataformas, pero en todo (excepto en HTML5) sólo se activará una vez en el inicio del juego cuando el motor de audio se inicialice por primera vez.
