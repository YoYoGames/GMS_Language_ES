El editor de extensiones

Un paquete de extensión puede utilizarse para añadir una serie de funciones y constantesadicionales al Lenguaje GameMaker(GML), o como una forma de transferir activos del juego o incluso proyectos enteros fácilmente entre personas. Si la extensión es para añadir funcionalidad extra, entonces las funciones utilizadas pueden enlazar tu juego con SDKsexternos o código fuente escrito en otros lenguajes (como Java para Android, por ejemplo), así como GML nativo.

Normalmente las extensiones de terceros serán de la GameMaker Mercado y aquellas a las que se haya suscrito serán listadas en la ventana de la Bibliotecadel Mercado y pueden ser instaladas desde allí. Si no ha obtenido su paquete de extensiones del Mercado, aún puede añadirlo a un proyecto haciendo de nuevo haciendo clic con el botón derecho del ratón en en cualquier lugar del navegador de activos y seleccionando Añadir existente, y luego navegando hasta la ubicación de la extensión para seleccionarla.

Si tienes un *.GMEZ creado con una versión anterior de GameMaker Studio puedes añadirlos seleccionando la opción de menú Importar extensión del botón derecho del ratón .

NOTA: Las extensiones del formato antiguo *.GEX ya no son compatibles y no pueden importarse a los proyectos de GameMaker Studio 2.

Una vez que haya añadido una extensión al Navegador de Activos, deberá darle un nombre y un número de versión y también establecer a qué plataformas desea exportarla. Desmarcando las plataformas seleccionadas, puede indicar a GameMaker Studio 2 que no exporte la extensión como parte del ejecutable para esas plataformas. También debes asegurarte de que las propiedades de la extensión coinciden con la(s) plataforma(s) de destino, y editar los objetivos de exportación de los archivos de extensión basándote en lo siguiente:

  • Todos los objetivos: Extensiones GML. Son extensiones realizadas con el lenguaje GameMaker(GML) y son compatibles con todas las plataformas de destino.
  • Windows(Estándar, UWP, XBoxOne): Puedes utilizar los archivos de windows DLL (Dynamic Link Libraries). Tenga en cuenta que si ha marcado la opción "Usar tiempo de ejecución de Windows x64 " en las opciones de juego de Windows , entonces todas las DLL del proyecto deben ser compatibles con 64 bits. Las DLL de 32 bits no funcionarán con el ejecutable de 64 bits.
  • macOS: El objetivo de macOS puede utilizar los archivos estándar DyLib para las extensiones.
  • Ubuntu (Linux): Para el objetivo de Ubuntu es necesario asegurarse de que su extensión utiliza archivos SO.
  • HTML5: Estas plataformas pueden utilizar las extensiones de JavaScript.
  • Android: Esta plataforma requiere que se escriba una extensión en Java o un SDK de Android Framework.
  • iOS / tvOS: Las extensiones para esta plataforma están escritas en Objective C y requieren un *.mm y un archivo *.h, o si se utiliza algún Frameworks, un archivo nativo precompilado *.a.
  • PlayStation: Para PlayStation 3 o 4 sus extensiones deben ser archivos PRX, y cada objetivo requiere un PRX diferente para ajustarse a la especificación del objetivo (ya sea PS3 o PS4). Si trabajas en una PSVita, la extensión debe ser SUPRX.

NOTA: Algunas extensiones deben seguir unas convenciones de nomenclatura específicas para que GameMaker identifique correctamente la plataforma y la arquitectura de la CPU para la que están diseñadas. Consulte la sección sobre la creación de extensiones para obtener más información.

Dado que una única extensión puede tener varios archivos asociados, es posible crear una única extensión para múltiples plataformas, en cuyo caso deberá especificar todas las plataformas compatibles en el editor principal de extensiones (es decir, marcar la casilla de verificación junto a ellas) y luego, en las propiedades de los archivos individuales de cada parte de la extensión, establecer la plataforma a la que deben exportarse esos archivos.

Para cambiar la plataforma de destino de un archivo de extensión o marcador de posición específico, primero debe abrir las propiedades de la extensión (haga doble clic en la extensión en el navegador de activos o utilice y seleccione Abrir propiedades), y luego, una vez en las propiedades de la extensión, seleccione el archivo de recursos de la derecha que desee editar:

NOTA: Si va a exportar al destino macOS o Ubuntu, entonces puede establecer la opción "Descomprimir como archivo zip" para conservar los permisos de archivo de la extensión.

Una vez que haya instalado su extensión, puede utilizar las funciones y constantes de la misma como lo haría con las funciones y constantes incorporadas en el Lenguaje GameMaker. Si su juego es multiplataforma (por ejemplo, está diseñado para ejecutarse tanto en Windows y Android ), entonces debe utilizar las constantes y funciones del sistema operativo para comprobar la plataforma antes de llamar a las funciones específicas de la extensión para evitar problemas, por ejemplo:

switch(os_type)
{
case os_android:
call_android_extension_function(x, y, z);
break;
case os_windows:
call_windows_extension_function(x, y, z);
break;
}

Tenga en cuenta que para las diferentes plataformas sus extensiones pueden compartir los nombres de las funciones, por lo que si tiene una extensión con archivos distintos para (por ejemplo) hacer aparecer un mensaje para Windows y para Android, ambos archivos de extensión pueden compartir el mismo nombre de función siempre que siempre que ambos estén configurados para exportar sólo a sus respectivas plataformas.

Extensiones de terceros deYoYo Games

Las versiones anteriores de GameMaker tenían mucha funcionalidad de terceros incorporada, pero esto era ineficiente y difícil de mantener, ya que si alguna funcionalidad específica era cambiada por el tercero, entonces todo el IDE de GameMaker y Tiempo de ejecución tendría que ser actualizado. Así, las versiones actuales han adoptado el uso de paquetes de extensiones para añadir soporte de terceros a sus juegos. Esto significa que puedes añadir sólo aquellas funciones extra que necesites y que incluso puedas adaptarlas y editarlas para que se ajusten a tus necesidades específicas.

La mayoría de estas extensiones se descargarán e instalarán por ti cuando selecciones una opción de juego específica (como Facebook o Google Mobile Ads, por ejemplo), pero puedes encontrar más información sobre ellas en la sección correspondiente a cada plataforma en la Base de conocimientos de YoYo Games:

  • Plataformas y configuración deGameMaker Studio 2

Puede encontrar las propias extensiones en la siguiente página del Marketplace:

  • Activos del mercado deYoYo Games

Tenga en cuenta que estas extensiones vienen con una demostración y un archivo de ayuda (en formato *.pdf ) que explica cómo utilizarlas, por lo que sus funciones no están cubiertas en el manual.
