Configuración del plugin GIT

Al empezar a utilizar el plugin de GIT, tenemos que configurar una identidad para que GIT haga commit como, lo que significa que tenemos que ir a la sección Plugins - Source Control (Git ) de las Preferencias y añadir los datos de autentificación.

Esta identidad de autenticación se utilizará para todos los proyectos futuros, y GameMaker Studio 2 le ofrece dos formas diferentes de configurarla de configurarla:

  • Nombre de usuario y contraseña: Al hacer clic en el botón "Añadirnuevo usuario/contraseña de autenticación", aparecerá la siguiente ventana en la que podrá introducir la URL del repositorio , el nombre de usuario y la contraseña. la contraseña:
    Si quiere que esto afecte sólo a un repositorio específico, coloque la URL del repositorio en el campo superior. Sin embargo, si es autenticación en todo el dominio que desea (por ejemplo: "bitbucket.org" o "github.com" ) entonces sólo coloque el dominio y nada más. Cuando algo necesita comprobar la autenticación, buscará primero una coincidencia con un repositorio específico, y luego una coincidencia con el dominio. Esto le permitirá tener una autenticación por defecto para un dominio, y anularla con detalles específicos para ciertos repositorios más tarde.

NOTA: Si está utilizando un repositorio externo, entonces debería utilizar el nombre de usuario y la contraseña asociados a la cuenta configurada para ese repositorio, pero si quiere utilizar un repositorio local, puede utilizar cualquier nombre de usuario y contraseña.

  • Pares declaves SSH: Al hacer clic en "Añadirnueva autenticación de pares de claves SSH" se le presentará la siguiente ventana en la que puede introducir la URL del repositorio y la contraseña antes de dar las rutas de acceso a los archivos de Clave Pública y Clave Privada requeridos:
    Al igual que las autenticaciones de Nombre de Usuario/Contraseña, puede tener autenticaciones específicas de dominio y de repositorio cuando utilice pares de claves SSH. Si ya tiene un SSH Keypair generado, puede configurar las rutas aquí, así como cualquier contraseña requerida para acceder a ellas. Tenga en cuenta que si necesita un acceso rápido a la clave pública, el botón de copia copiará automáticamente el contenido al portapapeles por usted. Si aún no tiene un par de claves SSH, al hacer clic en el botón Create KeyPair aparecerá un cuadro de diálogo de archivo donde colocar la clave privada, y la clave pública se creará junto a ella - esto rellenará las rutas de las claves pública y privada automáticamente por ti.

Estos ajustes se utilizarán ahora para comunicarse con el repositorio (IE: Git Hub, Bit Bucket, etc.) cuando se manejen solicitudes de clonación, push y pull. Sin embargo, todavía necesita configurar las opciones por proyecto, lo que puede hacerse abriendo las Opciones Principales y haciendo clic en la opción Activar Control de Fuentes. Esto activará el SCM para el proyecto actual.

NOTA: Si necesita volver a editar los detalles de autenticación, simplemente haga doble clic en la entrada y se abrirá la ventana de detalles para que pueda editarlos, pero no podrá cambiar el nombre de la URL. Si necesita cambiar la URL, tendrá que eliminar los datos de autenticación y volver a añadirlos. Para eliminar los datos de autenticación, haga clic en el botón de cierre situado a la izquierda de la fila en la ventana principal de Preferencias.
