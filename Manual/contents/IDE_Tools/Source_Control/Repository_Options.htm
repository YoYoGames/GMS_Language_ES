Opciones de repositorio

Una vez configuradas las credenciales de GIT y habilitado el control de fuentes para su proyecto, ahora necesita vincular el IDE de GameMaker Studio 2 a un repositorio, lo que se hace de una de las tres maneras siguientes:

  • Crear un repositorio de proyectos
  • Empujar el proyecto actual a un repositorio externo
  • Clonar un repositorio existente

En el primer caso, la creación de un repositorio del proyecto, esto creará un repositorio GIT directamente donde está su proyecto. En el lenguaje de GIT este es un "repositorio no desnudo", por lo que Push/Pull no tiene no tiene sentido en este caso, pero aún puedes confirmar y hacer un seguimiento de los cambios del proyecto, así como revertir los cambios y las revisiones si es necesario. Para configurar esto, sólo tienes que hacer clic en Control de Fuentes > Crear Repositorio de Proyecto en el menú principal.

Sin embargo, es posible que ya tenga un proyecto existente y quiera ponerlo en un repositorio externo para seguirlo y compartirlo entre su equipo. Esto requiere que un repositorio externo ya haya sido configurado, así como el nombre de usuario y la contraseña correctos establecidos en sus preferencias de identidad para acceder a él. Finalmente, como una función de conveniencia, puedes clonar un repositorio externo a través del IDE. De nuevo, necesitará el nombre de usuario y la contraseña del repositorio al que se va a conectar, configurados en sus preferencias de identidad, y entonces es sólo tiene que hacer clic en Control de Fuentes > Clonar Repositorio. Una vez hecho el clon, se abrirá automáticamente un explorador de archivos para que puedas abrir el proyecto si lo deseas.
