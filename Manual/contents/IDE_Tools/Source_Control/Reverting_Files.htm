Revertir archivos

En las secciones anteriores has visto cómo crear un repositorio y confirmar cosas en él, pero ¿qué pasa si cometes un error y quieres "volver" a un commit anterior? anterior? Vamos a cometer un error deliberado y ver cómo podemos tratar con él entonces...

En primer lugar, cree un nuevo recurso - en este ejemplo haremos un recurso Script - e inicialmente no verá ningún icono de estado, ya que en lo que respecta al control de origen aún no existe. En el recurso Script añade una simple línea de código, como

show_debug_message("Hola Mundo");

Si ahora cierra el editor de código, el script se guardará y el icono del archivo modificado aparecerá junto a él en el Navegador de Activos. Ahora, abra la ventana de confirmación y verá que hay cuatro cambios por etapas, ya que los nuevos recursos se añaden automáticamente:

Los cambios que se han puesto en marcha serán:

  • el archivo del proyecto .yyp
  • un archivo de vista
  • el propio script GML
  • el archivo .yy que hace referencia a él

Debes escribir un mensaje de confirmación y luego hacer clic en el botón Confirmar , tal y como hemos explicado anteriormente. Ahora, volvemos a nuestro script y cambiamos la línea de código a (por ejemplo):

show_debug_message("Hola mundo, ¿cómo estás?");

Cuando cerremos el editor de código, de nuevo aparecerá un icono rojo junto al script, pero nos hemos dado cuenta de que hemos cometido un error con nuestro código y queremos volver a una versión anterior para arreglarlo (obviamente, podrías abrir el script y arreglarlo fácilmente, ya que el ejemplo es muy simple, pero en proyectos grandes eso no es normalmente un enfoque factible). Lo que tenemos que hacer es revertir nuestros cambios.

Para revertir un cambio primero tenemos que abrir la ventana de confirmación de nuevo, así que vamos a Source Control > Commit Changes una vez más. Nuestro archivo modificado volverá a estar preparado para la confirmación, pero esto no es lo que queremos así que hacemos clic en el botón Unstage All para sacarlo del área de preparación.

Si ahora hacemos clic con el botón derecho sobre el archivo que hemos cambiado obtendremos un menú contextual con la opción de Revertir ruta:

Al seleccionar esta opción, se revertirán los archivos al estado anterior en el que se encontraban.

IMPORTANTE! Cuando revierta recibirá un mensaje pidiéndole que recargue o guarde el proyecto actual debido a que el comprobador de archivos ha detectado cambios en el disco:
Debe pulsar Recargar y no guardar, ya que al guardar se guardará lo que hay en la memoria en lugar de recargar los archivos revertidos.

Todo esto está muy bien para cuando editas algo y quieres volver a cambiar sin haber realizado una confirmación, pero ¿qué pasa cuando has confirmado uno o más cambios y quieres volver a una confirmación anterior ? Bueno, vamos a editar nuestro script de nuevo para decir:

show_debug_message("Esto es una mala idea");

Ahora lo confirmamos como antes para tenerlo bajo control de fuentes. Entonces, esto fue una mala idea y queremos retroceder, lo que significa que para empezar necesitamos abrir la ventana de Source Control > View History. Aquí veremos una lista de todos los commits anteriores ahora en esta ventana, empezando por el commit inicial, el commit para el script, y luego el commit para el script que hemos editado como "mala idea":

Ahora tenemos dos opciones. Podemos revertir una revisión entera - que revertirá todos los archivos que fueron cambiados a los de la confirmación elegida - o podemos elegir revertir una ruta - que simplemente revertirá un solo archivo. Vamos a hacer la primera opción para empezar:

  • Haga clic con el botón derecho en nuestra segunda confirmación en el panel del historial de confirmaciones (la confirmación "Hello World").
  • Seleccione Revertir a esta revisión y luego elija ReLoad en el mensaje de advertencia.

Debería ver que el script tiene ahora un icono de estado rojo , y si lo abrimos, tendremos lo siguiente:

Podemos entonces hacer un commit en el proyecto y volveremos a estar como antes. Sin embargo, también podemos deshacer esa acción para el archivo y recuperar nuestro commit de "mala idea" haciendo lo siguiente:

  • Abra la ventana Confirmar cambios.
  • Desmonta el script con el código "Hello World" en él.
  • Haga clic con el botón derecho y revertir la ruta.

Ahora volvemos a tener el archivo erróneo en el proyecto. Bien, podemos aprovechar esta oportunidad para revertir sólo esa ruta de archivo en lugar de revertir todo a una revisión anterior. Para ello tenemos que hacer lo siguiente:

  • Abrir la ventana de ver el historial.
  • Haz clic en la segunda confirmación (la de "Hola Mundo").
  • Haga clic con el botón derecho en nuestro script GML en el panel de Archivos y seleccione Revertir Ruta.

Deberíamos volver a tener nuestro código "Hola Mundo" y podemos entonces hacer un nuevo commit para actualizar el control de fuentes de nuevo con el archivo corregido.
