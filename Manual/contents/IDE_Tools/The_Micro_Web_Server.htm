El microservidor web

La función principal de esta pequeña herramienta es conectar y servir archivos a algunas de las plataformas de destino disponibles para GameMaker Studio 2 (como el objetivo HTML5, por ejemplo). Cuando se ejecuta un juego en un objetivo que requiere el Micro Web Server obtendrá una nueva ventana que se abre mientras el juego se sirve en el dispositivo elegido:

NOTA: La primera vez que ejecute un juego en un objetivo que requiera el Micro Web Server es posible que reciba una alerta de firewall. Debe permitir que el servidor web se comunique a través del cortafuegos si esto ocurre.

El Servidor Web mostrará una cierta cantidad de información de depuración relacionada con los archivos que se han servido, y tiene un botón para borrar esto en la parte inferior. Al lado del botón "Clear" está la ruta del archivo JS raíz del proyecto, mientras que debajo que tiene la opción de velocidad de la red. Esta opción está configurada como "Sin límite" por defecto, pero al hacer clic en ella se abrirá una lista de diferentes opciones de red:

Al cambiar esta configuración, también cambiará la opción del tamaño del paquete a la derecha. También puede editar esto manualmente, en cuyo caso la velocidad de la red se ajustará a "Personalizada". Si lo ajustas a cualquier otro valor que no sea el máximo, se reducirá la velocidad de transferencia de la red y te permitirá probar los juegos en un entorno con poco ancho de banda.

También puede ver la lista de IPs permitidas junto con el número de puerto que se está utilizando, pero estos no pueden ser editados en el Micro Web Server y deben ser establecidos en su lugar en la configuración del Administrador de Dispos itivos para el objetivo del dispositivo dado.
