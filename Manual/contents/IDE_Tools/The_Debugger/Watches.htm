Relojes

La idea del depurador es permitirle "observar" cada detalle de su juego. Esto se hace utilizando "ventanas de observación" que, por defecto, se abren en la ventana de salida anclada en la parte inferior del IDE. Puedes hacer clic en cada una de las diferentes pestañas para ver los relojes que contienen, algunos de los cuales pueden actualizarse en tiempo real (cuando se activa la depuración en tiempo real desde la barra de herramientas del depurador) mientras que otros requerirán que el depurador haya pausado el juego.

Cuando usted está viendo las variables en cualquiera de las ventanas, puede ser uno de los múltiples tipos de datos. Sin embargo, debido a la forma en que GameMaker Studio 2 almacena ciertas cosas, el depurador no siempre sabe exactamente lo que la variable. Esto se debe a que los valores de ID internos para las estructuras de datos o instancias son enteros y por lo tanto el depurador no siempre sabe si el valor entero que se almacena en una variable es para una estructura de datos o una instancia o es es simplemente un valor entero. Por esa razón, puedes hacer clic con el botón derecho del ratón en sobre cualquier valor de la variable y obtener una ventana emergente que enumera los tipos de datos disponibles que esta variable podría ser:

Las cadenas y los arrays no permiten esta opción, y cabe destacar que aunque una estructura de datos se almacene en una variable de instancia, su número de identificación es global, por lo que si tienes cuatro instancias y cada una crea una ds_list (por ejemplo), cada lista tendrá un ID de 0 a 3, y no todos serán 0.

Una vez que haya seleccionado su tipo de datos, la variable mostrará las entradas de la estructura simplemente haciendo clic en el pequeño junto a su nombre. Esto ampliará el contenido de las estructuras contenido, y también puedes cambiar algunos de los valores de las variables mostradas simplemente haciendo doble clic en sobre el valor y dándole uno nuevo.

NOTA: no se pueden reasignar matrices o estructuras de datos de esta manera, pero sí se pueden cambiar algunos de sus contenidos (no se pueden cambiar los valores "clave" en los mapas, ni el valor de prioridad para las colas de prioridad).

A continuación puede encontrar una lista de cada una de las diferentes ventanas del reloj junto con una explicación de para qué sirven y cómo utilizarlas:

Locales

La ventana de Locales muestra las propiedades del paso actual, que incluye el nombre del evento actual (o script) que se está actual (o script), el desplazamiento en el código de la Máquina Virtual (VM ) para el script actual (el acrónimo "PC" es para "contador de programa"), la instancia "propia" que ejecuta el bloque de código, y la "otra" instancia en el evento. Esta última parte sólo es válida cuando el evento es de colisión o hay un "with" utilizado en el código. Si la instancia está en colisión, "other" mostrará la otra instancia (y sus variables) en la colisión, pero si es un with entonces la instancia "Self" se convierte en la que está ejecutando el código, mientras que "Other" es entonces la instancia que realmente contiene el with. En todas las demás ocasiones, "otro" simplemente mostrará los mismos datos que "Yo".

Observe que las instancias "Self" y "Other" pueden expandirse haciendo clic en para ver una lista de todas las variables locales y de instancia que están declaradas actualmente para el bloque de código que se está recorriendo. Si hace clic con el botón derecho del ratón en sobre un valor, puede establecer su tipo para ver los detalles de las estructuras de datos.


Globales

La ventana de Globales mostrará una lista de todas las variables globales actualmente declaradas y sus valores. Si hace clic con el botón derecho del ratón en sobre un valor, puede establecer su tipo para ver los detalles de las estructuras de datos.


Relojes

Esta ventana es para añadir una variable específica que desea controlar. Puede ser una variable global o de ámbito de instancia , una variable incorporada o incluso un array o una estructura de datos. Si el código de instancia que está siendo Si el código de instancia que se está recorriendo tiene la variable observada, se mostrará aquí, lo que significa que puede seguir fácilmente un valor sin tener que buscarlo en las ventanas Locales o Globales . Si haces clic con el botón derecho del ratón en sobre un valor, puedes establecer su tipo, borrarlo o eliminar todas las observaciones de la ventana.


Instancia

Como su nombre indica, esta ventana mostrará todas las variables de la instancia, las variables locales para el evento de instancia, las variables incorporadas y los structs asociados con la instancia que se está recorriendo en la ventana Fuente (si se está haciendo referencia a una instancia utilizando la palabra clave "other" o la sentencia with , entonces esta ventana mostrará las propiedades de esa instancia). Si hace clic con el botón derecho del ratón en sobre un valor, puede establecer su tipo, eliminarlo o borrar todos los relojes de la ventana.


Todas las instancias

Si necesita ver todas las instancias que están actualmente en la sala, puedes seleccionar esta ventana y se listarán. También puedes ver el estado de cualquier variable incorporada, variables de instancia o structs que tengan, y puedes hacer clic con el botón derecho hacer clic en en un valor para establecer su tipo.


Instancia seleccionada

Esta ventana es para cuando seleccionas una instancia de tu juego. Esto se hace simplemente poniendo en pausa el juego y luego haciendo clic en cualquier instancia visible dentro de la ventana del juego. La instancia que se encuentra bajo el cursor del ratón tendrá entonces su información mostrada en la ventana de la Instancia Seleccionada Instancia seleccionada, donde puedes ver sus variables incorporadas, variables de instancia y estructuras, al igual que en la ventana de instancias.


Estados de Renderización

Este reloj muestra toda la información disponible para el estado del fotograma actual que se está renderizando, como el alfa de dibujo o el color de relleno. Esto no puede ser actualizado en tiempo real, por lo que tendrá que pausar el juego para actualizar y obtener los detalles del cuadro. detalles.


Superficies / Texturas

Este reloj le permite ver exactamente lo que se está dibujando en cualquier superficie que tengas en tu juego (incluyendo la superficie de la aplicación), así como ver laspáginas de texturasque están en la memoria. Puedes seleccionar texturas o superficies en el menú de la parte inferior del reloj, y luego pasar el ratón por encima del ID de la textura o superficie para que se muestre en una ventana emergente:

Tenga en cuenta que el depurador debe estar en pausa antes de que estos se actualicen.


Perfil

Esta ventana es donde puedes ver el perfil actual del sistema para el juego que se está probando y es donde puedes comprobar todo en tu juego, desde los eventos completos hasta las llamadas a funciones individuales para ver cómo están funcionando y detectar y detectar áreas problemáticas o cuellos de botella. Tampón

Si has inicializado algún buffer en tu juego, esta ventana te mostrará los datos del buffer. Puede seleccionar cualquier búfer creado estableciendo el valor del ID del b úfer, que comenzará en 0 para el primer búfer creado en su juego, y se incrementará en 1 para cada búfer consecutivo:

Si hace clic con el botón derecho del ratón en en la ventana de datos o hace clic en el botón de visualización de datos, puede seleccionar cómo se muestran los datos, estableciendo la alineación y el tipo, y también puede establecer cómo los datos desde el cuadro de selección de columnas. Aquí hay una ventana desplegable en la que puede seleccionar de una lista predeterminada de diseños.
