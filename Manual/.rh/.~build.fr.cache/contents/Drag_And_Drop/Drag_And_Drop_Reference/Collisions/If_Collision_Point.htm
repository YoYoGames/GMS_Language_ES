Si el punto de colisión Esta acción se utiliza para comprobar si hay una colisión con una o más instancias de un objeto dado en un punto específico de la sala y se evaluará como verdadero si se detecta una colisión, o falso en caso contrario. Se indica el objeto a comprobar y la posición a comprobar - que puede ser una posición absoluta en la sala o una posición relativa a la instancia - y también puede comprobar la bandera No para comprobar si no hay una colisión en la posición dada, lo que significa que la acción sólo se evaluará como verdadera si no se encuentran colisiones, de lo contrario se evaluará como falsa. También puede establecer la opción Excluirse a sí mismo para excluir la instancia que llama de la comprobación de colisiones. A continuación puede marcar/desmarcar la opción de devolver la lista. Si se marca esta opción, la acción creará y rellenará una estructura de datos de lista con todas las instancias en colisión con el punto y establecerá la variable Objetivo con el ID de la lista DS. Si no está marcada, sólo establecerá la variable Objetivo a un valor menor que 0 cuando no se detecte ninguna colisión, o el valor de ID único de una de las instancias en la colisión (tenga en cuenta que si hay varias instancias en colisión, no tiene forma de saber qué ID de instancia se devolverá cuando no se utilice una lista). Si ha seleccionado devolver una lista - y la variable Destino no está configurada como Temp - entonces también puede seleccionar la opción Destino libre. Si se establece como verdadero, la lista contenida en la variable de destino se destruirá y se creará una nueva lista (que se devolverá a la variable de destino) cada vez que se llame a la acción. y se creará una nueva lista (que se devolverá a la variable de destino) cada vez que se llame a la acción. De este modo, puede reutilizar la misma variable sin preocuparse por las fugas de memoria, ya que, si no marca la opción Free Target, se hace responsable de destruir la lista antes de volver a utilizar la variable en la acción. Tenga en cuenta que, independientemente de si esta opción está marcada o no, si ha configurado la acción como Devolución de lista, la variable de destino siempre contendrá un ID de lista, y como tal, tendrá que ser liberada cuando la instancia sea destruida o la sala termine (puedes usar el evento Clean Up para esto, por ejemplo, junto con la acción Free Data Structure ). Vale la pena señalar que puedes configurar la opción de Liberar Objetivo para utilizar una variable predefinida o una expresión y así controlar cuándo se libera la estructura de datos de la lista estableciendo la variable en otro lugar (o teniendo diferentes resultados para la expresión) siempre que se evalúe como verdadero/falso. Por último, debe proporcionar la variable de destino que desea que contenga el valor devuelto o el ID de la lista. Si marca la variable Target para mantener el valor devuelto como una variable Temp (local), entonces la acción creará esta variable para mantener el valor devuelto sólo hasta el final del evento, en este caso la opción Destino Libre no hace nada y usted será responsable de destruir la lista DS si esa opción ha sido marcada. Si no se suministra ninguna variable de destino y la opción Lista de Retorno está marcada, no se creará ninguna lista. IMPORTANTE! Las colisiones sólo se registrarán para aquellas instancias que tengan una máscara de colisión válida, es decir, que tengan un sprite asignado al sprite_index, o un sprite asignado al mask_index. Si las instancias del objeto que se no tienen una máscara de colisión, la colisión no será detectada, independientemente de lo que la instancia esté dibujando en ese momento.
