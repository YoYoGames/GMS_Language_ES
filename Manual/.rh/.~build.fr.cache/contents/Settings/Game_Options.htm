Opciones de juego Las opciones del juego se dividen en dos secciones, las opciones principales y la configuración de la plataforma. Las opciones principales comienzan con las opciones generales, que es donde se pueden establecer ciertas cosas que afectarán al juego independientemente de la plataforma que que está diseñado para ejecutarse. La ventana tiene las siguientes secciones: Usar DnD™ como tipo de script por defecto: Cuando esta opción está marcada, todos los nuevos scripts y eventos de objetos se abrirán utilizando el editor DnD™en lugar del editor de código. Fotogramas por segundo del juego: Este valor establece el límite del número de fotogramas por segundo que GameMaker Studio 2 debe tratar de mantener en sus juegos cuando se ejecuta. Este valor representa el número de veces que el bucle de juego se ejecutará en un segundo (un solo bucle de juego se considera un solo "fotograma") y se establece en un valor predeterminado de 60, aunque 30 se utiliza a menudo también. Hay muchos argumentos sobre qué velocidad de sala es la ideal, pero la respuesta real depende de qué tipo de juego que estés haciendo, el tamaño de la sala, el número de instancias y lo intensivo que sea el código. Por ejemplo, si tienes una sala grande y una velocidad de sala de 30, el juego puede parecer "entrecortado", ya que cuanto más grande sea el área que estás viendo pero si el juego es intensivo, establecer la velocidad de la sala a 120 puede hacer que el juego se "retarde", ya que los fps caen debido a la cantidad de procesamiento que se realiza. Así que tenga cuidado con este valor y trata de ajustarlo siempre a lo que tu juego necesita. Color de dibujo por defecto: El color por defecto para dibujar texto o foros (y otras cosas) cuando no se ha establecido ningún color mediante código. Color de dibujo por defecto fuera de la región de la sala: El color por defecto que se utilizará para aclarar el fondo de cualquier área fuera de la sala de juego. Generar imágenes del proyecto: Al hacer clic en esta opción se abrirá la herramienta de generación de imágenes del proyecto. Fecha de inicio del proyecto: La fecha en que se inició el proyecto. Tiempo de uso del proyecto: El tiempo que ha pasado desde que inició el proyecto. GUID del proyecto: Es el identificador único que se crea para cada proyecto dentro de GameMaker Studio 2. Si desea cambiarlo, puede hacer clic en el botón Crear nuevo GUID para generar uno nuevo. Generar mipmaps para páginas de textura separadas: Esta opción hará que GameMaker Studio 2 cree mipmaps para aquellos sprites que tengan marcada la opción de Página de Textura Separada en la configuración de Textura del Editor de Sprites. Activar el mipmapping significa que la página de textura original para el sprite debe ser escalada y filtrada en múltiples resoluciones dentro del archivo de textura. Esto está normalmente desactivado, pero si usted está usando vistas para acercarse o alejarse en grandes áreas de la sala de juegos, o está trabajando con un juego 3D, entonces usted puede querer habilitar esto y luego usar las Funciones de Mipmapping para ajustar cómo el juego trata con los diferentes niveles de mipmap. Tenga en cuenta que si usted no está marcando los sprites como el uso de las páginas de textura individual, todavía puede activar mipmapping en una base de grupo de texturas desde el Gestor de Grupos de Texturas. Esta opción está desactivada por defecto. ID deSteam: Si estás creando un juego que va a ser distribuido a través de Steam, entonces necesitas suministrar el ID de la aplicación de Steam aquí. Asegúrate de haber añadido la ruta al SDK de Steamworksen las Preferencias de la Plataforma cuando utilices esta opción, y asegúrate de que tienes todo configurado correctamente en el panel de Steamworks. Habilitar el control de fuentes: Si desea que el proyecto esté bajo SCM (Source Control Management) entonces necesita marcar esto. Está desactivado por defecto (ver aquí para más información sobre el control de fuentes). Autor: El nombre del autor del juego (o del editor o del estudio). Última modificación del proyecto: Esta sección simplemente muestra la fecha/hora de la última vez que se modificó el proyecto (no es editable). Aceptar la licencia de Spine: Aquí puedes marcar la casilla para que se te muestre la licencia de Spine Runtime y aceptarla. Esto le permitirá utilizar sprites de Spine en su juego (esta licencia también se mostrará la primera vez que intente añadir un sprite de Spine a tu proyecto y no hayas comprobado la licencia aquí de antemano). El Navegador de Activos también tendrá otras Opciones de Juego disponibles para usted dependiendo de las plataformas de destino que están disponibles para la licencia que usted tiene y no todos ellos pueden estar disponibles o visibles. Existen las siguientes opciones de juego específicas para cada plataforma: Windows macOS Ubuntu Windows UWP HTML5 Android iOS tvOS NOTA: Las diferentes opciones de juegos para consola no aparecen aquí debido a las diferentes restricciones legales que mantienen las diferentes compañías. Si tienes una licencia de consola, puedes obtener esta información en el servicio de asistencia de YoYo Games, pero primero debes ponerte en contacto con YoYo Games con tu correo electrónico de identificación de la consola y solicitar permiso para ver las diferentes secciones. Algunas licencias de GameMaker Studio 2 también pueden mostrar opciones de Juego para el Amazon Fire objetivo. Se trata de opciones heredadas, ya que las licencias más recientes admiten la plataforma Fire directamente utilizando las Android opciones de juego.
